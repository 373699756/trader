# 荐股全链路实时性与安全等价优化计划

状态：已实施、Review 并通过仓库验收

基线：`a401bbc7a8b4c91a8ef24d0a7b9ae77c9336d385`

## 1. 目标与不可改变项

荐股实时链路采用“不可变数据版本、事件驱动增量计算、并行数据获取、单一顺序提交、
本地结果优先发布”。本批只优化调度、缓存复用、诊断和发布时延，不改变：

- 基础硬过滤、候选获取、完整字段硬过滤、分板本地评分、风险动作、TopK 和行业集中度的
  既有业务顺序；
- 三板样本、板块可靠度、公司风险、字段完整性、时效性、动作门槛、固定 68/32 融合、
  冻结时间、冻结不可覆盖和快照哈希规则；
- 行业只在最终稳定选择时限制同行业最多两支，不参与评分；
- long 只展示当前观察快照，不冻结、不写推荐历史、不进入本批调度改造。

分板本地评分后，每策略、每次投影最多选出 **28 支**可靠候选送 DeepSeek 复核；未送审或
复核失败的候选继续使用本地评分，不因模型网络或预算退出荐股链。

## 2. 固定业务链路

1. 获取全市场行情和历史数据。
2. 执行初始硬过滤、分板候选预选和候选评分。
3. 获取候选行情、历史、结构化研究、盘中和尾盘字段。
4. 在完整字段上重新执行硬过滤，包括大股东减持、财务造假、立案调查、重大违法、资金
   占用、违规担保、强制退市、解禁、质押和基本面恶化。
5. 分板本地评分，选出最多 28 支可靠送审候选。
6. 本地 P6 草稿立即发布，DeepSeek 异步复核后才能形成相同基础版本的 hybrid 更新。
7. 执行风险阻断、字段完整性、行情时效、板块可靠性、观察窗口和分数门槛。
8. 可执行与观察候选分池，最后执行 TopK、板块比例和同行业最多两支约束。

## 3. 版本化实时执行

- 生产配置启用 `versioned_dag`，旧 schema v5 配置缺省保持 `serialized`，可通过配置重启
  回退；两种模式调用完全相同的过滤、评分、融合、动作和排名函数。
- 数据源完成后携带交易日、阶段、策略、市场/配置/schema 版本和内容身份进入后续计算；
  迟到、跨日、跨阶段、已被新版本替代或冻结后的结果禁止发布。
- 候选行情、市场新闻和公司风险变化立即提交同版本本地计算事件；计算使用独立 15 秒执行
  预算和 38 秒含排队预算，不占用来源事件自己的截止时间；周期 score 保留为漏事件恢复心跳。
- 本地评分完成即由主事件顺序提交 P6；DeepSeek 在独立池执行，结果通过有界内部事件返回
  主提交器。只有当前未冻结 P6 的交易日、数据版本、配置版本和策略身份仍一致时才发布
  hybrid；复核失败时发布本地降级终态，不能让页面永久停留“模型复核进行中”。
- 每策略只保留一个有效待提交 hybrid；旧结果被新版本替代后只记计数，不修改快照。
- 全市场、候选、研究和风险 I/O 继续使用现有有界来源通道；未知影响范围扩大重算，禁止
  为性能缩小安全检查范围；公司风险触发的重算保留 risk 优先级和预留队列容量。

## 4. TopK 与收盘专项

- TopK 报价使用一个独立 worker 和一个 latest-wins 待处理槽，不再等待串行主事件结束。
  覆盖层只更新价格、涨跌幅和行情时间，提交前校验交易日和基础 snapshot ID，绝不改变
  分数、动作、排序或正式快照身份。
- `close_quotes` 分别统计排队、执行、恢复已有冻结、P6 固化、全市场获取、完整性校验、
  预选、逐策略准备/评分/提交、long 观察刷新和结算耗时。
- 收盘缓存只依据当前不可变批次自身校验：同一交易日、14:59 后且不晚于校验时刻、正价格、
  三板各至少 100 只、20 日历史及成交额中位数完整、merge epoch 一致、无 partial/跨日/
  未来数据。历史 `last_error` 不再否定一个已经通过校验的批次。
- 完整缓存路径不再远程刷新全市场；冷启动最多一次全市场刷新。P6 收盘固化优先复用缓存
  中完整的目标代码，只补取缺失代码。
- 冻结事件继续优先且不等待普通迟到任务；冻结后的评分、DeepSeek 和覆盖层由现有 CAS
  与身份校验拒绝。

## 5. Web 中文诊断

- API 保留原始原因码用于兼容和审计，Web 通过受控字典显示简短中文。
- 当前已知原因及带板块前缀的可靠度原因全部翻译；未知原因、`last_error` 和外部异常不得
  原样进入页面，统一显示中文兜底，原始值仅进入浏览器诊断。
- 页面区分正常分数不足、风险或执行阻断、集中度限制、数据不完整和链路超时；正式荐股
  只统计 `executable`，观察项单独计数。
- 状态接口增加执行模式、独立覆盖层通道、排队/执行耗时，以及每策略正式/观察数量。

## 6. 验收

- 28 支送审上限、旧冻结回放、168 次物理请求硬上限及 long 零调用均通过确定性测试。
- 串行与 `versioned_dag` 对同一输入的候选、过滤、分数、风险、动作、排名、降级原因和
  哈希一致；随机完成顺序、迟到、跨 epoch、冻结竞态、慢源和队列压力均不改变安全结果。
- 本地 P6 不等待 DeepSeek；候选特征版本接纳到本地 P6 的 P95 目标为 2 秒。外部网络耗时
  单独统计，全市场规范版本到本地 P6 的内部处理 P95 目标为 2 秒。
- TopK 权威行情到浏览器目标不超过 2 秒；`close_quotes` 慢源基准至少降低 30%，完整缓存
  路径远程全市场请求为零，冷启动最多一次。
- 运行 format、Ruff、mypy、完整 pytest、package、仓库外 wheel 安装及
  1280x720、1440x900、1920x1080 桌面验收；独立 Review 到零已知发现后单提交并推送。
