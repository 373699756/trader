# 荐股全链路收益优化计划

> 状态：非权威执行计划。产品、架构、时间线、API、运维和验收契约以
> `docs/software-business-design.md` 为准；候选、过滤、公式、风险、融合和排名策略以
> `docs/recommendation-strategy.md` 为准。本文件描述待验证的优化路线，不表示活动策略
> 已经切换；实际晋级必须先更新对应权威契约。

## 1. 目标与审查结论

本计划审查从硬过滤、候选预选、本地评分、DeepSeek 融合、动作判定、TopK 到 Web 展示的
完整数据链路。优化目标不是增加推荐数量或提高表面分数，而是在扣除成本后提高净超额
收益，同时降低严重回撤率。

当前首要瓶颈是数据完整性，而不是评分权重。审查时的最新运行快照中，5,548 只股票有
5,349 只缺少 20 日流动性历史；tomorrow 和 d25 仅产生 4 个和 2 个观察项，板块数据可靠度
只有 0.50–0.75。v2 运行库仅有 5 个冻结快照，尚无足够的有效配对结算样本。因此，现阶段
直接降低门槛或调整权重会放大低覆盖数据的噪声，不能据此宣称收益改善。

审查确认以下结构性问题：

1. 低、中等级减持、解禁、质押和负面公告已经被硬过滤，后续分级软扣分规则实际无法
   生效。
2. 候选分与最终本地分使用不同组件，固定的 50 分、30% 缺失率和 Top120 可能提前淘汰
   最终高分股票，但当前没有候选召回率或反事实结果审计。
3. 缩量回踩和放量突破共用全部十个输入。缺少突破专属字段时，已经成立的回踩形态也会
   被判定为未知，反之亦然。
4. 缺失值在评分中经常按中性 50 处理，在可靠度和下行保护中又按降级处理，同一缺失在
   不同阶段的语义不一致。
5. 动作阈值、可靠度门槛和 TopK 约束均为静态规则，尚未通过足够的点时样本外结果证明
   其收益和回撤贡献。
6. Web 能显示最终动作和部分风险，但没有完整呈现股票从过滤、候选、评分、融合到动作
   判定的原因链，也没有区分策略低分与数据不足。

外部研究只用于确定验证方法，不直接移植参数。非线性交互可能有预测价值，但金融信号
噪声较高；多个候选方案必须控制多重检验，收益评估必须包含交易成本。参考：

- [Empirical Asset Pricing via Machine Learning](https://academic.oup.com/rfs/article/33/5/2223/5758276)
- [... and the Cross-Section of Expected Returns](https://www.nber.org/papers/w20592)
- [Transaction Costs of Factor Investing Strategies](https://papers.ssrn.com/sol3/papers.cfm?abstract_id=3359947)

## 2. 优化原则

- 生产 v17 在新版本通过收益门禁前保持不变；所有规则调整先作为 v18 影子策略运行。
- 固定保留融合公式
  `clamp(local_score * 0.68 + deepseek_score * 0.32 - deepseek_risk_penalty, 0, 100)`，
  并使用 `ROUND_HALF_UP` 保留两位小数。
- 不改变冻结时间、同日正式记录不可覆盖、188 次 DeepSeek 物理请求预算及结构化证据
  校验。
- 不使用未来数据，不因当天表现自动调权、自动降低门槛或自动晋级。
- 以净超额收益和严重回撤作为联合目标；模型分数、推荐数量和命中率只能作为辅助指标。
- today、tomorrow 和 d25 参与收益优化；long 继续作为观察池，不纳入短线收益晋级。

## 3. 第一阶段：建立可信基线

### 3.1 数据覆盖恢复

先修复历史预热和板块横截面覆盖，达到以下条件后才开始收益参数比较：

- 支持股票的 20 日历史覆盖率连续 5 个交易日不低于 95%；
- 主板、创业板和科创板当日横截面有效样本分别不少于 100；
- 关键行情、复权、证券身份、结构化风险和时间戳均通过点时校验；
- 数据降级时仍保留最近有效快照，但降级结果不能伪装成正式可执行结果。

### 3.2 决策轨迹和影子样本

为支持股票保存紧凑、不可变的点时决策轨迹，至少包括：

- 硬过滤结果、全部命中原因和结构化风险等级；
- 候选组件、候选分、缺失掩码和板块可靠度；
- 本地评分组件、本地风险事实和风险扣分；
- DeepSeek 是否参与、置信覆盖、结构化风险扣分和融合模式；
- 动作判定、下行保护结果、板块排名、TopK 跳过原因和最终排名；
- 数据、配置、策略、规则、融合、schema 和引擎版本。

同一份点时输入同时生成 v17 基线与 v18 影子结果。影子策略复用已有 DeepSeek 结果，不得
增加物理 HTTP 请求，也不得修改冻结记录、冻结 JSON 或普通 Web 推荐。

### 3.3 结果结算

沿用权威策略文档的结果定义：

- today/tomorrow 结算 T+1 最低价、收盘价、全市场等权基准、20bp 往返成本后的净超额
  收益、MAE 百分比和 `MAE/ATR20`；
- d25 结算 T+2、T+3、T+5 收益及 T+1 至 T+5 的最低价和 MAE；
- 同时报告 50bp 和 100bp 成本压力结果；
- 停牌、交易日缺失、价格不连续或公司行为无法可靠处理时标记
  `insufficient_data`，不得猜测补值。

## 4. 第二阶段：优化判断条件

### 4.1 硬过滤与结构化风险分层

以下条件继续保持硬过滤：

- 不支持的证券、ST/退市、停牌；
- 价格、成交额、报价时间或证券身份无效且无法验证；
- 上市前 5 个交易日、重新上市首日、退市整理首日；
- 一字涨跌停、黑名单、重大监管风险；
- 关键交易规则或高等级结构化证据确认不可执行。

结构化风险调整为分层组合：

- 高危监管、严重财务恶化和高等级事件保持 `unavailable`；
- 低、中等级减持、解禁、质押和负面公告进入有界软扣分；
- 同组风险采用最高等级，禁止硬过滤和软扣分重复计算；
- 结构化风险数据缺失或证据不可验证时只能进入观察池，不能静默通过。

### 4.2 热度组合条件

一字板继续硬过滤。普通热度不再单独淘汰，而是作为组合条件：

- 主板涨幅超过 8% 或成长板超过 16%，并且同时出现临近涨停、冲高回落或弱收盘时，
  降为观察；
- 只有普通高热度但量价、收盘结构和下行保护正常时，仅应用现有过热软扣分；
- 当前热度硬过滤规则保留为独立影子对照，未经验证不得直接替换生产规则。

### 4.3 流动性条件

生产环境暂时保留 20 日成交额中位数 5,000 万元硬底线。影子验证同时比较 3,000 万、
5,000 万和 1 亿元三档，并按策略、板块分别报告：

- 净超额收益；
- 严重回撤率；
- 50bp/100bp 成本压力；
- 可执行样本数、换手和集中度。

只有降低流动性门槛后仍通过成本和回撤门禁，才允许修改生产阈值。

### 4.4 入场形态

共同趋势前提仍为：

- `MA5 >= MA10 >= MA20`；
- MA20 斜率向上；
- 当前价格不低于 MA20。

两个形态分支独立判断：

- 缩量回踩只依赖 MA5/MA10 支撑距离和相对 5 日均量；
- 放量突破只依赖前 20 日高点、量比、收盘位置、突破偏离和行业上涨宽度；
- 缺少另一分支的专属输入，不得使已经完整的分支失效；
- 共同趋势破坏得 0 分；趋势成立但完整分支均未确认得 50 分；两个分支均缺少必要输入
  时保持 `null`。

形态分数使用配置化分段函数形成连续分，而不是只在阈值两侧从 100 跳到 50：

- 回踩支撑在 MA5 1% 或 MA10 2% 内为满分，分别向 2% 和 4% 线性衰减；
- 缩量比不高于 0.70 为满分，到 1.20 线性衰减为 0；
- 突破偏离 0%–3% 为满分，到 5% 线性衰减；
- 放量、收盘位置和行业宽度分别在现有 2.0、70、60 阈值达到满分，并向较弱区间线性
  衰减；
- 分支内使用最弱子条件作为该分支确认分，避免单个强指标掩盖失败条件。

### 4.5 缺失值和可靠度

所有评分组件同时返回数值和覆盖率，统一使用：

```text
effective_component = 50 + coverage * (raw_component - 50)
```

关键交易、风险或下行字段缺失仍转观察。一般因子缺失只向 50 收缩并降低可靠度，不能
提高分数；删除一个有效正向输入不得使最终分或动作等级上升。

## 5. 第三阶段：候选、评分、融合和 TopK

### 5.1 候选召回

不直接通过降低候选 50 分门槛扩池，而是增加“最终分乐观上界”：

- 已知组件使用真实值；
- 尚未获取的研究组件按理论最高值计算上界；
- 候选分不足但上界仍可能达到观察门槛的股票继续参加每板每策略 Top120 竞争；
- 每日记录最终 Top10 被预选遗漏的股票和原因；
- 历史影子验证的 Top10 候选召回率必须不低于 99%。

### 5.2 本地评分

第一轮不手工调整本地权重。先按策略、板块和市场状态计算：

- 因子分位组的净超额收益和严重回撤；
- 分数与未来结果的单调性；
- 单因子消融和组合交互；
- 缺失率、稳定性和时间漂移；
- 候选分、本地分与最终分的增量贡献。

仅预注册以下独立变体，避免无限试参：

1. 结构化风险分层；
2. 热度组合条件；
3. 分支独立的连续入场形态；
4. 候选乐观上界与召回保护；
5. 上述四项的合并挑战者。

### 5.3 DeepSeek 融合

- 固定 0.68/0.32 融合权重和现有结构化证据映射；
- 分别结算 local-only 与 hybrid 的同日同股配对结果；
- 检查 DeepSeek 各维度的覆盖率、校准、增量净收益和严重回撤贡献；
- 只有 hybrid 同时提高净超额收益并降低严重回撤时，才允许调整版本化的置信覆盖门槛；
- 不允许自由文本直接扣分，不增加运行时自动调权。

### 5.4 动作门槛和 TopK

- today、tomorrow、d25 分别离线校准动作门槛；
- 板块专属门槛只有在各板块拥有至少 100 条有效配对样本时才可启用，否则使用策略全局
  门槛；
- 现有板块占比和竞争组限制继续作为生产基线；
- 影子策略比较当前贪心 TopK 与按保守收益下界排序的组合；
- 新组合必须在相同集中度、成本和可执行约束下胜出，且不能增加严重回撤。

## 6. Web 决策链展示

推荐 API 增加顶层 `quality_summary`：

- 全市场数量；
- 硬过滤、候选、完成评分、模型复核、可执行和观察数量；
- 20 日历史覆盖率；
- 三板样本量和可靠度；
- 当前策略验证状态。

每条推荐增加 `decision_trace`：

- 候选分、乐观上界和候选排名；
- 数据覆盖率、板块可靠度和关键缺失项；
- 入场形态类型、形态分和未通过条件；
- 本地风险、模型风险及实际扣分来源；
- DeepSeek 是否实际参与融合；
- 动作门、下行保护和集中度跳过原因。

Web 必须明确区分“数据不足”“评分不足”“风险限制”“下行保护”和“集中度替换”。最终分
继续标注为排序分，不显示成上涨概率或预期收益。历史页仅对已完成结算的冻结推荐显示
净超额收益和 MAE/ATR；影子结果、未结算结果和数据不足结果不得显示为实际收益。

## 7. 接口与数据契约

计划新增以下内部不可变类型：

- `DecisionTrace`
- `RuleDecision`
- `ShadowRecommendation`
- `StrategyValidationResult`

SQLite 新增影子决策、影子结果和验证运行表，不修改现有冻结推荐及其结算结果。

推荐 API schema 以新增字段方式升级，并保持旧客户端兼容。Web 仍然只调用应用层只读
用例；HTTP 请求不得触发行情抓取、评分、DeepSeek、结果结算或写库。

策略晋级时创建新的策略、规则、配置、融合和 schema 版本，同步更新
`docs/software-business-design.md`、`docs/recommendation-strategy.md`、测试和
`CHANGELOG.md`，不得原地改写 v17。

## 8. 验证与晋级门禁

自动测试至少覆盖：

- 主板 8.00/8.01、成长板 16.00/16.01 及组合热度边界；
- 结构化风险低、中、高等级、同组去重和证据缺失；
- 回踩与突破分支独立、连续分边界和趋势破坏；
- 候选 50 分、30% 缺失率、Top120、乐观上界和召回审计；
- 可靠度 0.85、动作门槛及关键缺失转观察；
- 删除正向输入不能提高分数或动作；
- 高危风险不能被高分覆盖，低、中风险不能被硬过滤提前吞掉；
- 影子回放无未来数据、结果可重复、哈希一致且不新增 DeepSeek 请求；
- 正式与观察分池、TopK 集中度、冻结竞态和结果结算幂等；
- 三档桌面分辨率的决策轨迹展示。

收益验证采用按交易日分块的固定种子配对 bootstrap。预注册变体需要执行多重检验校正。
每个策略只有同时满足以下条件才允许晋级：

- 至少 60 个交易日；
- 至少 300 条有效配对样本；
- 净超额收益差的 95% 置信下界大于 0；
- 严重回撤率低于 v17；
- Top10 候选召回率不低于 99%；
- 20bp 主口径和 50bp/100bp 压力结果均已报告；
- 所有适用质量、测试、构建、安装和桌面 UI 门禁通过。

未达到门禁时继续保留 v17，v18 只能标记为研究或影子结果，不能描述为收益改善。

## 9. 明确不变项

- 固定融合公式和两位小数舍入；
- DeepSeek 188 次每日物理请求上限及独立预算桶；
- DeepSeek 自由文本不得直接决定扣分；
- today 11:20、tomorrow/d25 14:50 冻结及正式记录不可覆盖；
- 15:00 后 `close_fallback` 的既有规则；
- `Asia/Shanghai` 业务时间和可注入时钟；
- 数据源或 DeepSeek 失败不能阻塞本地推荐和只读 Web；
- 产品仅用于本地研究，不连接券商、不提供真实下单，也不保证收益。
