> 修订时间：2026-07-14

> 当前状态：研究计划，不代表允许修改冻结生产基线或形成实盘指令  
> 生产研究主策略：`tomorrow_picks`，冻结组合口径：每日 Top5

## 零、结论与执行原则

原计划关于 PIT、交易成本、影子运行、逐模块 OOS 验证和可回退开关的方向正确，但执行顺序需要重排。

**当前最高优先级不是开启高级模块，而是建立可用于研究的当前版本数据资产。**

截至 2026-07-14 的只读审计显示：

- 当前策略版本、当前验证基线下，三种策略的 `live_weight_samples` 均为 **0**；
- 本地日线历史只有 35 只股票、5,600 行，日期为 2025-11-11 至 2026-07-09；
- 最新全市场快照约 5,527 只，但 `main_net_flow_1d`、`order_imbalance`、基本面质量等拟新增字段的有效覆盖为 0；
- DeepSeek 研究库只有一次 `no_evidence` 批次，没有有效候选或反事实收益；
- P0 启动前冻结生产基线为 `drift_detected`，存在 3 个未登记的运行开关；
- 57 个相关测试通过，只能证明当前代码契约成立，不能证明收益有效。

因此本计划遵循以下顺序：

```text
修复冻结基线与数据链路
    ↓
建立 14:30 点时全市场数据和真实前瞻样本
    ↓
建立统一、可审计的实验与多重检验框架
    ↓
小范围优化退出和已可用因子
    ↓
预期收益 / 概率 / Meta / Regime / IC
    ↓
DeepSeek、事件 Alpha、Ensemble
    ↓
ML 排名和组合优化
```

核心目标是提高**成本后、风险调整后的日级 Top5 组合收益**，而不是提高单票评分、命中率或回测展示值。

## 一、策略边界

| 策略 | 角色 | 是否独立执行 | 主要评价单位 |
|---|---|---:|---|
| `short_term` | 盘中观察池和异动提示 | 否 | 对 `tomorrow_picks` 候选质量的增量贡献 |
| `tomorrow_picks` | T 日 14:30 后参考入场、T+1 退出 | 是 | 每日冻结 Top5 等权净收益 |
| `swing_picks` | T 日 14:30 后参考入场、T+2 至 T+5 退出 | 是 | 每日组合净收益及持仓期回撤 |

禁止把 `short_term` 当成第三个独立盈利策略，也禁止用它的观察收益替代可执行策略收益。

研究统计单位以**交易日组合**为主。同一天多只股票受共同市场冲击，不得简单视为多个独立样本。

## 二、当前实现的真实状态

### 2.1 已经开启或已接线

| 能力 | 当前状态 | 处理方式 |
|---|---|---|
| 市场冲击 | `ENABLE_MARKET_IMPACT=True` | 保留，校准参数和容量假设 |
| 幸存者修正 | `ENABLE_SURVIVORSHIP_CORRECTION=True` | 保留并持续审计退市样本 |
| Regime 主题上限 | `ENABLE_REGIME_THEME_CAP=True` | 保留，验证对回撤的真实贡献 |
| 组合分仓 | 已有置信度/风险/波动/主题/相关组/总仓约束 | 不用 Max-Sharpe 直接替换 |
| 预期收益排序 | 已有 artifact 和晋级门控，生产关闭 | 有样本后优先验证 |
| 概率校准 | 已接入诊断展示 | 继续诊断，暂不驱动交易 |
| Factor IC 加权 | 评分端已有入口，生产关闭 | 先修正 IC 统计方法和样本来源 |

### 2.2 不是“打开开关即可上线”

| 模块 | 缺口 | 结论 |
|---|---|---|
| Meta-Labeling | 推荐管线固定 `enforce=False`，模型按请求临时训练 | 需 artifact、门控和执行接线 |
| 交互项 | 只有 walk-forward 评估，没有生产 artifact/加载路径 | 不能直接开启 |
| Regime 权重 | 开关启用的是预设启发式 profile，不是评估函数拟合结果 | 必须区分静态 profile 与学习模型 |
| Event Alpha | 独立 `research_only`，没有生产排序接线 | 先解决事件证据和 OOS 标签 |
| Ensemble | `attach_ensemble_score` 硬编码 `shadow_only`，主推荐管线未调用 | 开关当前不能改变排名 |
| 增强形态因子 | 可计算，但评分组合未消费 | 必须逐因子接线和消融 |
| 资金流/委比 | 只有字段别名，当前实时覆盖为 0 | 数据可用前禁止接线 |
| 基本面 Alpha | 可抓取和 PIT 过滤，但功能关闭、当前覆盖为 0 | 优先用于 swing 研究，不直接全策略启用 |

### 2.3 需要修正的现有实现

1. `tail_auction.enabled` 在执行策略中硬编码为 `False`，`ENABLE_TAIL_AUCTION_SLIPPAGE` 单独改为 `True` 不会生效。
2. `close_vs_vwap` 实际使用 `(high + low + close) / 3` 作为分母，不是真实成交量加权平均价；应改名为 `close_vs_typical_price`，或用真实成交额/成交量计算 VWAP。
3. 当前 `Factor IC` 把不同日期和股票直接混合相关；应先逐日计算横截面 Rank IC，再评估 IC 均值、稳定性和衰减。
4. 当前所谓 FDR 门控主要是折数正负号检验，不是跨所有候选因子、参数和模块的统一 BH-FDR。
5. 四折下 `q=0.1` 的正负号检验需要 4/4 正面才通过；“3/4 正面并通过 FDR”与当前实现不一致。
6. P0 启动前，冻结基线 manifest 未登记 `DEEPSEEK_ENABLED`、`DEEPSEEK_INTERNAL_SCHEDULER_ENABLED`、`REALTIME_MARKET_SCHEDULER_ENABLED`，导致基线持续漂移。

- 基线漂移已消除，`production_baseline_status.status == "frozen"` 且 drift 为空；
- 任务租约已改为提交后的可见租约，不再用长事务阻塞全部定时任务；
- 行情缺页增加串行修复；新浪兜底已映射换手率、计算振幅，并由腾讯批量补充量比；全链路只读实测 5,523 只有效报价的三项覆盖率均为 100%；

### 3.1 消除冻结基线漂移

实施：

1. 判断 3 个未登记开关属于“影响推荐结果”还是“纯基础设施”。
2. 影响推荐结果的开关写入 production manifest；纯基础设施开关从输出指纹范围排除，但保留运行审计。
3. 冻结 manifest 后生成新的 `production_baseline_id`。
4. 禁止把不同 manifest、策略版本和执行策略的结果混在同一指标中。

验收：

- `production_baseline_status.status == "frozen"`；
- drift 列表为空；
- 同一输入可重放得到相同输出指纹。

### 3.2 打通当前策略版本的真实样本

当前配置版本：

- `tomorrow_picks_v13_missing_history_neutral`
- `swing_2_5d_v5_missing_history_neutral`
- `today_picks_v2_missing_history_neutral`

实施：

1. 检查每日自动快照是否用当前版本写入 `strategy_signals`。
2. 检查候选池、入选行、执行记录、结果标签能否形成完整外键链。
3. 检查 `promotion_eligible`、`label_status`、`validation_baseline_id` 是否与当前策略一致。
4. 修复 unknown/pending 长期不更新、旧版本仍写入或当前版本无信号的问题。

验收：

- 连续 5 个交易日保存当前版本信号；
- 候选池、入选 Top5、执行记录、结果标签均可复现；
- 已到期样本 `unknown_outcome_count == 0`；
- 当前版本样本的 baseline 匹配率为 100%。

### 3.3 接入真实的尾盘成本基线

实施：

1. 将执行策略中的 `tail_auction.enabled` 接到配置，但默认只在新研究基线启用。

验收：

## 四、P1：建设可研究的数据资产（第 1–6 周，持续进行）

截至 2026-07-14 的实施状态：

- 已保存压缩、带 SHA-256 校验的不可变全市场原始行情快照，并同步维护每日证券全集；
- 策略批次和候选行已统一使用 `real_forward`、`intraday_pit_replay`、`daily_proxy_replay`、`legacy_baseline`、`unknown`，缺失来源不再默认算作真实前瞻；
- 自动 14:30 快照失败会按配置重试至 14:50，盘后流水线不再隐式补造快照；
- 数据健康日报已覆盖点时有效率、行情新鲜度、字段覆盖、成本、DeepSeek 和历史市场覆盖；
- 当前运行库仍为 0 个真实前瞻交易日，60/120 日数据门槛尚未满足，因此 P1 状态为 **collecting**，不能宣告完成。

### 4.1 保存真实 14:30 点时快照

生产策略依赖 T 日 14:30 左右的价格、涨速、量比、换手和收盘位置。日线收盘数据无法忠实重建这些输入。

每日必须保存：

- 14:30–14:50 原始全市场报价及来源时间；
- 当日可交易股票全集，而不只是最终候选；
- 硬过滤前后状态和原因；
- 所有评分输入、缺失掩码和观测时间；
- 冻结 Top5 及未入选合格候选；
- 执行策略、目标仓位和成本假设。

如果历史上没有 14:30 数据，使用日线构造的 replay 必须标记为 `daily_proxy_replay`，只能做粗筛，不能用于生产晋级。

### 4.2 建立全市场 PIT 历史

目标：至少覆盖 2 年，并包含退市、ST、停牌和历史指数成分变化。

需要：

- 未复权/复权价格及复权因子；
- 历史证券状态、涨跌停规则和可成交性；
- 每日全市场可交易集合；
- 公告发布时间和财务报告可用时间；
- 真实成交额、换手、行业和市值；
- 如需真实 VWAP/订单流，则保存相应分钟或逐笔数据。

禁止只在“当前留下来的 35 只股票”上优化退出或因子。

### 4.3 样本分层

| 样本类型 | 用途 | 能否生产晋级 |
|---|---|---:|
| `real_forward` | 真实 14:30 冻结信号 | 是 |
| `intraday_pit_replay` | 有真实历史点时数据的回放 | 研究支持，不能替代真实门槛 |
| `daily_proxy_replay` | 日线近似回放 | 只做粗筛 |
| `legacy_baseline` | 旧策略或旧成本结果 | 仅对照 |

### 4.4 数据验收门槛

- Top5 入选样本 PIT 有效率 100%；
- 全市场基础报价覆盖率 ≥ 98%；
- 用于硬过滤和执行的字段覆盖率 ≥ 99%；
- 资金流等可选实时因子覆盖率 ≥ 95% 才允许进入评分实验；
- swing 基本面 PIT 覆盖率建议 ≥ 80%，否则必须采用明确的 missing/neutral 处理并做覆盖率消融；
- 当前 baseline 的真实前瞻样本至少 60 个独立交易日，复杂模型建议 120–250 日。

## 五、P2：统一实验、统计和晋级框架（第 2–4 周）

截至 2026-07-14 的实施状态：

- 实验账本已强制登记样本类型、数据截止点、生产/策略/执行版本、特征 schema、全部候选参数、试验次数、成本情景及预定通过/失败条件；
- 已预注册首个 Top5 expected-return challenger，目前仅为 `forward_collection`，没有填写或伪造收益结论；
- 组合报告已保存 baseline/challenger 同日净收益增量，并使用移动块 bootstrap 计算置信区间；
- BH-FDR 已按完整已登记试验族统一校正，未报告的已声明试验按未通过计入；同时报告并门控 Deflated Sharpe Ratio；
- OOS 报告、artifact 晋级和策略门禁已消费配对增量、FDR、DSR 结果；正面折数仅保留为稳定性诊断；
- 当前运行库的 fold prediction、OOS report 和真实 OOS 日均为 0，P2 状态为 **blocked_by_forward_data**，不能宣告完成。

### 5.1 实验必须预注册

每个实验写入 `experiment_registry`：

- 实验 ID、假设和唯一主指标；
- 数据截止时间和样本类型；
- 生产 baseline、策略版本、执行策略版本；
- 特征 schema 和代码指纹；
- 所有候选参数及总试验次数；
- Top-K，生产固定 K=5，K=3/10 只能做敏感性分析；
- 成本情景；
- 预定通过/失败条件。

未登记的临时尝试也必须计入试验次数，防止只报告最好结果。

### 5.2 时间序列 OOS

统一使用按交易日分组的 expanding/rolling walk-forward：

- `tomorrow_picks`：训练和测试间 purge/gap ≥ 1 个交易日；
- `swing_picks`：purge/gap ≥ 5 个交易日；
- 同一交易日所有股票只能进入同一折；
- 训练中完成标准化、分桶、特征筛选和模型拟合，禁止使用全样本预处理结果；
- 测试折只允许加载训练折 artifact。

### 5.3 主指标和诊断指标

主指标：

1. 冻结 Top5 日级组合平均净收益；
2. 相对当前生产 baseline 的日级净收益增量。

同时报告：

- 累计净收益、Sortino、最大回撤和最差日；
- 胜率、盈亏比、下行分位数；
- 相对沪深300的超额收益；
- 换手、成交率、费用、滑点和市场冲击；
- K=3/5/10 敏感性；
- 因子覆盖率和缺失分层表现；
- 不同市场状态、板块和市值分层表现；
- 概率模型的 Brier score、log loss、ECE 和可靠性曲线。

### 5.4 多重检验和不确定性

1. 对日级组合增量使用移动块或 stationary bootstrap，避免把相关日收益当独立样本。
2. BH-FDR 必须覆盖本轮全部候选因子、退出参数、模型和组合，不得只对“胜出的一个”计算。
3. 正面折数只作为稳定性诊断，不单独代表显著性。
4. 记录 Deflated Sharpe Ratio；试验数量足够时评估 PBO/CSCV。
5. 四折正态近似置信区间只能作诊断，不作为唯一晋级证据。

参考：

- [The Probability of Backtest Overfitting](https://papers.ssrn.com/sol3/papers.cfm?abstract_id=2326253)
- [The Deflated Sharpe Ratio](https://papers.ssrn.com/sol3/papers.cfm?abstract_id=2460551)
- [scikit-learn TimeSeriesSplit](https://scikit-learn.org/stable/modules/generated/sklearn.model_selection.TimeSeriesSplit.html)

### 5.5 统一晋级门槛

候选变更必须同时满足：

| 门控 | 标准 |
|---|---|
| 数据 | PIT、baseline、样本类型和字段覆盖达标 |
| OOS 经济增量 | Top5 日级平均净收益增量 > 0 |
| 不确定性 | 日级块 bootstrap 的 95% 增量置信下界 ≥ 0 |
| 多重检验 | 纳入全部试验后的 FDR/DSR 门控通过 |
| 稳定性 | 主要 OOS 折、K=3/5/10 和核心市场分层无集中失效 |
| 回撤 | 最大回撤不得比 baseline 恶化超过 1 个百分点，除非收益回撤比明确改善且预注册允许 |
| 成本压力 | base 成本后为正，高成本情景下不发生结论反转或灾难性恶化 |
| 运行灰度 | 先 5 个交易日确认运行无异常，再至少 20 日观察；最终仍以统计门控为准 |

任何单一指标改善都不能绕过其余门控。

## 六、P3：收益改进实验顺序（第 4–12 周及后续）

### 6.1 退出规则：第一项收益实验

当前已经支持 tomorrow/swing 两套配置，但默认均为 8/5/4，参数没有充分 OOS 证据。

#### Tomorrow 候选

首轮最多预注册 3–5 套：

1. 当前 8/5/4；
2. 仅保护性止损 + T+1 固定时点退出；
3. 波动率归一化止损 + T+1 固定时点退出；
4. 无固定止盈、保留尾盘/收盘退出；
5. 如有分钟数据，再测试盘中触发版本。

#### Swing 候选

1. 当前 8/5/4；
2. 波动率归一化止损 + 时间止损；
3. 固定止损 + 动态 trailing；
4. 有分钟数据后再测试分批止盈。

约束：

- 首轮不按 regime 拆分，避免参数数量膨胀；
- 每个 regime 至少有 60 个独立交易日后，才允许研究 regime 退出参数；
- 日线同时触及止盈和止损时，主结果采用保守的 stop-first，并同时报告上下界；
- 所有参数组合计入统一多重检验；
- 评价目标是组合净收益/回撤，不是选出历史收益最高的单组参数。

### 6.2 已计算形态因子：第二项收益实验

按单因子逐个消融：

1. `price_position_20d`；
2. `upper_wick_ratio` / `lower_wick_ratio`；
3. `consecutive_up_days` / `consecutive_down_days`；
4. `amplitude_5d_mean`；
5. 修正后的 `close_vs_typical_price` 或真实 VWAP。

建议用途：

- tomorrow：增强 `tail_setup_score` 和弱收盘风险；
- swing：用于过热、延续和波动风险；
- short_term：只评价其对 tomorrow 候选召回/精度的增量。

每次只接入一个因子，不允许一次打开 `ENABLE_ENHANCED_FACTORS` 后把所有字段同时加入评分。

### 6.3 资金流和基本面：数据满足后再研究

#### 资金流/委比

当前覆盖为 0，状态为 **blocked_by_data**。

解除阻塞条件：

- 明确数据源、更新时间、单位和异常处理；
- 14:30 前可用且有可信 `observed_at`；
- 连续 20 个交易日覆盖率 ≥ 95%；
- 缺失值不能被当作真实 0；
- 独立验证与成交额、涨幅、量比的相关性和增量 IC。

#### 基本面

优先用于 `swing_picks`：

- 质量、现金流、杠杆和盈利变化优先；
- 估值因子按行业/市场横截面标准化；
- 公告日只有在实际公开后才能使用；
- 基本面缺失股票单独分层，防止覆盖率本身成为隐含市值因子。

### 6.4 预期收益排序：首个模型化晋级候选

保留现有 nearest-feature 模型和 artifact 机制，先补充：

- 训练/预测特征可用性 schema；
- 每折独立构建 neighbor 集；
- 相似度距离和时间衰减敏感性；
- 对不确定性大的预测降权，而不是只看点预测；
- K=5 相对原始分数的日级组合增量。

门槛：

- 系统最低 60 个真实交易日，建议 120 日后再正式晋级；
- artifact baseline、版本、训练窗口和到期时间全部匹配；
- 缺少有效 feature peers 时必须回退原始排序，不能使用启发式预测补位。

### 6.5 概率校准与 Meta-Labeling

概率校准先保持诊断模式，完成以下修订后再参与仓位：

- 总样本建议 ≥ 150，且每个最终分桶建议 ≥ 30；
- 按时间折训练 isotonic/Platt/分桶候选并 OOS 比较；
- 以 Brier、ECE 和可靠性曲线为主，不用命中率替代校准质量；
- 明确 probability 是单票胜率还是组合条件概率。

Meta-Labeling 需要：

- 从“score + risk 的重复分桶”升级为正则化、可持久化模型；
- 只使用原始模型之外的正交信息，如可成交性、事件风险、预测不确定性和市场状态；
- 分别测试 `skip`、`reduced`、只做仓位缩放三种作用方式；
- 新增 artifact、过期、baseline 和 schema 门控；
- `ENABLE_META_LABELING` 与 `META_LABELING_ENFORCE_ACTION` 分离，影子计算不能自动变成执行过滤。

### 6.6 Factor IC、Regime 和交互项

#### Factor IC

修订为：

1. 每个交易日对候选横截面计算 Spearman Rank IC；
2. 再计算 IC 均值、ICIR、正 IC 比例、块 bootstrap CI；
3. 按 20/60/120 日窗口监控衰减；
4. 只允许在训练折生成权重 multiplier；
5. IC artifact 与 baseline/feature schema 绑定。

#### Regime

- 先校准 `market_regime` 本身的分类稳定性；
- 区分“现有静态 profile”与“数据拟合 profile”；
- 学习结果必须保存 artifact，生产评分只能加载通过门控的 artifact；
- 每个 regime 的独立交易日不足时回退统一权重；
- 先验证仓位/风险约束，再验证评分权重，避免双重使用同一 regime 信号。

#### 交互项

- 先从有经济含义的少量 pair 开始；
- 不自动从所有组合中选 8 个后只报告最佳结果；
- 交互模型必须有 artifact、正则化、delta cap 和生产接线；
- 与 LightGBM 等非线性模型视为同一假设族，统一计算试验次数。

### 6.7 DeepSeek、事件 Alpha 和 Ensemble

#### DeepSeek/其他 AI

当前优先级不是更换模型，而是恢复 evidence 和可验证输出：

1. 确认新闻、公告、基本面和行情证据能在 cutoff 前被加载；
2. 保持结构化 feature 输出、证据 ID、时间戳和 abstain；
3. 禁止 AI 直接覆盖本地排序或生成无法复算的主观分数；
4. 保存 local vs challenger 的同日反事实组合；
5. 以增量净收益、回撤、覆盖、abstain 和 token 成本评价；
6. 至少 60 个独立日且 OOS 增量通过统一门控后，才允许进入 ensemble。

#### Event Alpha

- 固定事件权重只能作为研究先验；
- 事件类型、首次公开时间、已计价程度必须可审计；
- 同一事件影响多只股票时按事件簇统计，避免把它们当独立样本；
- 无证据、过期事件和模型猜测必须返回中性/abstain。

#### Ensemble

- 先把 Event、Expected Return、Probability、Meta、DeepSeek 都变成真实 OOS 预测；
- 只用 cross-fitted OOS 预测训练 ensemble；
- 初期采用非负、和为 1 的简单权重或正则化 stacking；
- 不使用拍脑袋的固定权重直接上线；
- 任一子模型缺失时重新归一化可用模型，禁止用默认 50 大量稀释有效模型；
- 只有 ensemble 的增量优于最佳单模型，而不只是优于原始分数，才允许晋级。

### 6.8 组合层改进

当前已有分仓和多重风险约束，优先补充：

1. 使用真实历史收益协方差做相关性降权；
2. 对协方差使用 shrinkage，避免短样本逆矩阵不稳定；
3. 先比较风险平价、最小方差和当前启发式权重；
4. 校准 `expected_edge_multiplier`，而不是固定 `1 + expected/8`；
5. 把换手和交易成本直接放入权重调整目标。

在预期收益模型和协方差都稳定前，不实施无约束 Max-Sharpe。

## 七、P4：ML 排名（数据成熟后，可选）

### 7.1 启动条件

必须同时满足：

- 至少 250 个独立交易日；
- 至少 5,000 条有完整标签的候选行；
- 全市场 PIT 特征和退市证券覆盖；
- 每个 OOS 测试折至少 20 个交易日；
- P0–P3 的简单模型已有稳定 baseline；
- 试验注册、artifact、特征 schema 和模型解释工具可用。

### 7.2 LightGBM Ranker 设计

- query/group = `signal_date`；
- 标签不能直接使用连续收益，先在当日横截面按成本后收益转成整数 relevance；
- 训练目标可以是 LambdaRank，经济验收仍使用 Top5 真实净收益；
- `lambdarank_truncation_level` 围绕生产 K=5 预注册；
- 使用浅树、叶子最小样本、特征/行采样和 early stopping；
- 所有预处理只在训练折完成；
- 输出 SHAP/特征重要性仅作解释，不能替代 OOS 消融。

参考：[LightGBM Parameters](https://lightgbm.readthedocs.io/en/latest/Parameters.html)

### 7.3 Ridge 的正确定位

Ridge 不能无条件替代当前 3 档搜索：

- 当前目标是 Top-K 非线性组合指标，Ridge 默认优化均方误差；
- 评分权重需要非负、和为 1、上限约束；
- 收益标签重尾且时变，系数符号可能不稳定。

可比较的候选是：

- 非负 Ridge/Elastic Net；
- simplex 约束回归；
- 直接以 OOS 排名或组合收益评价的低维搜索。

只有连续模型在 OOS 上稳定优于当前低维搜索，才允许替换。

## 八、配置和发布协议

### 8.1 开关分层

每个新模块建议拆成：

```python
ENABLE_<MODULE>_SHADOW = True      # 只计算和记录
ENABLE_<MODULE>_ENFORCE = False    # 改变排序/过滤/仓位
<MODULE>_ARTIFACT_PATH = "..."     # 通过门控的不可变模型
```

不能用一个布尔开关同时表示“计算诊断”和“改变交易”。

### 8.2 每次发布必须同步

1. 更新实验登记和 OOS 报告；
2. 写入不可变 artifact；
3. 更新 production manifest；
4. 提升策略/执行/特征版本中受影响的部分；
5. 生成新的 baseline ID；
6. 保存灰度开始时间和回滚目标；
7. 验证旧 baseline 仍可重放。

### 8.3 自动回退

以下任一情况触发回退：

- artifact 缺失、过期、baseline/schema 不匹配；
- 实时因子覆盖跌破门槛；
- PIT 或行情新鲜度失败；
- unknown 标签或执行数据异常；
- 20 日滚动表现触发预注册风险阈值；
- 高成本情景恶化超过发布时假设；
- 子模型输出缺失且 ensemble 无安全降级路径。

回退目标必须是上一份冻结、可重放的 baseline，而不是“当前代码默认值”。

真正可持续的收益提升来自：

- 成本后可复现的日级组合收益；
- 较少且有经济含义的独立假设；
- 对全部尝试进行多重检验；

推荐池盘后补数与验证策略优化方案

  摘要

  - 15:00 后查询推荐池时，按“策略 + 交易日”逐类判断保存状态。
  - 已在 15:00 前冻结并成功保存的类别，继续显示原冻结结果，不重新选股、不覆盖。
  - 未保存成功的类别，首次在 15:00 后运行时按收盘行情重新评分，保存为盘后补充样本，并显示该结果。
  - 后续重复运行优先读取当天已保存的盘中或盘后结果，保证同日结果稳定。
  - 现有股票列和最新价列已调整为原宽度的约 2/3（132px → 88px、90px → 60px），保留该改动并补充前端回归测试。

  核心实现

  - 在推荐快照、验证批次、信号记录、候选快照和查询结果中增加 snapshot_phase：
      - preclose_tradeable：15:00 前冻结的可交易推荐；
      - close_fallback：15:00 后因缺失而按收盘行情补生成的推荐；
      - 旧数据迁移为 legacy_unknown，不改变历史收益。

  - 保存状态按 strategy_name + signal_date + snapshot_phase 判定，不能只查某策略的最新历史记录，避免把前一交易日结果误显示为今日结果。
  - 三个策略独立补缺：
      - “明日”已保存时不重算；
      - “2-5日”缺失时只补“2-5日”；
      - “今日”同理。

  - 收盘补生成必须验证行情时间戳不早于 15:00；行情源不可用、时间戳缺失或不是当日收盘数据时，不伪造收盘样本，并继续显示已有盘中结果或明确显示数据不可
    用。

  - 盘后推荐的评分价格使用 T 日收盘价；验证入场采用理论收盘价，明确标记为 close_fallback，不能与 15:00 前真实可交易样本混合统计。
  - “今日”盘后样本的验证周期定义为 T 日收盘至 T+1 收盘；“明日”和“2-5日”沿用各自原有持有周期。

  运行与展示流程

  - 15:00 前：
      - 正常生成、冻结并保存 preclose_tradeable；
      - 后续刷新继续返回当日已冻结结果。

  - 15:00 后：
      - 先查询当天每个策略是否存在成功保存的阶段；
      - 有 preclose_tradeable 或 close_fallback 时直接读取并展示；
      - 只有缺失策略才执行收盘行情评分、保存 close_fallback，然后返回该结果。

  - daily_job --after-close 调整为先执行“收盘推荐补缺”，再下载日线、回填收益、刷新组合基线和因子指标，确保当天收盘推荐进入验证数据库。
  - API 返回 snapshot_phase、as_of、price_basis、saved_at，前端明确显示“盘中冻结”或“收盘补充”，避免用户误以为两者都是同一入场口径。
  - 推荐池保留阶段切换能力；默认显示当天已保存的有效结果，若盘后刚补生成则显示收盘补充结果。

  数据库与统计

  - 编写幂等 SQLite migration，为相关表增加阶段字段、索引和阶段感知唯一约束。
  - 保存操作必须幂等：同一策略、日期、阶段重复执行只更新同一批次，不产生重复样本。
  - 验证统计按 strategy + snapshot_phase + baseline 分组；盘中冻结和收盘补充不混合计算收益、胜率和风险指标。
  - 建议至少积累 60 个真实交易日后再调整权重或策略规则，避免少量补充样本导致过拟合。

  测试与验收

  - 三策略部分成功时只补缺失策略。
  - 同日重复运行保持结果和批次 ID 稳定。
  - 收盘行情时间戳校验与行情失败降级。
  - 旧数据库迁移后历史查询和收益统计不变。
  - close_fallback 的 T+1 收盘验证口径正确。
  - API 阶段字段和前端阶段标识正确。
  - 股票列、最新价列宽度保持为原来的 2/3，并覆盖桌面和窄屏布局。
  - 完整运行现有推荐、快照、验证回填和前端契约测试。
