# 全工程架构与性能重构计划

状态：非权威执行计划，仅记录待实施批次与验收顺序。

产品、架构、时间线、API、运维和验收契约以
[软件业务设计文档](software-business-design.md) 为唯一权威；候选、过滤、公式、风险、
融合和排名策略以 [荐股策略文档](recommendation-strategy.md) 为唯一权威。本文不得覆盖
两份权威文档；实施引起契约变化时必须先更新对应权威文档与测试。

## 1. 目标与边界

- 对活动工程执行等价重构，保持候选、评分、融合、风险、动作、排名、冻结时点、188 次
  预算和冻结业务哈希不变；策略优化另立后续批次。
- 保留 Flask、只读 Web 和有界线程模型；性能工作聚焦数据合并、缓存、发布和内存热路径。
- 内部模块、类和接口不提供兼容别名；配置、CLI 和 Web payload 可在对应完整章节原子升级，
  但冻结历史必须通过只读源迁移保留。
- 活动源码单文件继续执行 800 行上限；同时用复杂度、参数、命名和依赖门禁判断职责是否需要
  拆分，不为满足任意更低行数机械拆文件。

当前固定负载测量继续作为性能章节的迁移前基线，不替代重复 fixture 门禁：5,500 行标准化
P95 为 217ms；两源合并为 1,020ms，对应 1,000ms 门限；三源统一快照为 1,616ms，对应
1,500ms 门限；三板三策略评分为 494ms，对应 1,000ms 门限。规范 JSON 序列化与哈希是合并
热路径的主要成本；评分线程版仅为串行约 0.78 倍，因此线程只承担失败域隔离，不能宣称
未经重新测量的 CPU 加速。

状态：completed

- 明确 raw/qfq、来源、观察日与公司行为语义；需要复权的数据拒绝消费未复权价格。

完成证据：行情 provider、标准化、合并、历史、研究和特征边界继续独立，并新增
`MarketSourceCoordinator`；DeepSeek 按 HTTP、schema、预算批次/汇总、缓存、请求执行、
状态和复核编排采用显式组合，快照观测持久化也移除 mixin。统一结构化失败分类覆盖超时、
截止、熔断、负缓存、取消、latest-wins 替代、无数据、限流、schema 和来源失败；日线携带
`PriceAdjustment`、来源和交易观察日，历史缓存、摘要与特征入口拒绝 raw。严格
`PLR0913` 债务由 58 降至 55；完整质量、打包、仓库外 wheel、性能和三档桌面验收记录见
本批 `CHANGELOG.md`。

## 4. 工程重构后的独立策略路线

以下项目保留原计划结论，但不属于本轮工程等价重构，也不得与上述章节混入同一提交：

- 以覆盖全部支持 A 股的紧凑点时历史概要扩大候选总体，完整历史只保留候选热集；先记录
  新旧总体的候选召回、行业覆盖、换手、分数和未来结果 shadow，不直接替换生产推荐。
- 复用同一特征、风险事实和 DeepSeek 结果生成活动与旧策略同期影子，不增加物理 HTTP；
  以独立表保存影子身份和结果，不修改冻结表、冻结 JSON 或普通 Web。
- 结果结算保留 20bp 主口径及 50/100bp 压力口径、MAE/ATR 和不可执行状态；按交易日做
  固定种子的配对 block bootstrap。至少 60 个交易日、300 条有效配对、净超额收益差 95%
  置信下界为正且严重回撤率下降后，才允许声称收益改善或另开策略版本晋级。
