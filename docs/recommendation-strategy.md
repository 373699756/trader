# 荐股策略文档

版本：v1（活动策略 `strategy_v16_board_scoring_ttd25_2026_07`）

引擎：`engine_v16_board_scoring_ttd25_2026_07`

融合：`fusion_v2_local68_deepseek32`

状态：today、tomorrow、d25 活动生产契约；long 当前观察契约

本文是候选、过滤、因子、评分、风险、DeepSeek、融合、动作、排名和策略验收的唯一
权威。产品、架构、时间线、发布、API、运维和工程路线以
[软件业务设计文档](software-business-design.md) 为准。策略用于研究，不构成投资建议，
不保证收益，不触发真实交易。

## 1. 策略目标与共同纪律

- `today`：盘中短线，面向 T+1；11:20 冻结。
- `tomorrow`：尾盘形成，面向 T+1；14:50 冻结。
- `d25`：尾盘形成，面向 T+2 至 T+5；14:50 冻结。
- `long`：固定观察池，只展示当前快照，不冻结、不产生动作、不写推荐历史。

允许 0 只推荐。所有计算只使用观察时点已经生效的数据；实时价格、涨跌停和冻结锚点
使用未复权价格，历史收益、均线、波动和回撤使用点时可得的前复权序列。行业、证券
状态、财务和规则按实际生效时间进入特征。每次输入绑定交易日、阶段、数据、规则、
行业、配置、因子 schema、策略和引擎版本。

缺失原始值保持 `null` 并记录 `missing_fields` 与 `missing_reasons`。允许用于确定性评分
的中性占位为 50，但不得把占位序列化成真实观测，也不得通过缺失字段获得额外优势。
非有限数值、未来数据、无时区数据、截止后结果和无法验证的证券状态不得用于可执行
推荐。

## 2. 板块身份与交易规则

板块固定为 `main`、`chinext`、`star` 和 `unsupported`。识别顺序为带生效时间的证券
主数据、交易所清单、行情源市场字段、六位代码前缀降级。前缀降级记录
`board_source=code_prefix_fallback` 和 `board_reliability=degraded`；来源冲突记录
`board_classification_conflict`，只能观察。

每个观察点保存：板块、来源、可靠度、交易所、上市日期、上市交易日龄、是否有价格
限制、交易所限制比例、策略热度上限、规则版本和生效日期。

首次上市第 0-5 个交易日、重新上市首日、退市整理首日直接排除，第 6 个交易日起才用
普通规则。上市日期或交易日龄不可验证时只能观察。普通热度边界：主板 8.00% 通过、
8.01% 剔除；创业板和科创板 16.00% 通过、16.01% 剔除。无价格限制状态不计算普通
`limit_proximity`。

## 3. 统一行情与点时合并

策略只消费不可变统一行情，不直接读取供应商对象。相同有效输入无论来源完成顺序都
产生相同字段归属、来源版本、冲突集合、缺失集合、合并 epoch 和规范 JSON 哈希。

同来源先按来源时间、接收时间选择较新字段；跨来源同时点按既定优先级。新鲜腾讯
候选价只能在与东方财富或新浪至少一个全市场观测偏差不超过 0.50% 时完成冲突复核，
不得自证。慢数据不得覆盖更新的实时价格。全源失败时沿用最近有效统一快照并降级，
不能生成空快照清除推荐。

## 4. 硬过滤

候选必须依次通过确定性硬过滤。活动阻断条件至少包括：

- 非 A 股、`unsupported` 板块、退市整理、暂停上市、停牌、不可交易或证券状态未知；
- ST、`*ST`、风险警示、黑名单和不能验证上市日龄；
- 上市第 0-5 个交易日、重新上市首日、退市整理首日；
- 价格、成交量、成交额、关键行情时间或数据版本缺失/非法；
- 20 日成交额中位数低于 5000 万元；
- 超过板块热度上限、接近一字涨跌停、封板或无法合理成交；
- 经点时 schema 验证的 `negative_announcement_level > 0`、
  `shareholder_reduction_level > 0`、`unlock_risk > 0`、`pledge_risk > 0` 或
  `financial_deterioration > 0.5`；
- 实时性超过动作边界、跨源价格冲突未复核、板块身份冲突；
- 核心字段缺失比例超过 30% 或板块数据可靠度不足以执行。

硬过滤是结构化规则，不能由 DeepSeek 放宽、删除或新增池外股票。可选告警进入
`optional_flags`，不冒充阻断原因；必需阻断保存在 `filter_reasons`。原因码必须细分到
具体板块和事实，例如 `chinext_board_too_hot`、`star_board_too_hot`，不能用模糊成长
板标签替代。

结构化风险来源失败、字段缺失或非有限时不能猜测为命中或无风险，候选保留本地降级
能力并记录 `structured_risk_unavailable`。减持事实覆盖控股股东、实际控制人、持股 5%
及以上股东和董监高的计划、实施及进展公告；已明确终止且未实施的计划不得误判。
减持、解禁、质押必须保持独立字段，不能用旧合并字段把失败来源补成 0。

## 5. 板内总体、同行与成交因子

横截面总体固定为：

```text
trade_date + phase + board + data_version
```

流动性、换手、波动、回撤、趋势、相对强度、同行差、领先滞后和行业强度均在板内按
2.5%/97.5% 截尾、归一化和稳定排序。有效板内横截面至少 100 只；不足时最多回退最近
5 个有效交易日的同板同阶段总体，不能借用其他板块。

同行收益差：

```text
peer_return = median(return_j for valid same-board same-industry peers, j != stock)
peer_gap = stock_return - peer_return
```

行业至少 10 只有效股票并排除目标自身。today 使用 1/3/5 日，tomorrow 使用 5/20 日，
d25 使用 20/60 日。活动 v16 固定为趋势延续：正 `peer_gap` 得分更高，不按结果切换到
补涨或反转方向。

领先组为同板同行业按过去 20 日成交额中位数的前 20%，至少 3 只，计算收益时排除目标
自身：

```text
leader_gap = leader_group_return - stock_return
```

`leader_gap` 越小得分越高；若与行业趋势没有独立信息则去重或保持缺失，禁止重复加分。

换手和成交冲击：

```text
turnover_shock_20 = current_turnover / median(turnover, 20)
amount_shock_20 = current_amount / median(amount, 20)
volume_price_confirmation = sign(return) * normalized_amount_shock
```

分母缺失、非正或非有限时保持缺失。换手使用带通函数
`B(x;0.8,1.1,2.0,4.0)`，成交额使用 `B(x;0.8,1.2,3.0,6.0)`。所有成交量、成交额和
换手指标统称 `flow_proxy`，不得解释为主力、机构或散户资金。

| --- | --- | --- | --- |
| 主板 | 流动30、日内25、换手20、同行15、完整10 | 流动35、同行15、趋势25、稳定15、完整10 | 流动30、残差20、趋势20、稳定15、执行10、完整5 |
| 创业板 | 流动30、日内25、换手20、同行15、完整10 | 流动20、同行30、趋势25、稳定15、完整10 | 流动20、残差30、趋势20、稳定10、执行15、完整5 |
| 科创板 | 流动30、日内25、换手20、同行15、完整10 | 流动25、同行15、趋势30、稳定20、完整10 | 流动25、残差15、趋势30、稳定15、执行10、完整5 |

权重以小数写入唯一策略配置，每组精确合计 1.0。today 虽同权重，三板仍分别计算总体、
可靠度和排名。候选要求 `candidate_score >= 50`、核心缺失不超过 30%，随后按候选分
降序、代码升序稳定排列，每板最多 120 只且不补满。

`board_data_reliability >= 0.85` 才能进入可执行链和 DeepSeek 队列。低可靠候选只能排在
可靠候选之后作为观察，不消耗 DeepSeek 预算。DeepSeek 不得新增池外股票。

冷启动历史预热只改变数据就绪过程，不改变候选公式或阈值。实时全市场行情先形成当前
epoch；预热代码按三板均衡稳定顺序在独立历史池连续分批取得历史日线。120 积分
Tushare 档优先使用 SDK `daily` 批量未复权数据，成交量从手换算为股、成交额从千元换算
为元；只有积分档达到 2000 时才允许启用 Tushare 前复权能力。供应商明确返回 `daily`
权限拒绝时不按限流重试，改以腾讯完整日 K 前复权响应作为历史主回退（成交量手转股、
成交额万元转元），东方财富作为第二回退。历史不足 20 个有效交易日的
代码记录为 `history_warming`，不得作为有效候选，也不得清除上一份有效推荐；已成功的
单只历史立即进入下一次筛选，同批其他代码超时不能回滚。达到策略数据条件后必须继续
完成硬过滤、九组预选、三板评分、全局 TopK 和 P6 发布，不能停在行情或历史内存池。
当 Tushare 或远端历史暂不可用时，可只读使用本机最近有效前复权日线种子；不足 20 日
仍按 `history_warming` 处理。全市场发现以本轮接收时间判断链路新鲜度，候选复核和评分
继续保留并检查原始来源时间，旧行情只能观察，不能因此获得可执行资格。
若一轮全市场硬过滤因 `history_warming`、缺失/非法流动性历史或行情接收时间陈旧而全部
为空，P3 必须保留最近有效候选身份并记录降级，等待候选定向报价与历史预热恢复；只有
存在完成数据条件的股票且仍无候选时，才允许用真实空结果替换候选池。保留身份不得绕过
后续原始 `source_time` 新鲜度、硬过滤、评分或动作门。

## 7. 本地评分

定义带通函数 `B(x;l,o1,o2,u)`，其中 `l < o1 <= o2 < u`：`x <= l` 或 `x >= u`
得 0，`o1 <= x <= o2` 得 100，两侧区间线性变化。参数非法时配置拒绝启动，缺失或
非有限输入按缺失规则处理。`P(x)` 是 0-100 横截面平均秩，反向因子必须登记并计算
`100-P(x)`。

统一计算边界：

```text
base_score = sum(component_score * component_weight)
local_score = clamp(base_score - local_risk_penalty, 0, 100)
```

组件和子组件权重分别精确合计 1.0，三板单独计算，输出统一为 0-100。活动权重：

| 板块 | today | tomorrow | d25 |
| --- | --- | --- | --- |
| 主板 | 日内30、换手20、同行20、流动执行20、稳定10 | 尾盘15、同行领先15、换手量价10、趋势25、稳定25、市场10 | 残差15、趋势25、质量价值25、稳定15、量价流动10、不过热10 |
| 创业板 | 日内30、换手20、同行20、流动执行20、稳定10 | 尾盘20、同行领先25、换手量价20、趋势20、稳定10、市场5 | 残差30、趋势20、质量价值10、稳定10、量价流动20、不过热10 |
| 科创板 | 日内30、换手20、同行20、流动执行20、稳定10 | 尾盘15、同行领先15、换手量价10、趋势30、稳定25、市场5 | 残差15、趋势30、质量价值25、稳定15、量价流动10、不过热5 |

today 子组件：日内结构使用 5 分钟涨幅、涨速、当日涨幅、量比；换手状态使用换手冲击、
成交额冲击和量价确认；同行使用同板同行业 1/3/5 日趋势延续；流动执行使用 20 日
成交额分位、适中换手和距价格限制安全度；稳定为低波动 50%、低最大回撤 50%。

tomorrow 子组件：尾盘使用 30 分钟收益、尾盘量比和收盘位置；同行领先为同行 5 日 40%、
20 日 40%、领先差反向 20%；换手量价为换手 35%、成交额 35%、量价确认 30%；稳定为
低波动/低回撤各 50%；市场状态固定映射 risk-on 60、neutral 50、risk-off 40。

d25 子组件：残差动量为同行 20 日 60%、60 日 40%；质量价值为质量 50%、价值 30%、
成长 20%；稳定为低波动/低回撤各 50%；量价流动为 20 日成交额分位 50%、换手 25%、
成交额 25%；不过热是显式组件。

d25 禁止再用 `base_score * overheat_factor * market_regime_factor` 双乘。市场状态只进入
登记组件、结构化风险上下文和审计，不直接缩放总分。

### 7.1 long 固定观察评分

long 使用版本控制的固定名单，按价值 30%、成长 25%、质量 20%、行业/政策 15%、风险
保护 10% 排序。财务只接受公告时间不晚于观察点、报告日不超过 550 天的最新记录；
仅有日期的记录按当日 23:59:59 生效。累计 EPS 和 ROE 按报告月 3/6/9/12 分别乘
4/2/4÷3/1 年化。

- 价值：`PE=未复权实时价/年化基本EPS`，不高于 10 得 100、达到 50 得 0；
  `PB=未复权实时价/每股净资产`，不高于 1 得 100、达到 8 得 0；中间线性，可用子项
  等权，非正分母保持缺失。
- 成长：营业收入、归母净利润和扣非归母净利润同比的可用值等权，
  `clamp(50 + 2 * mean_growth, 0, 100)`。
- 质量：`clamp(50 + (年化ROE-10)*2.5, 0, 100)` 与
  `clamp(扣非归母净利润/归母净利润*100, 0, 100)` 的可用值等权。
- 行业/政策：行业强度 60% + 点时政策证据 40%；政策分为
  `clamp(50 + 10*(正向去重命中-负向去重命中), 0, 100)`。
- 风险保护：低波动 50% + 低最大回撤 50%，缺失保持 `null`。

营业收入同比不高于 -10%、归母或扣非净利润同比不高于 -20% 时，
`financial_deterioration=1`。未来 90 日解禁占总股本低于 1%/1%-5%/5%-10%/不低于
10% 对应 0/1/2/3；累计质押占总股本低于 10%/10%-20%/20%-35%/不低于 35% 对应
0/1/2/3。公告最多读取最近 180 日、100 条，只有来源成功且 schema 合格才可确认 0。
目标价只能人工配置，未配置显示 `--`，模型不得虚构。

## 8. 执行质量、可靠度与容量

板内容量层级固定为：低于 P50 为 `liquidity_observe`，P50 至 P80 为
`normal_capacity`，P80 及以上为 `high_capacity`。板内样本不足时保持保守或缺失，
不能跨板借用阈值。

可靠度由来源新鲜度、字段完整度、板块身份、横截面规模、同行规模和冲突状态构成，
范围 0-1。可靠度低于 0.85、行情超动作年龄或价格冲突未解决时，不产生新的可执行动作，
但可展示最近有效推荐并标记观察/降级。

执行质量只描述在公开行情下的相对容量和价格限制风险，不预测真实成交，不生成仓位、
资金规模或下单建议。

## 9. 本地风险与结构化事实

本地风险先从 `base_score` 扣除，结果已经包含在 `local_score` 中。风险事实按规范键去重，
同一事实不因多来源或模型重复描述而重复扣分；本地风险总扣分有界，活动测试要求覆盖
七类事实、去重和 25 分截断。

活动短线风险及固定扣分为：

| 风险代码 | 含义 | 扣分 |
| --- | --- | ---: |
| `near_limit_crowding` | 接近板块价格限制的追高拥挤 | 5 |
| `price_volume_divergence` | 价格与成交量/成交额确认方向背离 | 4 |
| `high_volatility` | 20 日波动率进入登记的极端区间 | 3 |
| `short_term_overheat` | 5/10 日收益或 MA20 偏离极端 | 3 |
| `intraday_reversal` | 冲高回落且收盘位置低；today 至少 30 个有效分钟 | 4 |
| `liquidity_contraction` | 通过绝对成交额门后相对流动性骤降 | 3 |
| `trend_breakdown` | 价格、MA5/MA20 和短期收益共同破位 | 3 |

七项理论合计和截断上限均为 25；无事实时扣 0。普通连续弱势只由基础组件反映，不能
重复触发离散扣分。缺失、非有限或样本不足不触发扣分，只记录缺失。long 不使用这组
短线结构扣分。风险事实保存实际值、阈值、方向、严重度、证据、生效时间、互斥/叠加
组和本地规则版本。

模型自由文本、评级、置信度、情绪形容词或未经证据验证的结论不能直接决定扣分、veto、
动作或排名。

## 10. DeepSeek 结构化复核

DeepSeek 是可选复核器，不是候选生成器。默认使用受控双模型：V4-Flash 非思考主审，
V4-Pro 思考挑战者仅处理预注册的不确定、高风险或边界案例。供应商不可用、密钥缺失、
超时、预算耗尽或 schema 失败时回退本地推荐，并明确 `local_degraded` 或相应终态。

每只候选最多使用 16 条点时证据，首轮槽位为结构化行情 1、尾盘分钟 1、财务与研究
3、监管/质押/解禁等风险 4、新闻与公告 7。空槽按监管风险、财务研究、分钟量价、
新闻公告的顺序补给；补位后监管最多 6、财务最多 5、新闻公告最多 8，行情和分钟仍
各最多 1，总数不超过 16。跨来源同主体、事实类型、发布时间和摘要哈希的记录只保留
质量更高且更早接收的一条；存在有效相反材料时至少保留一条反证。

五维固定为价值质量、财务健康、资金量价、行业政策和风险质量。每维输出 0-100 的
`score`、0-1 的 `raw_confidence`、flags、evidence_ids 和 assessment。有效分与总分为：

```text
effective_i = 50 + (score_i - 50) * raw_confidence_i
deepseek_score = sum(effective_i * w_i)
confidence_coverage = sum(raw_confidence_i * w_i)
```

未知维度固定为 `score=50, raw_confidence=0, effective=50`，其余权重不重归一化：

| 策略 | 价值质量 | 财务健康 | 资金量价 | 行业政策 | 风险质量 |
| --- | ---: | ---: | ---: | ---: | ---: |
| today | 10% | 10% | 40% | 15% | 25% |
| tomorrow | 15% | 20% | 25% | 20% | 20% |
| d25 | 20% | 25% | 20% | 20% | 15% |
| long | 30% | 30% | 10% | 20% | 10% |

仅当 schema 和证据有效、结果未迟到、非 abstain、至少两个维度已知且
`confidence_coverage >= 0.50` 时应用模型分。高风险证据有效且置信度不低于 0.7，且
本地风险表登记了证据类型、有效期和 veto 资格时，才允许本地映射 veto。

输出必须通过严格 schema，包含代码、维度分、结构化风险事实、证据引用、主审/挑战者
状态、请求/实际模型、thinking 模式、原始与校准置信度、证据 manifest 哈希、校准版本
和受控评级。缺失、越界、代码错配、证据错配或未知字段按规则拒绝或修复；修复重试也
计入物理请求。

审计字段 `review_stage`、`challenger_status`、`rating` 和置信度只用于展示、持久化和
离线观察，不参与生产分数、动作或排序。挑战者不能放宽硬过滤；冲突时只可维持或更
保守。模型提出的风险必须经 schema、证据引用和本地规则映射后才形成
`deepseek_risk_penalty`。

| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: |

long。emergency 仅在正常桶耗尽且出现新高风险或冻结边界变化时使用，并同时计入对应

缓存键至少包含股票、策略、交易日、阶段、价格桶、量比桶、证据 manifest、模型、

解释必须按策略隔离。缓存淘汰不回退已发生的预算计数。

## 12. 融合、动作与稳定排名

固定公式：

```text
final_score = clamp(
    local_score * 0.68
    + deepseek_score * 0.32
    - deepseek_risk_penalty,
    0,
    100,
)
```

`local_score` 已扣本地风险，融合时禁止再次扣 `local_risk_penalty`。所有输入使用十进制
确定性计算，最终以 `ROUND_HALF_UP` 保留两位小数。固定验收向量结果必须为 `83.40`。

生产排序固定为：`final_score` 降序、`local_score` 降序、股票代码升序。模型评级、
置信度、挑战状态和自由文本不得改变同分顺序。

每个 `strategy + data_version` 只能发布一种模式：`hybrid` 要求排名保护集合中的候选均
已进入 applied 或 abstain 终态，分别使用融合或本地回退；若保护集合仍有 failed、late、
rejected 或未完成候选，整版必须为 `local_degraded` 并统一按本地分排名。排名保护集合
为本地分前 18 名与距动作门槛 5 分以内候选的并集；模式变化生成新发布版本并随冻结
固化。

活动动作阈值：

- today 09:36-10:30：`final_score >= 70` 可执行；
- today 10:30-11:20：`final_score >= 76` 可执行；
- tomorrow：`final_score >= 78` 可执行；
- d25：`final_score >= 76` 可执行；
- 距对应门槛 5 分以内为观察，其余不可执行。

可执行项还必须通过硬过滤、时间窗口、新鲜度、周期匹配且无 veto。冻结后动作不可改变。
旧动作和旧阈值只供带旧策略/引擎/schema 身份的冻结回放，不得成为活动默认值。

## 13. TopK 与集中度

today、tomorrow、d25 分别在九个板块草稿合并后全局稳定选择，不能按板先填满再拼接。
活动默认展示 Top10，接口允许 0-18，`top_n=0` 合法；允许不足或为 0，不为满足数量
放宽过滤和可靠度。

集中度约束：单一板块最多 `ceil(top_n * 60%)`；主板同一竞争组最多 3 只，创业板和
科创板同一竞争组最多 2 只。行业/同行过度集中时按稳定次序保留更高分候选，并保存被
跳过候选的板内原排名、竞争组、限制值和原因。约束只在已通过硬过滤和动作质量门的
候选间选择，不能引入池外票；不足时保持较少结果。

long 使用固定观察清单和当前报价，不参与三策略 TopK、集中度、冻结和历史日期列表。

## 16. 明确不采用的策略

- 不让 DeepSeek 自由文本、评级或置信度直接改变分数、风险、动作或排名。
- 不由 DeepSeek 新增候选、绕过硬过滤或选择方向。
- 不进行运行时自动调权、自动晋级、在线学习或看完结果后切换动量/反转。
- 不引入多智能体生产编排、第三方项目的交易结论、券商连接或真实下单。
- 不把 `flow_proxy` 解释为不可验证的主力或机构资金。
- 不把历史回测、shadow 审计或社区 Star 当作生产收益证据。
- 不为补满 TopK 放宽质量门，不把缺失值伪装成 0 或真实中性观测。
