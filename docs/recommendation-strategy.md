# 荐股策略文档

版本：v2（活动策略 `strategy_v20_review28_2026_07`）

引擎：`engine_v20_review28_2026_07`

融合：`fusion_v2_local68_deepseek32`

状态：today、tomorrow、d25 活动生产契约；long 当前观察契约

本文是候选、过滤、因子、评分、风险、DeepSeek、融合、动作、排名和策略验收的唯一
权威。产品、架构、时间线、发布、API、运维和工程路线以
[软件业务设计文档](software-business-design.md) 为准。策略用于研究，不构成投资建议，
不保证收益，不触发真实交易。

## 1. 策略目标与共同纪律

- `today`：盘中短线，面向 T+1；11:20 冻结。
- `tomorrow`：尾盘形成，面向 T+1；14:50 冻结。
- `d25`：尾盘形成，面向 T+2 至 T+5；14:50 冻结。
- `long`：固定长期潜力龙头研究池，只展示当前行情快照；warmup、上午和下午都可随统一
  行情生成当前观察快照，不运行荐股策略、不冻结、不产生交易动作、不写推荐历史。

允许 0 只推荐。所有计算只使用观察时点已经生效的数据；实时价格、涨跌停和冻结锚点
使用未复权价格，历史收益、均线、波动和回撤使用点时可得的前复权序列。行业、证券
状态、财务和规则按实际生效时间进入特征。每次输入绑定交易日、阶段、数据、规则、
行业、配置、因子 schema、策略和引擎版本。

缺失原始值保持 `null` 并记录 `missing_fields` 与 `missing_reasons`。允许用于确定性评分
的中性占位为 50，但不得把占位序列化成真实观测，也不得通过缺失字段获得额外优势。
非有限数值、未来数据、无时区数据、截止后结果和无法验证的证券状态不得用于可执行
推荐。

## 2. 板块身份与交易规则

板块固定为 `main`、`chinext`、`star` 和 `unsupported`。识别顺序为带生效时间的证券
主数据、交易所清单、行情源市场字段、六位代码前缀降级。前缀降级记录
`board_source=code_prefix_fallback` 和 `board_reliability=degraded`；来源冲突记录
`board_classification_conflict`，只能观察。

每个观察点保存：板块、来源、可靠度、交易所、上市日期、上市交易日龄、是否有价格
限制、交易所限制比例、策略热度上限、规则版本和生效日期。

首次上市第 0-5 个交易日、重新上市首日、退市整理首日直接排除，第 6 个交易日起才用
普通规则。上市日期或交易日龄不可验证时只能观察。普通热度边界：主板 8.00% 通过、
8.01% 剔除；创业板和科创板 16.00% 通过、16.01% 剔除。无价格限制状态不计算普通
`limit_proximity`。

## 3. 统一行情与点时合并

策略只消费不可变统一行情，不直接读取供应商对象。相同有效输入无论来源完成顺序都
产生相同字段归属、来源版本、冲突集合、缺失集合、合并 epoch 和规范 JSON 哈希。

同来源先按来源时间、接收时间选择较新字段；跨来源同时点按既定优先级。新鲜腾讯
候选价只能在与东方财富或新浪至少一个全市场观测偏差不超过 0.50% 时完成冲突复核，
不得自证。慢数据不得覆盖更新的实时价格。全源失败时沿用最近有效统一快照并降级，
不能生成空快照清除推荐。

## 4. 硬过滤

候选必须依次通过确定性硬过滤。活动阻断条件至少包括：

- 非 A 股、`unsupported` 板块、退市整理、暂停上市、停牌、不可交易或证券状态未知；
- ST、`*ST`、风险警示、黑名单和不能验证上市日龄；
- 上市第 0-5 个交易日、重新上市首日、退市整理首日；
- 价格、成交量、成交额、关键行情时间或数据版本缺失/非法；
- 20 日成交额中位数低于 5000 万元；
- 超过板块热度上限、接近一字涨跌停、封板或无法合理成交；
- 经点时 schema 和官方证据验证的 `major_shareholder_reduction`、
  `financial_fraud_history`、`official_investigation_history`、
  `major_illegal_history`、`fund_occupation_history`、`illegal_guarantee_history` 或
  `forced_delisting_risk` 命中；
- `unlock_risk > 0`、`pledge_risk > 0` 或 `financial_deterioration > 0.5`；
- 实时性超过动作边界、跨源价格冲突未复核、板块身份冲突；
- 核心字段缺失比例超过 30% 或板块数据可靠度不足以执行。

硬过滤是结构化规则，不能由 DeepSeek 放宽、删除或新增池外股票。可选告警进入
`optional_flags`，不冒充阻断原因；必需阻断保存在 `filter_reasons`。原因码必须细分到
具体板块和事实，例如 `chinext_board_too_hot`、`star_board_too_hot`，不能用模糊成长
板标签替代。

严重公司风险只接受发行人、交易所、监管或司法结构化披露及其稳定证据 ID；新闻标题、
DeepSeek 自由文本和普通风险提示、问询函、一般诉讼、亏损预告不能创建或解除硬事实。
规则固定为：

- 大股东指控股股东、实际控制人和持股不低于 5% 的股东；减持计划从公告起阻断，至官方
  完成或终止后 90 天解除，无终态则持续阻断；
- 官方决定确认的财务造假、虚假记载、重大遗漏、重大违法、资金占用和违规担保永久阻断；
- 正式立案调查和强制退市程序在存续期及官方结案、撤回或终止后三个自然年阻断，满三个
  自然年当日解除。

风险事实随研究缓存持久化合并，旧命中不能因本轮来源失败消失。来源失败或覆盖状态不可
确认时不能猜成 0：已确认事实继续阻断，其余候选保留本地快照但动作降为观察，并记录
`corporate_risk_history_unavailable`。减持、解禁、质押必须保持独立字段。

## 5. 板内总体与成交因子

横截面总体固定为：

```text
trade_date + phase + board + data_version
```

流动性、换手、波动、回撤、趋势和相对强度均在板内按
2.5%/97.5% 截尾、归一化和稳定排序。有效板内横截面至少 100 只；不足时最多回退最近
5 个有效交易日的同板同阶段总体，不能借用其他板块。

行业只保留为展示、审计和最终集中度标识，不进入候选分、本地分、DeepSeek 分、动作门或
下行保护。活动评分不得计算或读取 `peer_gap`、`leader_gap`、`industry_strength`、
`industry_breadth`、`industry_trend` 或 `industry_policy_score`；修改同一股票的行业字段
不能改变其分数。该约束也消除了行业缺失时同行两两计算造成的二次复杂度。

换手和成交冲击：

```text
turnover_shock_20 = current_turnover / median(turnover, 20)
amount_shock_20 = current_amount / median(amount, 20)
volume_price_confirmation = sign(return) * normalized_amount_shock
```

分母缺失、非正或非有限时保持缺失。换手使用带通函数
`B(x;0.8,1.1,2.0,4.0)`，成交额使用 `B(x;0.8,1.2,3.0,6.0)`。所有成交量、成交额和
换手指标统称 `flow_proxy`，不得解释为主力、机构或散户资金。

## 7. 本地评分

定义带通函数 `B(x;l,o1,o2,u)`，其中 `l < o1 <= o2 < u`：`x <= l` 或 `x >= u`
得 0，`o1 <= x <= o2` 得 100，两侧区间线性变化。参数非法时配置拒绝启动，缺失或
非有限输入按缺失规则处理。`P(x)` 是 0-100 横截面平均秩，反向因子必须登记并计算
`100-P(x)`。

统一计算边界：

```text
base_score = sum(component_score * component_weight)
local_score = clamp(base_score - local_risk_penalty, 0, 100)
```

组件和子组件权重分别精确合计 1.0，三板单独计算，输出统一为 0-100。活动权重：

### 7.1 确定性入场质量

v17 为 tomorrow 和 d25 增加 `entry_quality`，取缩量回踩与放量突破两类确定性形态的
较高值。热度本身不阻止入选；只有形态、下行保护和既有硬过滤共同决定是否可执行。

- 缩量回踩：`MA5 >= MA10 >= MA20`、MA20 斜率向上，价格距离 MA5 不超过 1% 或距离
  MA10 不超过 2%，点时 `volume_to_5d_average` 不高于 0.70，且价格不低于 MA20。
- 放量突破：价格不低于前 20 个完整交易日最高价，点时 `volume_to_5d_average` 不低于 2.0，
  收盘位置不低于 70，突破偏离不超过 5%。
- `volume_to_5d_average` 优先使用供应商点时 `volume_ratio`；供应商未提供时，使用同日
  点时成交额除以最近 5 个已完成交易日平均成交额派生，不使用同日历史 bar。
- 完整形态为 100，趋势前提成立但入场确认不完整为 50，趋势破坏为 0；必要输入缺失
  保持 `null`，不能以中性值冒充确认。

行业组件移除后，其权重按各策略剩余组件原比例归一化；`entry_quality` 保持上述确定性
形态定义。过热事实继续审计和软扣分，不再作为 d25 独立正向组件。

### 7.2 long 固定研究池

long 不是荐股策略，不计算价值、成长、质量、风险保护或任何旧长期评分，不调用
DeepSeek，不执行 TopK/行业集中度选择，也不把研究字段缺失标记为策略降级。long 只从
版本控制的 `long_watchlist.json` 读取固定股票池和分组，按配置顺序生成 `observe` 观察项，
评分字段固定为 0，仅用于保持 Web envelope 兼容，不代表投资判断。

long 固定池的准入语义是“长期潜力赛道中的头部或弹性龙头观察标的”：卡脖子行业优先
关键环节国产替代龙头，高成长赛道优先产业趋势清晰且全市场关注度靠前的龙头，低价潜力
股优先历史卡脖子/国产替代池内价格弹性更强的细分龙头。该准入语义只约束固定名单维护，
不构成收益承诺，也不允许运行时按旧 long 评分、DeepSeek 或实时价格自动换股。

长期分组分三类：

- `chokepoint`：卡脖子行业分组，来源为 `main` 历史中的
  `CHOKEPOINT_INDUSTRY_LEADERS` 并按关键核心技术、关键零部件、关键基础材料、关键软件和
  产业链供应链安全口径校正；每个行业最多 5 只固定龙头股。
- `future_growth`：高成长赛道分组，按 2026-07-24 可见官方产业政策口径和全市场长期涨幅
  复核形成固定观察池，覆盖低空经济、商业航天、AI 算力、具身智能、智能电动汽车、
  新型储能、创新药/高端医疗、生物制造和量子通信/6G 等方向；每个赛道最多 5 只。
- `low_price_potential`：低价潜力股只能从上述历史卡脖子/国产替代固定池中挑选价格弹性

`deepseek_skipped_no_eligible_candidates`；未发生物理请求时不得标记
`deepseek_incomplete`。后者只表示已有可复核候选但请求、预算、响应或校验未完整完成；
long 永久不进入复核，因此不得附加任一 DeepSeek 降级原因。

活动结构化事实接缝固定为 `deepseek_v4_review_facts_v1`。该接缝只允许模型返回可校验
事实，不允许自由文本、评级或置信度直接决定分数、风险、动作或排名。事实
集合固定包含：催化方向、重要度、确认状态、周期和引用；价格反映桶；基本面状态；行业
政策状态；监管、减持、解禁、质押、诉讼和业绩风险；事实冲突；证据覆盖；以及
`abstain`。每条事实必须引用本地 manifest 中的证据 ID、来源等级、发布时间和摘要哈希；
无法用 schema 与证据核对的字段必须拒绝或转为 abstain。

每只候选最多使用 12 条点时证据，首轮槽位仍按结构化行情 1、尾盘分钟 1、财务与研究
3、监管/质押/解禁等风险 4、新闻与公告 7 的顺序依次填充，达到 12 条即停止。空槽按
监管风险、财务研究、分钟量价、新闻公告的顺序补给；补位后监管最多 6、财务最多 5、
新闻公告最多 8，行情和分钟仍各最多 1，总数仍不得超过 12。跨来源同主体、事件 key
和证据分类的记录只保留质量更高且更早接收的一条；存在有效相反材料时至少保留一条
反证。证据 manifest 必须携带 `source_tier` 和 `event_key`，正向加分只接受官方来源或
两个独立可信来源支持。

V4 事实只由本地纯函数映射，原始维度从 50 分开始并限制在 30-70；模型不能直接返回
维度分、生产扣分或 veto：

| 结构化事实 | 本地原始变化 | 投影维度 |
| --- | --- | --- |
| 催化方向与重要度 | 高/中/低按方向记 `±15/±8/±3` | `value_quality` |
| 价格反映状态 | 未反映/部分反映/充分反映按方向记 `±12/±5/0` | `market_flow` |
| 基本面 | 改善 `+15`、稳定 `+3`、恶化 `-18` | `financial_health` |
| 行业政策 | 仅形成审计标签，不进入分数、覆盖率或已知维度数 | `industry_policy` |
| 风险事实 | 维度保持 50，只经本地风险规则形成 penalty/veto | `risk_quality` 与独立风险通道 |

证据质量同时作为维度 confidence，使变化值在融合前向 50 收缩：官方来源 `1.00`、两个
独立可信来源 `0.85`、单一可信来源 `0.65`、软信息 `0.40`，事实冲突时最高 `0.25`。

五维固定为价值质量、财务健康、资金量价、行业政策和风险质量。每维输出 0-100 的

```text
confidence_coverage = sum(raw_confidence_i * w_i)
```

| tomorrow | 18.75% | 25% | 31.25% | 0% | 25% |
| d25 | 25% | 31.25% | 25% | 0% | 18.75% |

## 12. 融合、动作与稳定排名

固定公式：

```text
final_score = clamp(
    local_score * 0.68
    + deepseek_score * 0.32
    - deepseek_risk_penalty,
    0,
    100,
)
```

`local_score` 已扣本地风险，融合时禁止再次扣 `local_risk_penalty`。所有输入使用十进制
确定性计算，最终以 `ROUND_HALF_UP` 保留两位小数。固定验收向量结果必须为 `83.40`。

生产排序固定为：`final_score` 降序、`local_score` 降序、股票代码升序。模型评级、
置信度、挑战状态和自由文本不得改变同分顺序。

本地评分完成后必须立即生成并尝试接纳 `projection_stage=local` 的 P6/SSE 快照，不能等待
DeepSeek。复核结果随后只对未冻结的同日草稿生成 `projection_stage=hybrid` 重发布；两阶段
必须有不同 `snapshot_id`/ETag，迟到结果不得覆盖冻结或更新版本。失败、超时、预算耗尽或
空响应保留已经发布的本地版。long 永远只发布 local，不调用 DeepSeek。

- today 09:36-10:30：`final_score >= 70` 可执行；

## 15. 策略验证与发布门禁

每条冻结推荐由后台结果结算器生成不可变 `RecommendationOutcome`，HTTP 请求不得触发
行情抓取、结算或写库。today/tomorrow 记录 T+1 最低价、收盘价、全市场等权基准收益、
20bp 往返模拟成本后的净超额收益、MAE 百分比和 `MAE/ATR20`；d25 分别记录 T+2、T+3、
T+5 收益及 T+1 至 T+5 的最低价和 MAE。停牌、交易日缺失、价格不连续或公司行为无法
可靠排除时标记 `insufficient_data`，不得猜测补值。

T+1 的严重回撤定义为 `MAE/ATR20 <= -1.5`，d25 定义为 `MAE/ATR20 <= -2.5`。活动运行只
建立 v18 结果审计，不生成旧策略同期影子；旧策略/v18 同期影子、配对自举和晋级门禁按
[全链路收益优化计划](strage.md) 的独立后续批次实施。在至少 60 个交易日、300 条有效
配对样本、净超额收益差的 95% 自举置信下界为正且严重回撤率下降前，不得描述为收益改善；
运行时不得自动调权或自动回退。

自动验证至少覆盖：

- 三板识别、冲突、上市第 1/5/6 日和主板 8.00/8.01、成长板 16.00/16.01；
- 行业、同行和竞争组字段变更不改变候选分、本地分、DeepSeek 分、动作或下行保护；
- 2.5%/97.5% 截尾、板内至少 100、最多 5 日回退和跨板隔离；
- 九组候选权重、九组本地权重精确合计 1.0；50 分、30%、120 只、0.85 边界；
- 换手和成交带通边界、缺失分母、d25 无双乘；
- ATR、缩量回踩、放量突破、单纯过热可执行和五类下行保护边界；
- 本地风险去重、上限、模型事实校验和禁止自由文本扣分；
- 168 物理请求、并发原子预留、失败/重试计数、14:48 截止和跨策略共享；
- 固定融合 `83.40`、ROUND_HALF_UP、排序键、TopK 60%、单行业/`unknown` 最多 2；
- 正式/观察分池分别执行行业上限，观察不挤占正式 TopK；
- local 先发布、hybrid 不同身份重发布、复核失败保留 local、冻结拒绝迟到 hybrid；
- 严重公司风险永久/90 日/三自然年边界、官方证据约束、注册表降级和旧命中保留；
- 结果结算幂等和无未来数据；
- 冻结前后竞态、旧版本回放、哈希一致性和 long 不持久化。

性能门禁使用固定 360 候选：单板预选 P95 不超过 250ms、单板单策略评分 250ms、单板

任何候选权重、评分权重、硬过滤、风险映射、动作门槛、TopK、模型路由或融合变化，

停牌、涨跌停、费用和滑点，预注册指标并控制多重检验；未取得真实前瞻交易日证据时，
只能描述结构、可靠性或性能改善，不能声称收益提高。

## 16. 明确不采用的策略

- 不让 DeepSeek 自由文本、评级或置信度直接改变分数、风险、动作或排名。
- 不由 DeepSeek 新增候选、绕过硬过滤或选择方向。
- 不进行运行时自动调权、自动晋级、在线学习或看完结果后切换动量/反转。
- 不引入多智能体生产编排、第三方项目的交易结论、券商连接或真实下单。
- 不把 `flow_proxy` 解释为不可验证的主力或机构资金。
- 不把历史回测、shadow 审计或社区 Star 当作生产收益证据。
- 不为补满 TopK 放宽质量门，不把缺失值伪装成 0 或真实中性观测。
