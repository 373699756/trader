# 荐股交易策略需求（Need）

## 1. 策略定义（业务口径）

### 1.1 `today_plan`（今早执行）
- 定义：当日短线可执行策略，目标是在盘中给出能即时执行的信号。
- 生效窗口：
  - 09:30–09:35：开盘观察
  - 09:36–10:30：主执行窗口，允许常规可执行信号。
  - 10:30–11:20：降级执行窗口，提高执行门槛。
  - 13:00–14:00：午后只观察/转入后续策略，不新增可执行信号。
- 交易口径：以当日实时点时特征与动态执行规则为主，关注当日收益与风险回收。
- 核心约束：
  - 必须通过本地硬过滤、执行窗口判断与风险门控。
  - `deepseek_veto=true` 或周期不匹配不得进入可执行 TopK。
  - 失败时允许返回 0 只，不能伪造推荐。

### 1.2 `tomorrow_plan`（明日策略）
- 定义：T 日尾盘后的明日持有策略，主要目标是 T+1 交易日实现规则退出收益。
- 生效窗口：T 日 14:30 后形成参考入场口径。
- 交易口径：T 日参考买入口径 + T+1 规则退场。
- 核心约束：
  - 禁止把 today 的短线情绪直接当作明日执行口径。
  - 强调次日催化可持续性与隔夜风险控制。
  - 受止损、止盈、滑点、冲击和停牌状态真实约束。

### 1.3 `d25_plan`（2–5 日策略）
- 定义：T 日 14:30 后参考入场，目标是 T+2 到 T+5 的可实现净收益。
- 交易口径：中短线波段持仓，偏向趋势延续、催化兑现后续、财务和行业支撑。
- 核心约束：
  - 不把 today 的纯短线情绪硬推入 swing。
  - 必须通过本地硬过滤、趋势/风格约束和风险门控。
  - T+2 起参与常规止盈，T+5 强制退出；T+1 允许保护性风险退出。

### 1.4 `long_plan`（长期持有展示池）
- 定义：独立观察池，不是交易执行策略。
- 目标：展示价值-成长-战略支撑的长期候选（约 1–5 年叙事），用于长期研究。该股票池比较固定，
- 交易属性：不产出交易动作，不参与三大策略收益验证。
- 口径要求：需至少命中价值、成长、政策/战略、卡脖子等主题线索之一，输出可解释排序。

## 2. 统一交易策略工作流

### 2.1 日内时间线与状态
- 09:15–09:30：共享预热，构建候选与证据。
- 09:30–09:35：today 观察，不输出可执行。
- 09:36–10:30：today 主执行。
- 10:30–11:20：today 降级执行。
- 13:00–14:00：tomorrow、d25。
- 14:20–14:48：补审窗口（新入围、证据变化、风险变化）。
- 14:48：停止生产可用的模型请求。后停止 DeepSeek 生产调用，推荐排序与可执行口径停止更新
- 14:50：后发布的盘中推荐名单和评分为“冻结口径快照”，不得回写交易执行策略（TopK、动作、veto、生效阈值）。
- 15:00 之后：仅用于收盘补充/展示，不替代盘中交易口径。如果程序运行在15:00之后，则获取收盘价，根据荐股测列展示，并保存数据。

### 2.2 候选构建与硬过滤
- 先形成本地候选池（默认 100–168 只），再做 DeepSeek 复审。
- 禁止 ST、退市风险、停牌高风险、低流动性、黑名单、明显异常价格与无效成交标的进入执行层。
- DeepSeek 仅能审查候选池内标的，不能新增池外股票。

### 2.3 评分与合并
- 统一排序口径：
  - `final_score = local_score * 0.75 + deepseek_score * 0.25 - risk_penalty`
- local_score 与 deepseek_score 均用于排序，不是收益承诺概率。
- DeepSeek 结果仅作为增强输入：  
  - `deepseek_veto=true` 不可进入可执行 TopK；  
  - 本地硬过滤失败者直接剔除；  
  - 结果不足/失败/失效时回退 `final_score=local_score`。

### 2.4 DeepSeek 使用规则
- 每日总上限：188 次。
- 按策略硬上限分配：
  - today_plan：70
  - tomorrow_plan：45
  - d25_plan：35
  - long_plan：18
  - shared_preheat：15
  - emergency_reserved：5
- 按时间窗口分配（预算执行阶段）：
  - 09:15-09:30 shared_preheat：15
  - 09:30-09:35 today_open_observe：15
  - 09:36-10:30 today_main：42
  - 10:30-11:20 today_late：13
  - 13:00-14:00 afternoon_main（tomorrow/swing/long_term）：65
  - 14:20-14:48 final_supplement：38
  - emergency：5
- 14:48 后停止生产可用调用，14:50 冻结后不回写交易口径。
- 物理请求层限制：每次请求最多审查 8 只候选。
- 逻辑汇总批次不计入 API 次数。
- 生产服务只读预计算结果，HTTP 推荐接口需异步调用 DeepSeek 避免阻塞。
- 缓存命中不计 API 次数；缓存条件变化后允许重审。
- 失败场景（超时、失败、超限、证据过期、schema 失败）必须保证本地链路可用，并显式标记降级来源。
- 无证据或证据不充分标记 `abstain`，不得高分。

## 3. 成交与风险口径
- 收益仿真统一按未复权参考价与冻结执行规则计算（费用、滑点、冲击、停牌/涨跌停约束真实落地）。
- 停牌、跌停导致无法交易状态需要保留真实状态，不填充虚假收益。
- 明日、2–5 日按各自持有期窗口独立计算收益字段。
- 验证、回填、晋级需按策略版本与日期口径一致，不得跨版本混算。
- DeepSeek 未参与、调用失败、超时、额度耗尽、证据过期、schema 失败时，推荐服务不得阻塞；必须返回本地链路结果。
  - 降级场景必须可见化：
      - 页面与快照需展示本地分、降级原因、是否复用本地分。
      - final_score 可写成 local_score，不得虚构深度分。

## 4. 统一边界与异常规则
- 交易策略只使用三策略：`today_plan`、`tomorrow_plan`、`d25_plan`；
- `long_plan`仅展示。
- 推荐口径可为 0 只；不能为凑满数量降低标准。
- 任何策略参数、执行口径、阈值或版本变化必须同步更新并明确生效版本。
- 页面与快照必须能展示：本地分、DeepSeek分、风险扣分、最终分、是否 DeepSeek 参与、veto 原因、降级原因（如有）。

## 5. 变更交付说明（该文档目标）
- 本文作为 `need.md` 的交易需求定义，不重复实现细节，仅保留可落地策略口径。
- 上线实施时以本文件为对照：  
  1) 策略定义是否一致；  
  2) 时间窗口是否一致；  
  3) DeepSeek 不新增池外候选是否执行；  
  4) 失败降级是否始终可用本地排序，不中断推荐服务。

## 6. 三类策略的荐股保存与展示需求（最终版）

### 6.1 适用范围
- 持久化与回溯仅覆盖三类：`today_plan`（今早）、`tomorrow_plan`（明日）、`d25_plan`（2–5日）。
- `long_plan` 仅展示，不持久化，不参与回溯与可靠性统计。
- 页面展示顺序固定为：`today_plan` → `tomorrow_plan` → `swing_plan`。

### 6.2 持久化最小字段
- 以“每策略每推荐日期每只股票”保存一条记录，主维度为 `strategy + recommend_date + stock_code`。
- 仅保存以下 5 个字段：
  - `strategy`（`today_plan` / `tomorrow_plan` / `d25_plan`）
  - `recommend_date`（仅到日）
  - `stock_code`（标的代码）
  - `anchor_price`（荐股锚点价）
  - `anchor_daily_return_pct`（荐股当天涨跌幅）
- 同一策略、同一日期、同一标的出现重复时按既定去重规则保留一条有效记录。

### 6.3 页面展示字段（固定列）
- 列表必须展示 6 列：
  1. 排名
  2. 股票
  3. 锚点价格（来自 `anchor_price`）
  4. 锚点涨跌（对应 `anchor_daily_return_pct`）
  5. 今日涨跌（当日实时涨跌幅）
  6. 锚点到现在（锚点到当前动态涨跌幅）
- “锚点到现在”不作为持久化快照值，每次刷新/轮询实时重算展示。
- “锚点到现在”必须定义为“当前时刻动态重算”，不是快照持久化值。
  - 今日涨跌字段为“当下行情层实时涨跌”；锚点涨跌字段为“荐股当日快照涨跌”。
  - 同一策略同日不得在列表里混入不同阶段的已冻结口径数据。

### 6.4 查询与回溯规则
- 列表查询必须按 `strategy + recommend_date` 读取，默认日期为当日。
- 历史回看仅通过切换 `recommend_date` 实现，切策略不得跨策略混合展示。
- 可靠性验证只基于持久化三类策略，按 `strategy + recommend_date + stock_code` 口径做回溯与对比。

## 7. 评分系统输入口径与指标来源（非代码）

  - 第一层（全局）：实时源有效性校验、合法代码/交易条件、黑名单、基础硬过滤（低成交额、涨幅过大、近涨停不可买等）
  - 第二层（共享）：事件与风险事件入池（减持、解禁、质押、公告风险）
  - 第三层（共享）：历史因子与市场状态构建（趋势/波动/流动性特征）
  - 第四层（策略层）：today/tomorrow/d25 各自执行独立资格门槛（如你们的 tomorrow_hard_reject、swing 约束）
  - 第五层（融合层）：本地评分 + DeepSeek 复核（不新增池外）后按 final_score 冻结与排序

### 7.1 数据来源（不是券商交易下单源）

- 评分链路不是接券商下单接口，而是行情与数据供应链：
  - 全市场实时行情主源：东方财富；
  - 失败回退：新浪实时行情（可配置）
  - 推荐池代码逐批补充报价：腾讯；
  - 行业/热度：AKShare 的行业榜和个股板块字段；
  - 新闻与事件：AKShare 的财联社电报/财新精选、个股新闻、解禁、减持、减持持股、财报日历、质押等接口；
  - 财务：AKShare 财务分析指标；
  - 历史序列与回测：AKShare 日线（复权）+ 本地历史库（原始/复权缓存）。
- 行情、新闻和事件都带抓取时间戳，不能用“无源字段”替代。
- 可以结合deepseek api获取相关信息

### 7.2 硬过滤（不参与任何评分）
- 先于所有策略分数计算执行；失败者不计入 `local_score/deepseek_score/final_score`：
  - 非合法A股代码、ST/退市风险、无有效价格；
  - 低成交额（默认阈值 5000 万）；
  - 一字板/极端封板；
  - 涨幅过大（接近不可交易区）；
  - “接近涨停不可买”区间；
  - 其他黑名单/异常事件风险项。
- 当日涨幅过大规则按市场上限分开执行：默认主板主阈值、创业/科创为成长阈值；按你的口径是“主板 >9%、创业/科创 >10% 时剔除”。
- 关键点：**只要被硬过滤命中，后续分数完全不计算，不是降权，是剔除。**

### 7.3 指标参与评分的逻辑（纯逻辑口径）
- 先把每只股票映射为统一字段（价格、当日涨幅、五分钟涨跌、量比、换手、振幅、MA 偏离、20/60/回撤段涨跌、历史波动、行业热度、风险标记等）。
- 对每个子维度做横截面排名/分位，缺失时降级为中性值，不与零混淆。
- 每个策略按“组件分 → 权重加权 → 风险扣分 → 市场环境修正/过热抑制”得到 `local_score`。

#### 7.3.1 today_plan（今早）
- 组件：动量、流动性、行业、情绪、风险保护。
- 权重：35%、25%、10%、20%、10%。
- 风险保护包含：追高/尾盘异常、近涨停拥挤、文本风险特征等。

#### 7.3.2 tomorrow_plan（明日）
- 组件：流动性、动量、趋势、历史优势、执行分、尾盘结构。
- 权重：流动性 30%、动量 20%、执行 15%、尾盘结构 15%、历史优势 20%。
- 叠加晚盘可执行性约束、周边风险扣分后再形成最终分。

#### 7.3.3 d25_plan（2-5 日）
- 组件：动量、趋势、流动性、执行分、不过度延伸。
- 权重：30%、25%、20%、15%、10%。
- 再乘过热抑制与市场状态权重后输出。

### 7.4 最终合成与 DeepSeek
- DeepSeek 只在已通过硬过滤候选池内复核，不新增候选。
- 产出展示分采用：  
  `final_score = local_score * 0.75 + deepseek_score * 0.25 - risk_penalty`。
- 任一 API 或审查失败时，必须保证推荐链路可用，回退到本地分数。
- DeepSeek veto 或周期不匹配不算可执行，即使局部分高也不入可执行 TopK。

#### DeepSeek 五步需求（代码对应版）

  1. 价值与质量因子复核（value_quality）

  - 目标：对候选的估值与质量做“合理性复核”
  - 输入：本地财务/估值参数（PE/PB、ROE、净利率、毛利率、营收/利润增速、负债率、现金流等）+ 证据文本
  - 输出：value_quality.assessment（结论）、value_quality.confidence（置信度）、value_quality.flags（质量异常标签）
  - 约束：缺关键财务证据时可触发 abstain，并标记需降级

  2. 财务健康深度复核（financial_health）

  - 目标：判断公司盈利与现金流持续性、拐点风险
  - 输入：利润趋势、现金流趋势、盈利质量相关字段、异常/风险标签
  - 输出：financial_health.profit_trend、financial_health.cashflow_trend、confidence、flags
  - 约束：出现现金流急转差/盈利恶化高风险时标记风险分/扣分项

  3. 市场资金面与量价复核（market_flow）

  - 目标：评估近一月资金与量价是否与“可持续上涨”一致
  - 输入：volume_ratio、换手率、委比、涨跌速度、主力净流入、振幅等 + 行情事件
  - 输出：market_flow.flow_health、price_flow_divergence、confidence、flags
  - 约束：若出现量价背离、异动放大但缺基本面支撑，可产生 risk_flags

  4. 行业/政策支持复核（industry_policy）

  - 目标：评估行业景气与政策环境是否支持该票在目标周期内兑现
  - 输入：行业行业面板、事件标题/新闻标签、政策相关证据
  - 输出：industry_policy.industry_outlook、policy_relevance、confidence、flags
  - 约束：政策方向与目标策略不一致时降低 deepseek_strategy_fit

  5. 风险综合评估（risk_assessment）

  - 目标：提取短线风险、监管风险、业绩分化、隔夜/舆情风险
  - 输入：公告风险标记、事件风险字段、资金面偏离、财务异常
  - 输出：risk_assessment.risk_level、risk_penalty、flags
  - 约束：高风险会触发 deepseek_veto（默认不可执行）或增加 risk_penalty



## Web 布局需求（v1）

  1. 页面框架与主导航

  - 顶层采用单页布局，支持两个主 Tab：
      - 推荐
      - 策略

  - 默认进入推荐池页，顶部状态条常显。
  - 所有页面必须保留“仅供研究，不构成投资建议”底部说明。

  2. 推荐页（核心推荐区）

  - 顶部第一行是刷新与状态入口：
      - 刷新按钮
      - 状态卡片：状态、行情更新、推送状态、行情时间、市场情绪、DEEPSEEK API 已用/剩余、DeepSeek 状态。
      
  - 表格区域上方有“推荐池筛选动作摘要”区（当前策略是否有推荐/可执行数/执行口径提示等）。
  - 列表 Tab 顺序固定为：今早、明日、2-5日、长期。
  - 默认展开今早 Tab。
  - 切换 Tab 时支持懒加载（无历史数据时显示“切换后加载”占位文本），切回已有 Tab 用缓存优先渲染。

  3. 今日 / 明日 / 2-5 日列表（同结构）

  - 表头固定 9 列：
      1. 排名
      2. 股票（含代码、名称、行业）
      3. 最新价
      4. 今日涨跌
      6. 成交额（含成交额，换手率）
      7. 总市值
      8. 操作评分（本地分/DeepSeek分/风险扣分/最终分）
      9. 动作（执行/观察/不可执行）
      10. 原因(解释这样评分的说明)
  - 行显示要求：
      - 每行至少包含股票名、名称、行业与评分链路核心结果。
      - 涨跌、换手、成交额等指标带颜色语义（涨红跌绿）。
      - 不可执行行要可视化体现（如 tag/颜色/禁用动作）。

  4. 长期列表（展示池）

  - 显示为单独表：
      - 列名：股票、行业，最新价、涨幅、目标价、长期观察依据。

  - 长期池只展示，不纳入交易动作与三策略可执行交易环节。

  - 推荐数据刷新机制：
      - 支持 SSE 实时推送；
      - 无推送时降级 60 秒轮询；
      - 异常或无数据时显示明确占位文案（例如“加载中/等待推荐池数据/切换到本页后加载/暂无数据”）。

  6. 策略页布局

  - 增加工具区：
      - 股票代码输入 + 走势预测
      - 工具结果展示区
  - 验证区是 validation-workbench 下两个分块：
      - validation-batches（方批次区）保存批次列表（分页），按日期/类型/数量/保存时间显示
      - validation-results（下方明细区）明细表（11 列），显示排名、类型、股票、锚点价、锚点涨跌、主周期、主周期收益、交易成本、执行状态、今日涨跌、锚点到现在
  - 分析结果必须支持策略 Tab（今日、明日、2-5日）切换。

  7. 适配与可用性约束

  - 所有表格必须有横向滚动容器，避免窄屏列挤压。
  - 主页、状态卡、按钮、表头需在窄屏下换行保留可读性。
  - 保留当前配色和字号体系，支持高频更新时行/状态更新不卡顿。
  - 所有空状态、加载状态、错误状态不可空白，必须有文案。

  8. 验收标准

  - 默认显示推荐池页；推荐池有 4 个 Tab，顺序及长期仅展示行为一致。
  - 任何时段推荐页和验证页均能从接口失败中降级显示旧快照/错误提示，不出现白屏。
  - 今日/明日/2-5 日列表均满足列定义；长期列表仅展示不出现交易动作依赖。
  - 验证页左侧批次与右侧明细能保持联动，翻页后仍可选择指定批次查看明细。
