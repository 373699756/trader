# 荐股策略与预测说明

本文档是荐股业务规则的唯一说明，内容以当前代码为准。软件架构、接口、数据文件和运行方式见 [software_design.md](software_design.md)。

所有结果仅用于研究，不构成投资建议，也不保证盈利。系统能保证的是策略口径可追溯、结果可回填、未通过验证时禁止执行，不能保证未来收益。

## 0. 生产冻结、评分取舍与当前状态

当前策略优化的基本判断是：系统不缺更多评分组件，缺的是足够真实、可复现、样本外的组合净收益证据。因此生产期默认使用冻结规则分作为候选排序，不把未经验证的预期收益、概率、Meta-Label、事件分、固定权重集成或 DeepSeek 输出接入生产排序、过滤或仓位。

截至 2026-07-12 的本地 readiness 审计，`.runtime/strategy_validation.sqlite3` 中真实 OOS 仍为 0：

| 项目 | 当前值 | 门槛 | 状态 |
|---|---:|---:|---|
| 真实 OOS 交易日 | 0 | 60 | blocked |
| 日级组合基线交易日 | 0 | 60 | blocked |
| DeepSeek/事件有效事件日 | 0 | 60 | blocked |
| 正式替换 OOS 交易日 | 0 | 120 | blocked |

这些 blocker 意味着：P3 样本门控、P4 可重建收益模型 artifact、P5 组合消融结论、P6 DeepSeek/事件反事实收益试验和 P7 灰度都不能标为完成。工程可以继续完善采集、审计和展示，但不能用回放样本、同日股票数或 shadow 诊断冒充真实 OOS 交易日。

生产冻结事实：

- 机器事实源是 `config/production_baseline.json`，启动时默认 `PRODUCTION_FREEZE_ENABLED=1`。
- 策略版本、明日策略 Top-K、候选过滤阈值、退出规则和生产权重均按冻结清单锁定。
- `tomorrow_picks` 是首个研究策略，生产 K 固定为 5；K=3/10 只用于敏感性诊断。
- 每个推荐响应和验证信号都带 `generation`，包含基线 ID、策略版本、排序字段、有效开关、权重/输入/输出指纹和重放上下文。
- `baseline_status=drift_detected` 表示代码或运行配置与冻结清单不一致，不能混入同一实验基线。

评分取舍：

| 机制 | 当前生产状态 | 说明 |
|---|---|---|
| 候选资格硬过滤 | 保留 | ST/退市、停牌、无法买入、流动性和数据缺失是可交易约束，不伪装成 alpha。 |
| 冻结规则分 | 保留 | 无合格模型前的唯一生产排序与候选压缩器，不解释为概率或预期收益。 |
| 真实成交与成本模型 | 保留 | 入场、退出、未成交、费用、滑点和冲击成本决定收益是否可采信。 |
| `ScoreCalibrator` 分桶概率 | 诊断-only | 不参与交易，不作为 `p_win` 替代。 |
| 启发式 `expected_return` fallback | 移除生产作用 | 不再用原始分手工换算伪收益；样本不足时只输出不可用诊断。 |
| 手工 `rank_score` 拼分 | 移除生产作用 | 排序只允许冻结规则分或通过门控后的 `predicted_net_return`。 |
| Meta-Labeling / Event Alpha / Ensemble | shadow/research-only | 不改变 `trade_action`、排序、过滤或仓位。 |
| DeepSeek rerank/veto/penalty | shadow-only | 只做结构化事件抽取和反事实归因；安全类硬风险另行处理。 |

唯一生产排序规则：

```text
无合格收益模型: ranking_key = frozen_rule_score
收益模型通过 G3/G4 后: shadow_ranking_key = predicted_net_return
收益模型通过 G5 后: production_ranking_key = predicted_net_return
```

G3/G4 只代表 shadow eligible，不能替换生产排序；只有 G5 通过后才允许把 `predicted_net_return` 作为生产排序。`p_win`、`downside_q10`、预测区间和 DeepSeek 观点只能作为解释、拒绝低置信交易或研究字段；不能手工拼成第二套排序分。

## 1. 策略总览

| 页面名称 | 内部存储名 | 当前版本 | 主验证周期 | 执行定位 |
|---|---|---|---|---|
| 盘中强势观察 | `short_term` | `short_term_v2_observation` | 次日表现仅辅助 | 永久零仓位观察池 |
| 明日优先 | `tomorrow_picks` | `tomorrow_picks_v9_next_open` | 次日开盘至次日收盘 | 盘后确认、次日开盘执行 |
| 2-5日持有 | `swing_picks` | `swing_2_5d_v3_next_open_exit` | 次日开盘至动态退出 | 次日开盘入场，最多持有5个交易日 |

代码中的规范名称是 `today_picks`、`tomorrow_picks`、`swing_2_5d_picks`，SQLite 和现有接口继续使用上表的内部存储名。映射定义在 [strategies/types.py](../stock_analyzer/strategies/types.py)。验证库、API 参数、指标缓存、`baseline_id` 和运行时 gate 一律使用内部存储名；规范名只在策略类型映射内部使用。

推荐数量是上限，不是必须凑满。严格池没有结果时，系统会尝试展示通过基础安全过滤的备选；备选始终 `execution_allowed=false`、仓位为0。只有基础候选本身为空时才返回空列表。

## 2. 公共候选链路

三类结果共享以下前置处理：

1. 行情标准化，只保留支持的沪深主板、创业板和科创板代码。
2. 排除 ST、退市、停牌、无有效价格和低流动性股票。
3. 合并历史量价因子、事件风险、基本面、行业强度、热度和舆情等可用数据。
4. 计算市场状态，按 `risk_on`、`balanced`、`risk_off` 调整分数和组合暴露。
5. 本地策略评分后执行硬过滤、过热抑制和主题集中度限制。
6. P0 冻结期 DeepSeek 只在本地候选池内做 shadow 复核，不新增股票，也不改变生产排序、过滤或仓位。
7. 最后应用真实验证门控；门控失败时保留展示，但全部降为零仓位备选。

历史因子由 `ENABLE_HISTORY_FACTORS=1` 默认开启，包括 3/5/10/20 日收益、均线偏离、成交额变化、突破和波动率等。请求缺少缓存时只后台分批预热，不阻塞推荐接口。历史因子覆盖不足会在健康状态中报警；2-5日持有策略还会直接降级为不可执行。

## 3. 盘中强势观察

入口是 `score_today_picks(...)`，核心实现位于 [scoring.py](../stock_analyzer/scoring.py)。

### 3.1 目标与因子

用于发现当日盘中已经走强、但尚未明显透支的股票，主要观察：

- 当日涨跌幅、涨速、五分钟变化、量比、换手和成交额。
- 短周期动量、行业强度、热度和舆情。
- 高涨幅、高量比、高换手、冲高回落和事件风险。

最低分默认由 `TODAY_RECOMMENDATION_MIN_SCORE=60` 控制，页面展示上限由通用推荐上限控制。

### 3.2 执行约束

当前系统没有分钟级同日入场和退出验证，因此该策略只承担观察功能：

- 所有行都标记为 `intraday_observation`。
- `execution_allowed=false`，`trade_action.position_size=0`。
- 可以进入自动快照作为辅助验证样本，但不进入生产执行门控，也不形成买入指令。
- 次日收益只用于辅助归因，不能证明盘中买入策略有效。

## 4. 明日优先

入口是 `score_tomorrow_picks(...)`，目标是从尾盘结构、成交承接、温和动能和买入安全中筛选次日机会。

### 4.1 分时模式

策略按行情时间分成两个阶段：

- `09:30 <= 时间 < 14:30`：放宽部分早盘阈值，收盘结构因子使用中性值。结果仅供盘中观察，全部仓位为0，不能提前当作尾盘买入信号。
- `14:30 <= 时间`：恢复严格尾盘过滤和收盘结构评分，重新确认是否能进入重点观察。

放宽模式只解决早盘数据不完整导致候选池为空的问题，不代表降低最终执行标准。

### 4.2 评分与风险

主要正向因素：

- 流动性：成交额、换手率。
- 动能：温和涨幅、涨速、量比、60日相对强度。
- 趋势：60日和年内位置、振幅控制。
- 尾盘结构：收盘位置、日内承接、假拉升风险。
- 历史边际：近期收益、均线、突破和波动率。

主要风险控制：

- 接近涨停、极端量比、极端换手、过高振幅和速度异常直接过滤。
- 默认对涨幅 `4.5%-7%` 且收盘位置低于 `0.6` 的股票增加7分风险惩罚。
- 重点观察默认最低分68，风险惩罚、过热抑制、60日和年内涨幅还要通过额外门槛。
- 单一主题最多2只重点观察，页面同主题最多5只。

### 4.3 主推与备选

- 页面最多展示8只，最多5只进入重点观察。
- 严格池为空时调用备选池，不返回一张看似“系统没运行”的空表。
- 备选池仍执行基础流动性、过热、涨停风险、振幅和历史风险过滤。
- 备选只用于继续观察，仓位固定为0。
- 验证快照可保存最多36只宽候选，用于积累样本；页面收窄不应导致验证样本同时收窄。

### 4.4 执行与收益口径

执行假设是盘后形成候选、下一交易日开盘入场。展示可以保留 `next_close_return` 毛收益，即下一交易日开盘到收盘的收益；门控、模型训练和晋级只使用 `primary_return_net` / 主周期净收益，必须扣除双边成本与流动性滑点。盘后快照不得按已经确定的当日收盘价回填可执行收益。

以下情况从可执行成交样本中标记为未成交或跳过，仍必须计入样本守恒、未成交率和机会成本，不能静默删除：

- 次日涨停或一字板导致不可买。
- 次日开盘相对信号价高开超过 `TOMORROW_HIGH_OPEN_SKIP_PCT`，默认3%，避免追高回落。

系统还保留最多5日的动态退出收益作为辅助统计，用于观察“次日偏弱后2-5日转正”的比例，但它不是明日优先的主门控指标。

## 5. 2-5日持有

入口是 `score_swing_2_5d_picks(...)`，目标是寻找短周期趋势仍在、放量温和、没有明显透支的候选。

### 5.1 评分重点

- 5/10/20日收益和均线位置。
- 成交额、换手、量比和成交额变化。
- 突破、波动率、振幅和过热程度。
- 市场状态、行业扩散和主题集中度。

默认最低分是60。历史因子可用率低于 `SWING_MIN_HISTORY_FACTOR_COVERAGE=0.30` 时，最多展示8只降级候选，但全部禁止执行；这避免策略在核心趋势因子缺失时退化成单纯按当日成交量选股。

### 5.2 动态退出

主指标是 `exit_return`，从下一交易日开盘价开始模拟，最多持有5个交易日：

- 止盈：`+8%`。
- 止损：`-5%`。
- 移动止损回撤：`4%`。
- 未触发条件时在期限结束退出。

统计结果扣除交易成本和按流动性估算的滑点。页面的“2-5日退出”必须优先读取 `exit_return`，不能用固定第5日收盘收益或不可成交的信号日收盘价收益替代动态退出结果。跳空跌破止损价按次日实际开盘价成交；连续封死跌停必须延迟到首个可卖交易日。

## 6. 分层与执行语义

推荐行的核心字段：

| 字段 | 含义 |
|---|---|
| `tier=primary_watch` | 通过当前策略和验证门控、可进入执行评估的重点观察 |
| `tier=backup_pool` | 可展示但不可执行的备选 |
| `execution_allowed` | 是否允许进入后续组合构建 |
| `trade_action.position_size` | 建议仓位；备选和盘中观察必须为0 |
| `recommendation_class` | `intraday_observation`、`next_day_priority` 或 `hold_2_5d` |
| `profit_window` | 仅盘中观察、次日或2-5个交易日 |

前端以 `execution_allowed` 和仓位为最终准则。即使旧缓存残留了“买入”文本，只要 `execution_allowed=false`，页面也必须显示“备选观察、仓位0、不执行”。组合构建同样排除这些行。

## 7. 真实验证门控

生产门控只适用于明日优先和2-5日持有，计算逻辑位于 [app_support.py](../stock_analyzer/app_support.py) 和 [strategy_validation.py](../stock_analyzer/strategy_validation.py)。

全部条件同时满足才允许 `execution_allowed=true`、非零仓位和生产执行评估：

1. 当前策略版本至少有60个真实前瞻交易日。
2. 真实交易日主周期平均净收益大于0。
3. 真实交易日净胜率不低于50%。
4. 真实交易日组合净收益的95%置信下界大于0。
5. 按日组合收益复利计算的最大回撤优于默认硬限制 `-8%`。

门控读取失败也按失败处理。失败、样本不足或策略退场都不会隐藏整张表，而是将候选降为备选观察并把仓位清零。此时页面仍可展示 `backup_pool`，但不得显示为可执行重点推荐。

交易成本基准为 `VALIDATION_TRADE_COST_PCT=0.25%`，再按成交额档位增加滑点。门控优先使用真实交易日聚合指标，历史回放不计入真实日门槛。

### 7.1 科学评价标准

生产排序和验证必须复现可执行交易，而不是评价单只股票的理论涨跌。统计单位固定为交易日组合收益；同日多只股票不是独立样本。

主检验指标：

```text
daily_incremental_net_return
  = 新方案每日等权 Top-K 净收益 - 冻结基线每日等权 Top-K 净收益
```

生产晋级必须同时满足：

- OOS 日均增量净收益为正，且按交易日 block bootstrap 的 95% 置信区间下界大于 0。
- 扣除多重试验后仍通过；少量预注册候选用 Holm，持续研究用 Benjamini-Hochberg FDR。
- OOS Sortino 不下降，最大回撤恶化不超过预设容忍度，换手和容量约束通过。
- 至少 60 个 OOS 交易日用于首次灰度，120 个 OOS 交易日用于正式替换；每个关键市场状态至少 20 日，否则只使用全局模型。
- 结果不能由单月、单股或单一行业贡献；去掉最佳 5 个交易日后仍不为负。
- 高成本情景下仍不显著劣于基线。

`胜率`、`平均单股收益`、`综合分`、`IC`、`AUC`、`解释文本质量`均为诊断指标，不能单独触发上线。

### 7.2 时序验证协议

统一使用 anchored walk-forward，并按持有期执行 purge/embargo：

1. 训练集只使用测试窗口开始前已经成熟的标签。
2. `tomorrow_picks` purge 至少 1 个交易日；`swing_picks` purge 至少最大持有期 5 个交易日。
3. 超参数选择仅在训练/验证窗口完成；最终测试窗口只评估一次。
4. 所有候选模型在完全相同的日期、候选池、Top-K、成本和缺失样本上做配对比较。
5. 保存每折预测，而不只保存汇总值，以便重算、审计和反事实比较。
6. 每次新增因子或阈值都登记为一次试验；禁止反复查看测试集后继续调参。

### 7.3 因子保留规则

对每个因子执行 leave-one-feature-out 消融，不凭金融叙事保留。因子进入生产必须满足：

- 覆盖率不低于 95%，缺失值不能通过未来数据回填。
- 在多数 OOS 折中方向稳定，且去掉该因子后 Top-K 增量净收益下降。
- 加入该因子后组合净收益的 bootstrap CI 改善，或在净收益不降时明显降低回撤/换手。
- 与已有因子高度冗余时只保留更稳定、便宜、及时且易解释的一项。
- 连续两个 60 日监控窗口无边际贡献或方向反转，自动降为 shadow；再持续一个窗口仍无贡献则删除。

优先审计顺序：

1. 流动性、可买入性、价格限制归入资格/成本层，不参与 alpha 归因。
2. 中短期动量、相对行业/市场强度、量价确认作为首批 alpha 候选。
3. 过热、振幅、波动率、回撤归入风险或收益模型特征，避免在过滤、扣分和仓位中重复惩罚。
4. 尾盘结构仅在 14:30 后特征完整时参与明日模型；盘中不得填充中性值后假装完整。
5. 基本面与事件必须 point-in-time 对齐后再进入研究，无法确认发布时间的字段删除。

## 8. 历史回放与调参

生产回放版本为 `replay_v2_production`，只支持明日优先和2-5日持有。回放会重新构造历史横截面，并调用当前生产评分、硬过滤和分层逻辑；旧回放版本不进入当前指标。

回放只有历史量价数据，缺少当时完整盘口、新闻和真实执行条件，因此只能粗筛，不能替代真实前瞻样本。

权重校准使用 walk-forward OOS：

- 至少30个有效交易日才允许自动评估参数应用。
- 至少60个真实交易日。
- 平均净收益、净胜率和回撤门控必须通过。
- OOS 改善超过配置边际且多数 fold 为正，才允许写入权重覆盖文件。
- 因子 IC 至少30个样本后才允许启用自动加权。

## 9. DeepSeek 的角色与费用控制

DeepSeek 是本地规则后的 shadow 风险研究层，不是收益保证器：

- 可以输出 `priority`、`watch`、`avoid`、`veto`、风险惩罚和理由，但冻结期只用于反事实归因；生产只允许代码规则识别的硬风险，不采纳 LLM 主观 veto。
- 只复核本地候选，不扫全市场、不新增股票。
- 默认只复核通过执行门控且非零仓位的候选；备选池、盘中观察、`watch_only` 和仓位0候选不进入 API payload。
- `DEEPSEEK_SHADOW_ONLY=1` 时融合分和离线规则不会接管生产排序；解除冻结必须形成新基线。
- 归因结果不为正时应降低对应策略 alpha，必要时降到0，只保留本地策略。
- 新闻上下文和 DeepSeek 大盘门控默认关闭，只有真实归因支持后才应开启。

默认调度只覆盖 `tomorrow_picks`：

- 09:30-11:30、13:00-14:30：最多每半小时一个槽位，使用基础模型并限制候选数。
- 14:30-15:00：按请求和候选变化调用，不固定死时刻；相同候选复用结果，最短间隔默认60秒。
- 每日总调用上限默认30次，Pro上限5次。
- 推荐页缓存、同槽结果和候选签名相同的结果会复用。
- 验证页 GET 不触发 DeepSeek；盘后验证复盘只有新增至少5个真实交易日才再次调用。

三策略总入口可使用一次批量 rerank，减少重复 prompt 和公共候选字段。DeepSeek 故障、超时、限额或未配置密钥时全部降级到本地结果。

## 10. 个股预测

`GET /api/stock-prediction/<code>` 聚合股票是否进入三类候选、市场状态、风险和多策略共识，输出的是方向倾向，不是目标价保证。

处理顺序：

1. 优先使用实时标准候选。
2. 股票被基础过滤时返回过滤原因和风险诊断。
3. 实时源缺失时可用历史行情兜底，但明确标记无法确认当前交易状态。
4. 默认只运行本地预测；请求参数 `deepseek=1` 时才生成 DeepSeek 优化建议。

`ENABLE_STANCE_TRACKING=1` 时，优化建议中的 stance 会保存并按最多5日动态退出回填。该开关默认关闭，未开启时不能把个股 stance 当成已经验证的信号。

## 11. 维护规则

- 策略名称、版本、主收益字段或退出参数变化时，必须同步更新本文档和对应测试。
- 页面展示口径、验证口径和后端门控必须使用同一主收益字段。
- 新策略版本不能混入旧版本指标；回放与真实样本必须分开统计。
- 任何参数优化先 shadow、再 OOS、最后人工确认，不允许 DeepSeek 直接修改正式权重。
- 评估策略时至少报告净收益、真实日净胜率、最大回撤、样本交易日数和执行跳过数，不能只看单票最高收益。
- 一项机制累计两个完整 OOS 窗口仍无正增量时，移除生产接线和页面主展示。
- DeepSeek/事件特征累计 60 个有效事件日无正增量时，关闭自动调用，仅保留人工按需分析。
- 当全部策略均未通过健康门控时，正确输出是“无推荐/现金”，不是降低标准凑足股票。
