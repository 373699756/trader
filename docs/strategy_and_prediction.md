# 荐股、评分与预测唯一规范

本文档是工程内荐股、评分、退出、验证、实时行情和 DeepSeek 接入的唯一业务规范。`docs/plan.md` 仅保留历史研究计划，不作为当前口径来源；其他文档只能引用本文，不再维护独立策略口径。

系统只提供研究推荐，不连接券商、不自动下单，也不承诺或保证收益。策略优化目标是提高扣除成本后的真实样本外收益与稳定性；“高概率”必须由持续积累的真实数据验证，不能由模型措辞代替。

## 1. 只保留三个策略

| 页面名称 | 存储名 | 当前版本 | 目标 | 是否模拟交易 |
|---|---|---|---|---|
| 今天策略 | `short_term` | `today_picks_v2_missing_history_neutral` | 推荐信号后到当天收盘仍可能继续上涨，并记录明日策略是否重合（用于分层标识） | 否，只评价 T 日延续 |
| 明日策略 | `tomorrow_picks` | `tomorrow_picks_v13_missing_history_neutral` | T 日 14:30 后参考买入，T+1 按规则退出取得净收益 | 是，P0 最高优先级 |
| 2–5 日策略 | `swing_picks` | `swing_2_5d_v5_missing_history_neutral` | T 日 14:30 后参考买入，在 T+2 至 T+5 获取净收益 | 是 |

注：三项“当前版本”以 `config/runtime.json` 的实际配置为准，本文中的口径描述以该配置运行时为准。

`short_term` 可以进入自动快照作为辅助验证样本，但不属于可执行策略，不生成模拟持仓、费用或成交记录。

验证库、API 参数、指标缓存、`baseline_id` 必须统一使用表中的存储名，不得重新引入页面别名作为持久化主键。

旧的收盘集合竞价专用策略、v10/v11 challenger 排名、通用 LLM 混分以及额外荐股观点不再属于生产策略。

## 2. 每日正确时间线

### 2.1 盘中预计算

1. 交易日上午 09:40–11:20、下午 13:00–14:20，内部调度器最多每 20 分钟触发一次候选预计算，午休不运行。
2. 每次先由本地评分器形成宽候选，再由 DeepSeek Flash 批量提取公告、新闻和事件特征。
3. 只更新新进入候选、证据哈希变化或缓存失效的股票；命中缓存时不调用 API。
4. 14:20 是当天最后一个固定计划槽位；14:30 前最多使用 30 次调用，为尾盘按需分析至少预留 20 次。
5. 14:30–14:48 不设固定调用点，只在候选变化、新证据出现或用户按需触发时异步预计算。
6. DeepSeek 调用的硬截止时间是 14:48，单交易日所有策略和模型共享最多 50 次请求尝试。
7. 当前 DeepSeek Meta 未通过 G3，所有 DeepSeek 结果均为影子特征；14:30 后结果不能改写正式排名。

### 2.2 推荐冻结

1. T 日 14:30 形成三个策略的本地初版，推荐 HTTP 请求不等待 DeepSeek。
2. 14:30–14:48 推荐允许随本地行情、候选池和截止前已完成的本地特征微调。
3. 14:48 停止生产可用的 API 请求并开始最终计算，最迟 T 日 14:50 冻结并发布推荐。
4. 14:50 后正式股票名单、排名、动作和评分不可变；价格、涨跌幅、量比、换手率和成交额继续更新。
5. 明日和 2–5 日策略的参考买入窗口为 T 日 14:30–14:50。系统只给推荐，用户是否及何时成交不由系统控制。
6. 本策略不再假设 14:55–15:00 集合竞价成交，也不使用收盘价伪装成 14:30 后的实际参考入场价。

### 2.3 盘后

- 回填今天策略的 T 日收盘延续收益。
- 到期后回填明日和 2–5 日策略的未复权成交结果。
- 构建本地 Top K 与 DeepSeek Meta 挑战 Top K 的同池反事实。
- 可使用 Pro 模型分析难例，但盘后结果不能回写当天已冻结推荐。

### 2.4 三层实时行情与推送

行情刷新和策略重算分层处理，不能用一次全市场下载承担页面实时展示：

| 时段 | 全市场发现层 | 候选池层 | 最终推荐池层 |
|---|---:|---:|---:|
| 09:25–14:20 | 300 秒 | 150 只、60 秒 | 15 秒 |
| 14:20–14:30 | 180 秒 | 150 只、30 秒 | 10 秒 |
| 14:30–14:48 | 120 秒 | 150 只、15 秒 | 5 秒 |
| 14:48–14:50 | 强制最终全市场刷新 | 15 秒 | 5 秒 |
| 14:50–15:10 | 停止重排 | 停止重排 | 15 秒，仅更新行情 |
| 午休、收盘后 | 停止计划刷新 | 停止计划刷新 | 打开页面时读取最近成功快照 |

执行规则：

- 应用进程内只能有一个中央行情调度线程；SSE 客户端只消费共享快照，不能各自下载行情。
- 上一次同层任务未结束时跳过下一次，禁止重叠下载。
- SSE 每秒检查逐股票行情版本，仅在任一股票时间戳变化时推送完整一致载荷。
- SSE 断线自动重连；HTTP 每 60 秒轮询一次作为兜底。
- 最终推荐池使用代码级批量行情，不能为更新 18 只股票重新下载整个市场。
- 候选池行情合并进本地策略输入；候选变化只触发确定性重算，不触发 DeepSeek HTTP 请求。

每只股票必须独立保存：

```text
quote_timestamp
quote_source
quote_age_seconds
quote_stale
quote_source_deviation_pct
quote_source_mismatch
```

不得用批次内最大时间戳代表所有股票。盘中单只股票超过 30 秒未更新必须标记过期；缺失股票只对该股票回退。主备源在相近时间内价格偏差超过 0.5% 时标记异常，不得静默覆盖。

运行目标：14:30 前最终推荐池行情年龄 P95 不超过 20 秒，14:30 后不超过 8 秒，全市场有效覆盖率不低于 98%。这些是运行质量目标，不是收益承诺。

## 3. 公共数据和硬过滤

三个策略共享以下数据纪律：

- 信号和成交验证使用未复权原始价格；复权价格只用于历史趋势因子。
- 行情、公告、新闻、财务和风险事件必须有可审计时间戳，且不能晚于信号截止时间。
- 只覆盖主板、创业板和科创板的有效 A 股代码。
- 排除 ST、退市风险、无有效价格、成交额不足、不可买的一字涨停和明确黑名单股票。
- 默认最低成交额为 5000 万元。
- 主板和成长板使用不同的可买涨幅上限，避免已经接近涨停的不可实现推荐。
- 风险门控可以输出 0 只推荐；不得为了凑满数量而降低标准。

共同候选处理顺序：

```text
实时点时行情
→ 代码/市场/价格/成交额硬过滤
→ 事件与黑名单风险
→ 历史量价和市场状态特征
→ 策略独立资格过滤
→ 策略评分与风险扣分
→ 主题集中度限制
→ 验证健康门控
→ 读取DeepSeek预计算特征
→ 冻结推荐
```

`score` 是 0–100 的相对排序分，不是上涨概率、目标收益率或盈利承诺。只有经过样本外校准的概率字段才允许命名为 probability；当前 DeepSeek Meta 概率仍是影子字段。

## 4. 今天策略

### 4.1 目标和标签

今天策略回答的问题是：“从当前信号时点到 T 日收盘是否仍能延续上涨？并记录是否与明日策略重合。”

```text
today_continuation_return = T日收盘未复权价 / 信号时点参考价 - 1
today_positive = today_continuation_return > 0
```

A 股当天新买不能当天卖，因此该策略：

- `execution_allowed=false`；
- 不建立模拟仓位；
- 不扣虚构的买卖费用和滑点；
- 不把 T+1 行情误作今天策略结果；
- 只评价信号时点至收盘的观察收益。

今天策略采用“单层观测”机制：

1. `TODAY_RECOMMENDATION_MIN_SCORE`（默认 60）形成“今日延续重点观察”；
2. 若第一层不足则不再补齐，直接返回已满足条件的观测池（不足显示为空）。

明日重合不再是“是否显示”的硬条件。明日重合仅用于层内优先级和展示标记，不改变显示资格。

### 4.2 评分重点

本地评分主要覆盖：

- 当日涨幅、速度、五分钟变化和收盘位置；
- 量比、成交额、换手和流动性；
- 行业扩散、热度和市场风险状态；
- 冲高回落、过热、异常波动和事件风险。

冻结基线的概念权重为动量 35%、流动性 25%、行业 10%、情绪 20%、风险保护 10%；实际最终分还会经过硬过滤、风险扣分和市场状态修正。只要有候选，今天策略会展示满足 `TODAY_RECOMMENDATION_MIN_SCORE` 的强观察；不足则返回空列表。

### 4.3 DeepSeek 特征

DeepSeek 只判断事件是否仍在发酵、是否已兑现、是否与“今天剩余时段”匹配以及是否存在当日回落风险。它不改变今天策略的本地排序。

## 5. 明日策略（P0）

### 5.1 目标和标签

明日策略是系统最重要的收益策略：T 日 14:30 后按信号参考价观察入场，在 T+1 依据固定规则退出。

```text
tomorrow_expected_return = T+1规则退出价 / T日信号参考价 - 1 - 费用 - 滑点 - 冲击成本
tomorrow_positive = tomorrow_expected_return > 0
primary_return_field = signal_exit_return
```

T+1 最高价、最低价、最大潜在涨幅只用于诊断，不能替代实际规则退出收益，也不能计作“赚钱次数”。

### 5.2 严格候选过滤

14:30 后的严格候选通常需要同时满足：

- 当日涨幅处于可买区间，不能接近涨停或明显走弱；
- 量比约在 0.9–5.0 的健康区间；
- 换手率约在 1.5%–20%；
- 振幅低于 12%；
- 收盘位置和尾盘承接不能明显偏弱；
- 速度约在 -2.2 至 4.2，避免急拉追高和快速回落；
- 历史趋势不能明显下行，20 日波动不能过高；
- 成交额和近涨停流动性必须可执行。

阈值来自冻结生产基线；改变阈值必须发布新策略版本，不能与旧样本混算。

### 5.3 评分构成

明日策略的核心分量是：

- 流动性：成交额与换手的横截面位置；
- 动量：当日涨幅、速度、量比和中期涨幅；
- 趋势：60 日、年内趋势和振幅质量；
- 历史优势：5/10/20 日收益、均线距离、突破和历史相似信号；
- 执行质量：是否容易追高、滑点和成交冲击；
- 尾盘结构：收盘位置、尾盘承接及假拉升风险；
- 风险扣分：过热、弱收盘、异常换手、波动、事件和黑名单风险；
- 市场状态修正：风险开、均衡、风险关三种环境。

冻结基线的主权重为流动性 30%、动量 20%、执行 15%、尾盘结构 15%、历史优势 20%。`score` 仍然只是排序分。正式 Top K 预注册为 5，Top3 和 Top10 只用于敏感性诊断。

### 5.4 分层和空仓

- 达到主门槛并通过主题、市场和验证门控的股票标记为“重点观察”。
- 未通过主门槛但满足备选最低分的只能进入“备选观察”，不应被解释为正式买入建议。
- 同一主题的重点观察默认最多 2 只，防止组合收益依赖单一题材。
- 主池为空或真实验证门控失败时，允许正式推荐 0 只。

### 5.5 T+1 退出

验证按 T+1 单日窗口执行：

- 止损阈值默认 5%；
- 止盈阈值默认 8%；
- 移动止损默认 4%；
- 当日日线同时触发止盈和止损但无法确定先后时，采用保守顺序，先判断止损；
- 一字跌停无法卖出时，推迟到首个可交易日并如实记录；
- 未到期、停牌、价格缺失或无法成交的结果标记 pending/unfilled/unknown，不能填充为 0 收益。

## 6. 2–5 日策略

### 6.1 目标和标签

2–5 日策略在 T 日 14:30 后按信号参考价观察入场，追求 T+2 至 T+5 的可实现净收益。

```text
swing_expected_return = 规则退出价 / T日信号参考价 - 1 - 费用 - 滑点 - 冲击成本
swing_positive = swing_expected_return > 0
primary_return_field = signal_exit_return
```

### 6.2 评分重点

- 5/10/20 日动量和 20 日突破；
- 均线位置、中期趋势和行业强度；
- 成交量持续性、成交额和换手质量；
- 波动率、过度延伸和回撤风险；
- 执行成本、主题集中度和市场状态。

冻结基线的概念权重为动量 30%、趋势 25%、流动性 20%、执行 15%、不过度延伸 10%。

### 6.3 T+1 至 T+5 退出纪律

- T+1 已经允许卖出，但只允许保护性止损和风险退出。
- 正常止盈最早从 T+2 开始。
- T+2 至 T+5 按止盈 8%、止损 5%、移动止损 4% 的冻结规则动态退出。
- 未提前触发时 T+5 强制按规则结束持有。
- 一字跌停导致不可卖出时延后到首个可交易时点，并记录真实延迟。

这条规则避免使用 T+1 日内高点虚构 T 日买入后的即时止盈，也避免事后挑选 2–5 日中的最佳卖点。

## 7. 成交与成本口径

明日和 2–5 日策略都只是推荐，但为了验证收益，系统采用冻结的模拟执行策略：

- 入场：T 日 14:30 后的信号时未复权参考报价；
- 费用：双边费率；
- 滑点：按成交额分层；
- 市场冲击：按目标金额与成交额估算并设置上限；
- 涨停：不可买则 unfilled；
- 跌停：不可卖则延迟；
- 停牌/退市：保留真实状态，不合成虚假价格；
- 低、中、高成本场景同时记录，生产主指标使用冻结基准场景。

今天策略是观察信号，目标仓位、费用、滑点和冲击均为 0。

验证与晋级统一使用扣除冻结成本场景后的 `primary_return_net`；原始收益只用于审计，不作为生产晋级指标。

## 8. DeepSeek 的唯一生产定位

DeepSeek 是点时证据结构化器，不是荐股器。它只能处理本地宽候选，不能新增股票，不能输出生产 `llm_score`、主观上涨概率、目标价或买卖动作。

### 8.1 输入

每个请求包含：

- 策略、Prompt 版本、`snapshot_id` 和 `cutoff_at`；
- 股票代码、名称、市场和行业；
- 截止时间前可见的公告/新闻 `evidence_id`、标题、来源和发布时间；
- 必要的量价上下文与已确认风险事实。

输入不包含本地最终排名、是否进入 Top5、未来结果或人工预期答案，以免模型复制本地结论。

### 8.2 五维结构化研究

三个策略共享同一份点时证据缓存，但按各自周期解释以下五类信息：

1. 价值与质量：估值、盈利质量、资本回报、负债结构和经营稳定性。
2. 财务健康：盈利趋势、毛利和费用变化、经营现金流、自由现金流及利润与现金流背离。
3. 市场与资金：近一个月趋势、主力资金方向、成交额、量比、换手率、委比及尾盘承接是否健康。
4. 行业与政策：行业景气、竞争位置、成长空间、公司发展方向与可验证政策支持。
5. 综合风险：减持、解禁、监管、诉讼、业绩、估值透支、拥挤交易和可能造成价格波动的因素。

缺失数据必须显式标记，不能由 DeepSeek 猜测补齐。五维结果只形成结构化 Meta 特征；在 G3 前不直接加减生产分、不否决候选，也不能新增股票。

### 8.3 三套 Prompt

- 今天：事件能否影响信号后至当天收盘；
- 明日：次日催化、隔夜风险与当日是否已经定价；
- 2–5 日：催化持续性、题材真实性和数日事件风险。

三套 Prompt 有独立版本，任何实质修改都必须重启对应挑战样本。

### 8.4 严格输出

每只股票必须返回：

```json
{
  "code": "600000",
  "event_type": "业绩",
  "event_direction": 1,
  "event_strength": 76,
  "event_reliability": 88,
  "novelty": 62,
  "priced_in": 35,
  "time_horizon": "next_day",
  "overnight_risk": 24,
  "regulatory_risk": 8,
  "theme_truth": 81,
  "uncertainty": 20,
  "abstain": false,
  "evidence_ids": ["evidence_123"],
  "risk_flags": [],
  "reason": "事件与明日周期匹配"
}
```

本地校验要求：

- 所有必填字段必须存在；
- 数值必须在约定范围内；
- 代码必须来自输入候选池且不能重复；
- `evidence_ids` 必须是该股票输入证据的子集；
- 无证据或无法判断时必须 `abstain=true`；
- 未知代码、越界证据、非法数组和缺字段结果全部拒收；
- `reason` 只用于展示，不作为训练特征。

### 8.5 缓存与失败降级

缓存身份由策略、Prompt、模型、特征 Schema、代码、证据哈希和确定性风险事实组成。证据、Prompt、模型或 Schema 任一变化都会自动失效并重新分析。

以下情况全部保持本地结果不变：

- API Key 缺失或功能关闭；
- 超时、限流、余额不足或服务异常；
- 14:48 硬截止已到，或结果晚于对应策略允许使用的点时边界；
- 非法 JSON 或 Schema 校验失败；
- 无证据、证据过期或结果迟到。

生产推荐函数中没有 DeepSeek HTTP 调用。旧单策略 rerank、批量 rerank、市场观点、个股同步复核及其兼容入口均已删除；DeepSeek 只通过外部预计算任务写入点时结构化特征。

## 9. DeepSeek Meta 影子模型

三种策略独立训练，不能共享标签或模型。输入为本地分、本地量价上下文和 DeepSeek 结构化字段；`abstain` 和缺失本身也是特征。

影子输出：

```text
shadow_probability
shadow_expected_return_increment_pct
shadow_rank_key
```

当前这些字段只用于研究：

- 不改 `score`；
- 不改正式 rank；
- 不过滤候选；
- 不覆盖已冻结推荐；
- `DEEPSEEK_META_PRODUCTION_ENABLED` 硬编码为 false。

训练使用冻结的完整合格候选池，而不是只使用当天正式 Top5。未入选候选也按冻结执行策略计算可实现结果；涨跌停未成交按现金零收益处理，结果未成熟或无法复现的日期不进入训练。

训练使用按日期扩展窗口，测试日永远晚于训练截止日。最初训练窗口不计入 60/120 个真实 OOS 交易日。缺失 DeepSeek 特征的候选保留在同池中并显式标记 `deepseek_missing`，避免只统计 API 成功样本造成选择偏差。

同日同时生成三组 Top K：通过当天正式门控的生产本地推荐、本地字段消融 Meta、完整 DeepSeek Meta。备选观察不是生产持仓，主池为 0 时生产对照为现金。G2 的 DeepSeek 纯增量使用“完整 Meta - 本地字段消融 Meta”，避免把本地分的二次校准收益错误归因给 DeepSeek；最大回撤仍与生产本地 Top K 比较。增量收益使用移动块自助法计算 95% 置信区间。

## 10. 存储与点时约束

### `deepseek_analysis_batches`

记录策略、快照、证据截止、完成时间、Prompt/Schema/模型版本、请求响应哈希、Token、延迟、状态和错误。API Key 不入库。

### `deepseek_candidate_features`

记录候选代码、结构化字段、证据 ID/哈希、有效期、弃权状态和校验结果。推荐查询必须同时满足：

```text
feature.cutoff_at <= T 14:30
feature.completed_at <= T 14:30
feature.valid = true
prompt_version匹配
feature_schema_version匹配
model_name匹配
未过期
```

以上是当前正式推荐可读取的保守边界。14:30–14:48 按需完成的特征只进入影子研究和次日样本，不得在 G3 前改写当天正式排名。

### `deepseek_counterfactual_outcomes`

记录同日同池的生产本地 Top K、本地字段消融 Top K、DeepSeek 挑战 Top K、替换股票、各组净收益、DeepSeek 纯增量、相对生产增量和回撤。它不能修改原始推荐信号。

## 11. 真实验证与晋级

### G0 工程门槛

- 推荐链无外部 API 等待；
- 点时 Schema 和证据边界可审计；
- API 失败与纯本地基线一致；
- 标签、成交和成本可复现。

### G1 影子样本

- 至少 60 个真实样本外交易日；
- 明日策略至少 300 个可评价样本；
- DeepSeek 有效覆盖率至少 80%；
- 至少覆盖两类市场状态。

### G2 实验门槛

- 同池 Top5 的累计增量净收益为正；
- 移动块自助 95% 置信区间下界大于 0；
- 最大回撤没有显著恶化；
- 正增量至少覆盖两个行业和两种市场状态，单一行业贡献不超过 70%，单一市场状态贡献不超过 80%。

### G3 有限挑战

即使未来开放，也只能每天最多替换 1 只、移动最多 2 位，且 DeepSeek 不能单独否决。当前代码不开放 G3 生产接管。

### G4 正式 Meta

- 至少 120 个真实样本外交易日；
- 扣除交易成本后增量持续为正；
- Prompt、模型、特征 Schema 和标签全部冻结；
- 重新发布生产基线并人工确认。

截至当前工程版本，G0 的代码路径已建立；G1–G4 必须依靠未来真实交易日积累，不能通过回测、补数据或文档勾选伪造完成。

## 12. 后台任务

预计算：

```bash
python -m stock_analyzer.jobs deepseek-precompute --strategy all --market all
```

生产环境默认由项目进程内调度器在 09:40–14:20 约每 20 分钟执行变化检查，午休不调用；不安装 systemd timer。命令仍可用于手动运维或故障补算。任务只对新候选或证据哈希变化执行 API 请求；缓存命中和无证据弃权不消耗调用次数。14:30–14:48 不设固定调用点，可按需执行同一命令；14:48 为 API 硬截止，14:50 冻结最终推荐。所有策略和模型共享每交易日 50 次调用上限。

快照冻结：

```bash
python -m stock_analyzer.jobs snapshot --strategy all --market all
```

影子 Meta 与反事实：

```bash
python -m stock_analyzer.jobs deepseek-meta-build --strategy all --min-train-days 60 --top-k 5
```

不足真实日数时命令返回 `insufficient_real_oos_days`，不会生成可用于生产的结论。

## 13. 页面语义

- 展示今天、明日、2–5 日三个策略，不再展示额外荐股策略。
- `score` 明确标注为综合排序分，不显示成上涨概率。
- DeepSeek 只显示事件类型、证据、风险、弃权和预计算状态。
- 不显示未经校准的 `horizon_up_score`、`llm_score` 或“AI 上涨概率”。
- `local_only` 表示当天没有截止前有效 DeepSeek 特征，不是错误，正式排序仍由本地基线给出。
- 个股预测接口不再因 `deepseek=1` 同步请求模型。

## 14. 维护规则

- 策略目标、信号时间、入场、退出、主收益字段、成本或阈值变化，必须发布新版本并同步本文档。
- 新旧策略版本、Prompt 版本和特征 Schema 的指标不能混合。
- 页面、快照、验证、OOS 和 Meta 必须使用相同主收益字段。
- 任何模型改进都先影子、再真实 OOS、再人工晋级，不允许自动修改生产权重。
- 评估至少报告净收益、真实日胜率、最大回撤、覆盖交易日、样本数、unfilled 和数据未知数。
- 连续两个完整 OOS 窗口没有正增量的机制应从生产接线中移除。
- 所有策略都未通过健康门控时，正确结果是无推荐和保留现金。
