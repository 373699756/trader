# 荐股策略详细说明

> **重要免责声明**
>
> 本项目所有策略都不能保证盈利，也无法保证盈利。当前实现是本地启发式量化打分、风控诊断和验证闭环，不是训练好的收益预测模型，也不是确定性价格预测。
>
> 回测和回放样本不等于实盘。真实交易会受到 T+1、涨跌停不可成交、滑点、冲击成本、停牌、财报雷、监管变化、因子拥挤和行情源缺失影响。系统的价值是把候选股票、风险证据和验证结果放到同一个流程里，让你用样本判断策略是否值得继续观察。

## 1. 当前策略总览

| 策略 | 入口/名称 | 目标周期 | 主评估周期 | 当前定位 |
|---|---|---:|---:|---|
| 短期推荐 | `short_term` | 盘中到次日 | 次日 | 盘中强势、动量、量价、人气和舆情 |
| 长期推荐 | `long_term` | 数周到数月 | 20 日 | 技术趋势版中期趋势，基本面默认关闭可选启用 |
| 明天预测 | `tomorrow_picks` | 次日 | 次日 | 15:00 后筛次日可能冲高且仍可买的票 |
| 波段 5-10 日 | `swing_picks` | 5-10 日 | 10 日 | 短周期趋势延续、温和放量、不过热 |
| 中长期 1-3 月 | `position_picks` | 1-3 月 | 20 日 | 中期趋势、质量/估值可选因子、主题和流动性 |
| 科技潜力 | `tech_potential` | 主题潜力 | 20 日 | 科技主题 + 早期趋势 + 未透支 |
| 卡脖子 | `chokepoint_picks` | 供应链上游 | 20 日 | Serenity chokepoint 供应链瓶颈方法论 |
| 反转低波 | `reversal_picks` | 1-2 周 | 5 日 | 超跌反转、低波、换手不过热 |
| 小市值价值 | `smallcap_value_picks` | 1-3 月 | 20 日 | 小流通市值 + 低 PE/PB + 流动性护栏 |
| 量价突破 | `breakout_picks` | 5-10 日 | 10 日 | 均线多头或 20 日新高 + 放量突破 |
| 单票预测 | 首页输入股票代码 | 短期 + 长期 | 不单独入库 | 对指定股票做策略命中和风控诊断 |

首页“多策略共识”当前聚合 6 个实时主策略：`short_term`、`long_term`、`tomorrow_picks`、`swing_picks`、`position_picks`、`tech_potential`，并保留 Top 30。卡脖子、反转低波、小市值价值、量价突破是独立策略和验证对象，其中卡脖子也参与单票长期预测。

## 2. 数据刷新与基础股票池

系统默认 `REFRESH_SECONDS=30`，接口缓存按 30 秒刷新实时行情。行情不是交易所直连实盘盘口，而是第三方公开数据源，可能延迟、缺字段或短时失败。

所有正向推荐先通过 `prepare_candidates()` 基础过滤：

| 过滤项 | 当前规则 |
|---|---|
| 股票代码 | 仅支持常见 A 股前缀：主板、创业板、科创板 |
| 名称风险 | 剔除 `ST`、退市标记 |
| 价格 | 现价必须大于 0 |
| 流动性 | 成交额必须不低于 `MIN_TURNOVER`，默认 5000 万 |
| 当日跌幅 | 当日跌幅必须大于 -8% |
| 当日涨幅 | 总体不超过 `MAX_RECOMMENDED_GAIN`，默认 12% |
| 可买性 | 主板默认不超过 6.5%，创业板/科创板默认不超过 10% |
| 极端状态 | 剔除高位一字板、近涨停且不可交易风险明显的票 |

如果用户在首页输入的股票不在当前可推荐股票池内，系统不会再报“无法预测”并停止，而是进入风险诊断模式：仍返回短期和长期结论，但默认从风险角度解释原因，例如 ST/退市、停牌、无有效现价、成交额不足、当日跌幅过大、涨幅透支、实时行情和历史行情均不可用。

## 3. 市况自适应权重

`build_market_regime()` 用当前候选池计算盘面环境：

| 指标 | 含义 |
|---|---|
| `breadth_pct` | 上涨股票占比 |
| `strong_pct` | 涨幅 ≥ 3% 的强势股占比 |
| `weak_pct` | 跌幅 ≤ -3% 的弱势股占比 |
| `median_pct_chg` | 候选池中位涨跌幅 |
| `avg_amplitude` | 平均振幅，过高代表分歧和追高风险 |

市况分数 `score` 达到 68 以上为偏进攻，42 以下为偏防守，中间为均衡震荡。

| 市况 | 权重变化 | 策略含义 |
|---|---|---|
| 偏进攻 `risk_on` | 放大动量、趋势、突破、量能，压低低波和质量保守项 | 可以看强势延续，但仍要防止高位追涨 |
| 偏防守 `risk_off` | 压低动量、突破、量能，放大低波、质量、流动性 | 优先低波、不过热和买入安全，不因高分追高 |
| 均衡震荡 `balanced` | 轻微压缩动量/突破，略偏低波和质量 | 防假突破，优先多策略共识和验证表现 |

市况调整不是展示标签，而是直接进入各策略最终分数：`_regime_component()` 会以 50 分为中性点放大或压缩因子的边际优势，`_market_regime_adjustment()` 还会按策略类型给顺风/逆风加减分。

各策略上线分数已经集中到 `STRATEGY_COMBINERS` 和 `_DEFAULT_WEIGHTS`：每个策略先计算组件分，再由 `_combine_details()` 统一按声明式权重、市场状态、风险惩罚和 `overheat_damp` 合成最终分。`.runtime/weights.json` 可以覆盖这些上线权重，不再只覆盖 AlphaLite 回测权重。

超涨/高位风险现在收敛为单一硬门控：`overheat_damp`。累计涨幅、YTD、20 日涨幅不再同时在风险惩罚里重复扣分；风险惩罚只负责盘中追高、量比/换手过热、振幅/波动、流动性和负面舆情等更独立的风险。

## 4. 主工作流：强共识 + Agent 风控优先

默认推荐使用“强共识 + Agent 风控优先”，而不是单独追某个榜单最高分。

执行流程：

1. 先看市场状态。如果市况偏防守，所有短线、突破、题材追涨策略都要降级观察。
2. 看“多策略共识”。同一股票至少被 2 个策略同时选中，才有更强的横向确认。
3. 看 Serenity Profile。优先 `quality_score` 高、`risk_score` 低、`confidence_score` 不低的数据。
4. 看 TradingAgents 委员会。避开 `风控否决`，优先 `组合经理批准` 或 `交易员小仓试单`。
5. 看策略验证。真实前瞻样本优先于历史回放样本，主周期净胜率和净收益要扣成本后再判断。

多策略共识合并时会记录：

| 字段 | 解释 |
|---|---|
| `appearances` | 进入了多少个策略榜 |
| `avg_score` | 多策略平均裸分 |
| `avg_quality` | Serenity 平均质量分 |
| `avg_risk` | Serenity 平均风险分 |
| `avg_agent_score` | Agent 委员会平均最终分 |
| `consensus_score` | 综合共识分 |
| `actions` | Serenity/Agent 动作建议 |

共识分会用两个机制防止误判：

| 机制 | 作用 |
|---|---|
| 共识极化拉伸 | 策略一致时把分数向两端拉开，策略分歧时向 50 压缩 |
| 策略可信度乘子 | 用策略验证的主周期净胜率/净收益调整权重；真实样本不足时只轻度参考回放；真实样本衰减触发后可降权或置零 |

组合约束不新增选股逻辑，只把最近一次保存快照转换成建议仓位；当前 Web 入口暂时隐藏，能力保留在后端：

| 约束 | 默认值 | 含义 |
|---|---:|---|
| 最大持仓数 | `PORTFOLIO_MAX_POSITIONS=10` | 只取该策略最近保存快照的前 N 只 |
| 单票上限 | `PORTFOLIO_SINGLE_CAP=0.15` | 单只股票建议仓位不超过 15% |
| 主题/行业上限 | `PORTFOLIO_THEME_CAP=0.35` | 同主题或行业暴露不超过 35% |
| 权重基础 | 质量/置信度 ÷ 风险 ÷ 波动代理 | 高置信、低风险、低波动给更高基础权重 |
| 不可行处理 | 保留现金 | 如果候选太少或主题过于集中，不强行把剩余权重摊回去 |
| 纸面绩效 | `paper_trades` / `portfolio_nav` | 记录入场、退出、净收益、组合净值、最大回撤和现金占比 |
| 总仓缩放 | 市况因子 × 回撤因子 | 偏防守市和纸面回撤扩大时自动保留更多现金 |

因此组合页解决的是“买多少、是否过度集中、纸面组合是否真赚钱”的问题，不解决“必然上涨”的问题。纸面绩效通过 `/api/portfolio/performance` 和 `/api/paper-trades` 展示，仍然不是实盘成交记录。

总仓控制字段：

| 字段 | 解释 |
|---|---|
| `gross_exposure_pct` | 建议总股票仓位，权重合计不再固定为 100% |
| `cash_pct` | 因市况、防回撤或约束不可行而保留的现金比例 |
| `regime_factor` | 市况因子，偏进攻接近 1，均衡约 0.7，偏防守约 0.4 |
| `drawdown_factor` | 纸面组合回撤因子，超过 `PORTFOLIO_DD_LEVEL_1/2` 后阶梯降仓 |
| `gross_reasons` | 触发总仓调整的文字原因 |

默认总仓参数：`PORTFOLIO_GROSS_RISK_ON=1.0`、`PORTFOLIO_GROSS_BALANCED=0.7`、`PORTFOLIO_GROSS_RISK_OFF=0.4`；回撤超过 8% 默认乘 0.7，超过 15% 默认乘 0.4。该机制是防守仓位建议，不是择时保证。

事件风险层默认关闭，开启 `ENABLE_EVENT_RISK=1` 后生效：

| 风险 | 数据来源 | 处理 |
|---|---|---|
| 限售解禁 | AKShare/Tushare 可用接口 | 未来窗口内解禁比例越高，`event_risk_penalty` 越高 |
| 股权质押 | Tushare `pledge_stat` 优先 | 大股东质押比例过高会提高风险分 |
| 减持计划 | AKShare/Cninfo 可用接口 | 命中减持公告后标记事件风险 |
| 财报窗口 | 业绩披露/预告日历 | 披露日前后短窗口标记不确定性风险 |

事件风险进入三层：行级 `event_risk` 解释、Serenity `risk_score` 加分、Agent 风控理由。若开启 `EVENT_RISK_HARD_FILTER=1`，高危票可在候选阶段直接剔除。减持公告默认只看近 `EVENT_RISK_REDUCTION_LOOKBACK_DAYS=120` 天，解禁风险需要解禁比例字段。数据源失败或未开启时该层 no-op，并在 `/api/health` 标注状态。

长期黑名单硬过滤默认开启，用于处理“历史财务造假、重大违法、重大处罚、审计无法表示意见、重大内控缺陷、长期严重负面信息”等不应进入推荐池的股票。维护文件为 `.runtime/risk_blacklist.json` 或 `.runtime/risk_blacklist.csv`，模板见 `docs/risk_blacklist.example.json` 和 `docs/risk_blacklist.example.csv`。该层与短期事件风险分离：事件风险处理未来/近期窗口，长期黑名单处理历史重大负面记录。

| 字段 | 含义 |
|---|---|
| `code` | 6 位股票代码 |
| `name` | 股票简称，辅助人工核对 |
| `level` | `critical` / `high` 默认硬过滤，`medium` 默认只加风险 |
| `category` | `financial_fraud`、`audit_opinion`、`major_penalty`、`negative_history` 等 |
| `reason` | 风险原因，会展示为“黑名单风险” |
| `source` | 证监会处罚、交易所纪律处分、公司公告、审计报告或可信新闻链接 |
| `hard_exclude` | 是否强制剔除 |
| `expires_at` | 可选，到期后自动忽略 |

该黑名单层进入三层：候选池硬过滤、单票预测风险诊断、Serenity/Agent 风控扣分。即使关闭 `RISK_BLACKLIST_HARD_FILTER`，命中项仍会显著提高风险分并在解释中展示。

基本面与因子 IC 框架默认关闭，开启 `ENABLE_FUNDAMENTALS=1` 后生效：

| 因子 | 当前字段 |
|---|---|
| 质量 | ROE、毛利率、资产负债率 |
| 价值 | PE、PB，缺失时回到中性分 |
| 业绩超预期 | 业绩预告/快报或净利润增速代理 |
| 评级调整 | 分析师/盈利预测调整字段，缺失时中性 |

基本面数据按日缓存到 `.runtime/fundamentals.json`，抓取失败安全降级，不阻断实时推荐。若基本面字段全空或全 0，质量/业绩/评级分会回到中性 50，并标记 `fundamental_degraded=True`，避免缺失数据被误算成高分。`daily_job --update` 后会自动刷新 `.runtime/factor_ic.json`；也可单独执行 `python -m stock_analyzer.daily_job --factor-ic --strategy all`。IC 用真实前瞻样本的因子值与主周期净收益计算 Spearman 相关，用于识别长期无效或可能有 alpha 的因子。

默认情况下 IC 只展示、不改变打分。显式开启 `ENABLE_FACTOR_IC_WEIGHTING=1` 后，`_combine_details()` 会读取 `.runtime/factor_ic.json`：样本数达到 `FACTOR_IC_MIN_SAMPLES` 的正 IC 因子轻微增权、负 IC 因子轻微降权，调整幅度由 `FACTOR_IC_WEIGHT_BAND` 限制并保持总权重归一。样本不足时不能开启该功能作为调仓依据。

## 5. TradingAgents 与 Serenity 映射

本项目没有复制或运行 `TauricResearch/TradingAgents` 的代码，也没有默认接入 LLM。当前是确定性本地映射：

| 角色 | 当前输入 |
|---|---|
| 技术分析师 | 动量、趋势、均线偏离、突破、买入安全 |
| 情绪分析师 | 舆情分、风险关键词、热度 |
| 基本面代理/增强 | 默认使用主题、行业、流动性、不过热、质量代理；开启基本面后加入质量、估值、业绩超预期和评级调整 |
| 新闻/市场环境 | `market_regime` 和顺风/逆风分 |
| 牛方研究员 | 汇总技术、主题、情绪、流动性和市场顺风证据 |
| 熊方研究员 | 汇总追高、透支、负面舆情、波动和流动性风险 |
| 交易员 | 牛方分、熊方风险、市场环境转换成可执行分 |
| 风控经理 | 对高量比、高换手、高振幅、当日追高做风险压分；阶段涨幅透支由 `overheat_damp` 统一硬门控 |
| 组合经理 | 输出 `final_score`、`stance` 和 `final_action_label` |

Serenity Profile 负责把每一行策略结果转成可解释的质量、风险、置信度和动作标签：

| 字段 | 解释 |
|---|---|
| `quality_score` | 裸分、因子均值、置信度、Agent 分和风险扣分合成 |
| `risk_score` | 追高、盘中波动、策略风险惩罚、逆风市况和 Agent 风险合成；阶段涨幅透支来自 `overheat_damp` |
| `confidence_score` | 有效因子数量、数据覆盖、顺风市况和 Agent 分共同决定 |
| `data_coverage` | 历史因子覆盖率；低于阈值时高评级会被强制降级 |
| `action_label` | 优先跟踪、小仓观察、等待确认、只观察 |

Verdict 分档：

| 分档 | 大致含义 |
|---|---|
| `strong_buy` / 强烈关注 | 分高且风险不高，只能理解为重点观察 |
| `buy` / 关注 | 有正向信号，但仍需人工复核 |
| `watch` / 观察 | 有部分信号或数据不足 |
| `reduce` / 谨慎 | 风险或分歧偏高 |
| `avoid` / 回避 | 分数低或风控证据明显 |

## 6. 策略逐条说明

### 6.1 短期推荐 `short_term`

目标是发现盘中强势、次日仍可能延续的股票。

| 维度 | 当前实现 |
|---|---|
| 周期 | 盘中到次日 |
| 核心因子 | 当日涨幅、涨速、量比、3/5 日收益、5 日成交额变化、20 日突破、换手、成交额、行业强度、热榜、舆情 |
| 默认权重 | 动量 55%、流动性 15%、行业 8%、热度 7%、舆情 15% |
| 额外风控 | 负面舆情、近涨停低承接、`reversal_tilt` 反转修正会进入 `risk_penalty_parts`；超涨统一由 `overheat_damp` 抑制 |
| 适合市况 | 偏进攻或有主线的强势市 |
| 主要风险 | 高位接力失败、冲高回落、热点退潮、免费舆情缺失 |
| 验证口径 | 次日净收益/净胜率 |

`reversal_tilt` 默认关闭。若离线校准发现 A 股短线反转优于动量，可以通过 `.runtime/weights.json` 打开，让近期涨幅越高的票被额外减分。

### 6.2 长期推荐 `long_term`

目标是寻找中期趋势稳定、流动性足、风险不过热的股票。

| 维度 | 当前实现 |
|---|---|
| 周期 | 数周到数月 |
| 核心因子 | 20 日收益、60 日涨幅、YTD 涨幅、MA20 偏离、10 日收益、5 日成交额变化、20 日波动率、行业强度、舆情 |
| 默认权重 | 趋势 42%、流动性 20%、行业 13%、舆情 13%、动量 7%、热度 5% |
| 额外风控 | 负面舆情、MA20 偏离过远、高波动、低成交额扣分；累计超涨统一由 `overheat_damp` 抑制 |
| 适合市况 | 趋势较稳或防守市中的稳健趋势股 |
| 主要风险 | 基本面默认关闭且数据可能缺失，不能把技术趋势直接当基本面价值策略 |
| 验证口径 | 20 日主周期净收益/净胜率 |

### 6.3 明天预测 `tomorrow_picks`

目标是 15:00 后筛选次日可能延续、但仍有可买性的股票。

| 维度 | 当前实现 |
|---|---|
| 周期 | 次日 |
| 核心因子 | 成交额、换手率、温和涨幅、涨速、量比、60 日涨幅、YTD、振幅、买入安全、收盘结构 |
| 默认权重 | 流动性 24%、动能 22%、趋势 16%、买入安全 18%、收盘结构 20% |
| 买入安全 | 当日涨幅越接近可买上限，买入安全分越低；下跌票买入安全分低 |
| 风险扣分 | 高涨幅、高振幅、高换手、高量比、收盘回落、日内透支、末段急拉/急跌；YTD/阶段涨幅由 `overheat_damp` 统一处理 |
| 适合市况 | 下午资金承接强、有板块扩散的市场 |
| 主要风险 | 次日低开、冲高回落、收盘假拉升、涨停附近无法成交 |
| 验证口径 | 次日开盘入场到开盘/最高/最低/收盘的对比，主看次日开盘至收盘净收益/净胜率 |

### 6.4 波段 5-10 日 `swing_picks`

目标是抓短周期趋势延续，不追极端过热。

| 维度 | 当前实现 |
|---|---|
| 周期 | 5-10 日 |
| 前置过滤 | 当日涨幅 ≤ 8%，60 日涨幅 ≤ 85%，YTD ≤ 130%，60 日涨幅 ≥ -18% |
| 核心因子 | 5/10/20 日收益、MA5/MA20 偏离、5 日成交额变化、20 日突破、20 日波动率、成交额、换手、量比 |
| 默认权重 | 动量 34%、趋势 26%、流动性 20%、买入安全 12%、不过热 8% |
| 风险扣分 | 当日涨幅过高、量比过热、换手过热、高波动、MA5 偏离过远；60 日/YTD 透支由 `overheat_damp` 统一处理 |
| 适合市况 | 趋势市或板块主线延续 |
| 主要风险 | 震荡市假突破、追高后横盘、回撤触发止损 |
| 验证口径 | 10 日主周期净收益/净胜率 |

### 6.5 中长期 1-3 月 `position_picks`

目标是中期趋势和质量代理更均衡的候选。

| 维度 | 当前实现 |
|---|---|
| 周期 | 1-3 月 |
| 前置过滤 | 当日涨幅 ≤ 6%，60 日涨幅 ≤ 75%，YTD ≤ 120%，60 日涨幅 ≥ -25% |
| 核心因子 | 60 日/YTD 趋势、20/10 日收益、MA20 偏离、20 日低波、5 日成交额变化、科技/先进制造主题 |
| 默认权重 | 趋势 34%、质量代理 26%、流动性 20%、主题 12%、买入安全 8% |
| 质量代理 | 默认用不过热、低波、温和量能、成交额变化；开启基本面后混入质量/估值/业绩/评级分 |
| 风险扣分 | 当日涨幅过高、高振幅/高波动、低成交额；阶段涨幅透支由 `overheat_damp` 统一处理 |
| 适合市况 | 中期趋势修复或结构性行情 |
| 主要风险 | 基本面数据默认关闭或可能缺失，技术趋势仍可能被财报和业绩证伪 |
| 验证口径 | 20 日主周期净收益/净胜率 |

### 6.6 科技潜力 `tech_potential`

目标是找前沿科技方向中尚未明显透支、仍具流动性和早期趋势的股票。

| 维度 | 当前实现 |
|---|---|
| 周期 | 主题潜力，通常按 20 日验证 |
| 前置过滤 | 60 日涨幅 ≤ 90%，YTD ≤ 150%，60 日涨幅 ≥ -25% |
| 主题匹配 | AI/算力、半导体、机器人/智能制造、低空/商业航天、智能汽车/车联网、新材料/高端电子、脑机/医疗科技；创业板/科创板或宽科技关键词可归入泛科技 |
| 默认权重 | 主题 24%、卡脖子 10%、流动性 18%、早期趋势 20%、不过热 14%、量能 8%、买入安全 6% |
| 风险扣分 | 复用明天预测的盘中追高/量价过热风险，并额外惩罚当日涨幅过高；60 日/YTD 透支由 `overheat_damp` 统一处理 |
| 适合市况 | 科技主线扩散、风险偏好改善、主题从龙头向上游扩散 |
| 主要风险 | 主题识别依赖名称和行业字段，可能错过隐性业务或误判概念股 |
| 验证口径 | 20 日主周期净收益/净胜率 |

### 6.7 卡脖子 `chokepoint_picks`

| 维度 | 当前实现 |
|---|---|

| 入选条件 | 必须命中卡脖子关键词，否则不进入榜单 |

| 卡脖子分 | 命中上游关键词给基础加分；60 日涨幅在 -5% 到 25% 额外加分，涨幅过大扣分 |
| 适合市况 | 科技产业链补短板、国产替代、供给约束被市场重新定价 |
| 主要风险 | 关键词匹配可能误判，主题兑现慢，业绩兑现和订单验证缺失 |

### 6.8 反转低波 `reversal_picks`

目标是寻找“跌过但没坏、波动不高、换手不躁动”的短线均值回复机会。

| 维度 | 当前实现 |
|---|---|
| 周期 | 1-2 周 |
| 核心因子 | `oversold_calm_score`（20 日反转、20 日低波、不过热复合）、换手率低、成交额 |
| 默认权重 | 超跌冷静复合 50%、换手不过热 22%、流动性 28% |
| 历史因子缺失 | 20 日收益缺失时回退 60 日涨幅；低波缺失时回退当日振幅 |
| 风险扣分 | 20 日极端下跌、放量急杀、当日继续大跌等接飞刀信号 |
| 适合市况 | 震荡市、超跌修复市、杀估值后企稳阶段 |
| 主要风险 | 基本面暴雷、趋势性下跌、流动性继续收缩 |
| 验证口径 | 5 日主周期净收益/净胜率 |

### 6.9 小市值价值 `smallcap_value_picks`

目标是捕捉小流通市值和低估值风格，但用多重护栏过滤尾部风险。

| 维度 | 当前实现 |
|---|---|
| 周期 | 1-3 月 |
| 入选条件 | 必须有 `float_market_cap`、`pe_dynamic`、`pb`；PE/PB 必须大于 0 |
| 市值下限 | `SMALLCAP_MIN_FLOAT_CAP` 默认 8 亿，过滤纯壳和退市风险 |
| 默认权重 | 小市值 34%、价值 22%、流动性 20%、稳定复合 24%（低波 + 不过热） |
| 风险扣分 | 使用中长期风险扣分：高波动、低成交额、盘中追高；涨幅透支由 `overheat_damp` 统一处理 |
| 适合市况 | 小盘风格占优、流动性宽松、风险偏好改善 |
| 主要风险 | 微盘股流动性踩踏、壳风险、退市风险、风格切换剧烈回撤 |
| 验证口径 | 20 日主周期净收益/净胜率 |

如果当前行情源不提供市值/估值字段，该策略会返回空榜并在 `meta.note` 标注原因。这不是策略失效，而是数据字段不足。

### 6.10 量价突破 `breakout_picks`

目标是找趋势确认型标的：均线多头排列或 20 日新高，同时量能突破。

| 维度 | 当前实现 |
|---|---|
| 周期 | 5-10 日 |
| 核心因子 | 当日涨幅、涨速、60 日趋势、MA20 偏离、MA 多头、20 日新高、5 日均量突破、买入安全 |
| 默认权重 | 动量 30%、突破强度 26%、量能突破 18%、趋势 16%、买入安全 10% |
| 历史因子模式 | 有 MA/20 日新高/5 日均量时，优先使用多头排列或 20 日新高 |
| 实时兜底模式 | 历史因子缺失或单票覆盖不完整时，用实时涨幅、涨速、量比、成交额和 60 日趋势做保守代理 |
| 性能处理 | 前台默认只读历史缓存，不在页面请求里批量远程拉 K 线，避免卡顿；远程补历史交给后台任务 |
| 风险扣分 | 复用明天预测风险，并做过热乘法抑制 |
| 适合市况 | 趋势市、主线加速、板块放量扩散 |
| 主要风险 | 震荡市假突破、次日高开低走、量能不可持续 |
| 验证口径 | 10 日主周期净收益/净胜率 |

空榜常见原因：实时代理条件太严格、候选池当天没有真正放量突破、涨幅接近风控上限被过滤。历史因子覆盖不足时不会直接空榜，而是允许强实时量价票以“实时兜底/代理”进入。

## 7. 单票短期/长期预测

首页输入股票代码后，系统会调用 `build_stock_prediction()`，同时输出短期和长期倾向。

| 预测维度 | 使用策略 |
|---|---|
| 短期 | `short_term`、`tomorrow_picks`、`swing_picks` |
| 长期 | `long_term`、`position_picks`、`tech_potential`、`chokepoint_picks` |

如果股票进入候选池：

| 计算项 | 解释 |
|---|---|
| `strategy_hits` | 命中了哪些策略、排名、分数、动作和风险 |
| `missed_strategies` | 没命中的策略和原因 |
| `consensus` | 同一周期内部的策略共识 |
| `prediction` | 根据命中策略方向分、平均风险、共识次数和市况给出偏涨、震荡、偏弱 |

如果股票被过滤或行情缺失：

| 数据情况 | 行为 |
|---|---|
| 实时行情缺失但历史行情可用 | 使用历史行情兜底，并提示实时交易状态不可确认 |
| 实时和历史均不可用 | 返回高风险诊断，不给正向预测 |
| 股票不在基础股票池 | 仍输出短期/长期，但结论通常是高风险、偏弱或长期缺少正向证据 |

这类结果不是“预测失败”，而是风控判断：数据不足或交易状态不可确认时，系统默认把它当风险票处理。

## 8. 策略验证口径

策略验证不是简单看次日涨跌，而是按策略目标周期评估：

| 策略 | 主周期 |
|---|---:|
| `short_term`、`tomorrow_picks` | 次日 |
| `reversal_picks` | 5 日 |
| `swing_picks`、`breakout_picks` | 10 日 |
| `long_term`、`position_picks`、`tech_potential`、`chokepoint_picks`、`smallcap_value_picks` | 20 日 |

当前明天预测验证以次日开盘作为执行口径，对比次日开盘、最高、最低和收盘，并以真实前瞻样本的扣成本次日收盘收益（即开盘后收益）作为主复盘口径。历史回放样本只显示为参考，不进入主记分牌、净胜率走势、次日对比和自动迭代。

验证会扣 `VALIDATION_TRADE_COST_PCT` 固定成本，默认 0.25%；再按成交额分档追加滑点，使小市值/低成交额样本的净收益更保守。同时模拟退出规则：

| 规则 | 默认值 | 说明 |
|---|---:|---|
| 固定止损 | 5% | 窗口内最低价触发则按止损价退出 |
| 止盈 | 8% | 窗口内最高价触发则按止盈价退出 |
| 移动止损 | 4% | 入场后最高价回撤触发退出 |
| 到期退出 | 持有到主周期最后一天 | 未触发止损/止盈/移动止损时使用 |

日线验证无法精确还原盘中先后顺序。如果同一天同时触发止损和止盈，系统按保守口径先算止损。当前加入两条可成交性近似：一字涨停/封板买不进的样本不计入结果；封跌停止损无法按止损价卖出时，退出顺延到下一交易日开盘。盘口容量和真实冲击成本仍可能让实盘差于验证结果。

流动性滑点默认分档：

| 成交额 | 额外滑点 |
|---|---:|
| ≥ 10 亿 | `VALIDATION_SLIPPAGE_HIGH_TURNOVER_PCT=0.05%` |
| 3 亿 - 10 亿 | `VALIDATION_SLIPPAGE_MID_TURNOVER_PCT=0.12%` |
| 1 亿 - 3 亿 | `VALIDATION_SLIPPAGE_LOW_TURNOVER_PCT=0.25%` |
| < 1 亿 | `VALIDATION_SLIPPAGE_MICRO_TURNOVER_PCT=0.45%` |

策略衰减退场：

| 条件 | 动作 |
|---|---|
| 真实样本数 < `STRATEGY_DECAY_MIN_REAL_SAMPLES=20` | `pending/probation`，避免样本不足误杀，但共识中保守降权 |
| 真实主周期净胜率 ≥ `STRATEGY_RETIRE_WINRATE=48%` 且净收益 ≥ 0 | 继续参与共识，表现一般时仍按 probation 降权 |
| 真实主周期净胜率 < 48%，或真实主周期净收益 < 0 | `retired`，该策略本轮共识权重置零 |

### 自动分批更新计划

历史 K 线下载和已保存样本结果回填不再依赖 Web 按钮。真实样本入库默认由应用服务在交易日 15:00 自动保存；`daily_job --snapshot` 可作为外部兜底，K 线预取和结果回填由应用后台分批任务与 `daily_job --update` 共同完成：

```bash
python -m stock_analyzer.daily_job --snapshot --strategy all --market all
python -m stock_analyzer.daily_job --update --strategy all
python -m stock_analyzer.daily_job --paper-trade --strategy all
```

| 阶段 | 处理方式 |
|---|---|
| 保存快照 | 服务运行时交易日 15:00 自动保存 `tomorrow_picks`；若 15:00 后才启动服务，当天会补保存一次 |
| 保存兜底 | 可保留 `daily_job --snapshot` cron；同一天同版本样本会替换旧样本，不会重复累计 |
| 启动延迟 | 默认启动后 1800 秒（约 30 分钟）执行首轮，避免阻塞首页行情和策略加载 |
| 策略扫描 | 默认遍历全部验证策略；可用 `VALIDATION_AUTO_UPDATE_STRATEGIES=tomorrow_picks,swing_picks` 限定 |
| 代码收集 | 每个策略从已保存信号里取最近代码，单轮最多 `VALIDATION_AUTO_UPDATE_MAX_CODES_PER_RUN=160` |
| 分批下载 | 每批 `VALIDATION_AUTO_UPDATE_BATCH_SIZE=40` 只股票，先查 `.runtime/history_cache.sqlite3`，缺失或过期再请求行情源 |
| 结果回填 | 每批历史预取完成后，只对该批代码调用 `update_outcomes(..., codes=batch)`，避免全量重复扫描 |
| 纸面交易 | 执行 `daily_job --paper-trade`，按最近快照建议权重、次日开盘近似入场、策略主周期退出并扣成本；默认按持有期摊分每日新开资本，避免重叠持仓收益被高估 |
| 运行频率 | 默认每 `VALIDATION_AUTO_UPDATE_INTERVAL_SECONDS=1800` 秒执行一轮 |
| 状态展示 | Web 调 `/api/strategy-validation/auto-update-status` 展示最近一轮批次数、缓存数、下载数、失败数和更新数 |
| 失败处理 | 单轮异常写入 `last_error`；下一轮继续，不阻断前台推荐 |

推荐 crontab：

```cron
00 15 * * 1-5 cd /home/cp/Public/trader && .venv/bin/python -m stock_analyzer.daily_job --snapshot --strategy all --market all >> .runtime/daily_job.log 2>&1
30 17 * * 1-5 cd /home/cp/Public/trader && .venv/bin/python -m stock_analyzer.daily_job --update --strategy all >> .runtime/daily_job.log 2>&1
30 18 * * 1-5 cd /home/cp/Public/trader && .venv/bin/python -m stock_analyzer.daily_job --update --paper-trade --strategy all >> .runtime/daily_job.log 2>&1
```

边界原则：

1. 自动任务只回填已经保存过的真实前瞻样本结果，不凭空制造过去推荐。
2. 历史回放补样本仍标记为 `*_replay_v1`，只能粗筛，不能替代真实前瞻样本。
3. 批大小和间隔必须保守，避免免费行情源限流导致整轮失败。
4. 权重校准只读取真实样本；回放样本不会写入上线权重。

## 9. 上线权重校准

上线 scorer、真实验证样本和权重覆盖已经闭合到同一套定义：`score_*_candidates()` 保存的 `raw_json` 会包含组件分、`risk_penalty_parts`、`regime_weight_profile`、`base_score`、`raw_score` 和 `overheat_damp`。校准时直接读取这些真实前瞻样本，用同一个 `_combine_details()` 重算 final 分并重新排序。

命令：

```bash
python -m stock_analyzer.calibrate --calibrate-live-weights tomorrow_picks --dry-run
python -m stock_analyzer.calibrate --calibrate-live-weights all --dry-run
```

写入条件：

| 条件 | 默认值 |
|---|---:|
| 真实样本数 | `CALIBRATE_MIN_SAMPLES=30` |
| 因子覆盖 | 样本平均 `data_coverage >= CALIBRATE_MIN_COVERAGE=0.5` |
| Walk-forward | `CALIBRATE_WALK_FORWARD_FOLDS=4` 个扩展窗 |
| OOS 目标函数改善 | OOS 均值改善 > `CALIBRATE_IMPROVE_MARGIN=0.05` |
| OOS fold 多数胜出 | 改进 fold 数必须超过半数 |
| 权重范围 | 单项 clamp 到合理区间后归一 |
| 样本类型 | 只使用真实前瞻样本，不使用 replay 样本 |

目标函数分两套口径：通用校准仍以 `avg_period_return` 这类超额口径为主；明日预测（`tomorrow_picks`）则严格按“次日主周期净收益”（`absolute_avg_period_return`）来优化权重，避免用信号口径引入假优势。如果样本不足、因子覆盖不足、OOS 改善不够或只是 `--dry-run`，系统不会写 `.runtime/weights.json`。如果满足条件，会写入 `weights.<strategy>`，运行时由 `WEIGHTS` 自动加载覆盖。

## 10. 样本不足怎么处理

样本不足指验证库里已经走完该策略主周期的有效样本少于 30 条，不是单纯缺少 K 线。

处理方式：

1. 每天盘后保存预测，形成真实前瞻样本。这是金标准。
2. 后台自动分批更新已保存预测的未来结果。它只能更新已有信号，不能凭空生成过去信号。
3. 使用“回放历史补样本”做离线粗筛。回放会用历史量价代理生成样本，但主题、估值、实时热度和行业字段不能完全历史还原。
4. 共识权重优先采信真实样本；真实样本不足时，回放样本只会轻度影响可信度。
5. 样本数少于 30 条时，不要把胜率当成稳定结论；少于 10 条真实样本时，更只能看方向参考。

## 11. 常见空榜和低分原因

| 现象 | 常见原因 |
|---|---|
| 卡脖子打分榜为空 | 当日基础股票池没有命中关键词，或命中票被涨幅/流动性/过热门控过滤；行业 Tab 仍应展示代表票 |
| 量价突破为空 | 历史均线因子缺失、没有多头/20 日新高、实时代理条件不满足、涨幅接近风控上限 |
| 小市值价值为空 | 行情源缺少流通市值/PE/PB，或 PE/PB 非正、流通市值低于护栏 |
| 科技潜力为空 | 名称/行业字段没有匹配科技主题，或 60 日/YTD 涨幅过热 |
| 单票预测偏弱 | 未进入策略榜、风险分过高、市场偏防守、历史因子覆盖不足 |
| 高分但动作是观察 | Agent 风控、数据覆盖、追高/透支风险把动作降级 |

## 13. 最终使用原则

1. 不存在保证赚钱的策略，所有策略只能做概率和风险排序。
2. 主策略应优先看强共识、低风险、Agent 未否决、验证样本逐步改善的股票。
3. 独立策略适合做补充视角，不建议因为某一个策略高分就单票重仓。
4. 被过滤股票不是系统“不会预测”，而是系统认为风险证据优先于正向打分。
5. 样本不足时，先积累真实前瞻样本；历史回放只能辅助筛选，不能替代实盘验证。
