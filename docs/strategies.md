# 盈利策略说明与依据

> **重要免责声明（必读）**
>
> **本项目任何策略都不保证盈利，也无法保证盈利。** 文中引用的收益/胜率多为回测或原作者宣称值，**不是实盘业绩**。回测普遍存在幸存者偏差、未来函数、按信号价成交假设等问题；因子会拥挤、会衰减；交易成本、滑点、T+1、涨跌停无法成交都会侵蚀纸面收益。
>

>
> 结果仅供研究，不构成投资建议。请自行承担投资风险。

## 一、为什么短线偏反转而不是动量？

学术证据（同行评审）显示：在 A 股**短周期横截面**上，经典 12-1 动量效应弱到接近无效甚至为负，真正稳健存活的异象集中在 **反转(reversal)、低波动(low-volatility)、换手/流动性(turnover)** 三类。

- Li, Liu, Liu, Wei, *Replicating and Digesting Anomalies in the Chinese A-Share Market*, **Management Science 2024**：复现 469 个异象，约 83% 在 A 股无显著超额，能存活的主要是波动、换手、反转类。
- A 股 T+1 与散户主导结构导致显著的**短期反向（contrarian）效应**（*Review of International Economics 2024*）。
- 低波动效应在中国市场被描述为**最稳健**的因子之一。

因此本轮新增的策略**有意识地偏向反转/低波/换手回避**，并对原 `short_term` 策略提供了一个**可回测验证的动量→反转修正开关**（`reversal_tilt`，默认关闭，由回测决定是否启用）。

## 二、新增策略逐条说明

每条都给出：**思路 → 主因子 → 数据来源 → 适用市况 → 风险与护栏**。所有策略行都带 `verdict` 评级、`bull/bear` 双分，并接入「策略验证」记分牌。

### 1. 反转低波（`reversal_picks`，`/api/reversal-picks`）

- **思路**：挑"近期跌过头、但波动低、换手不躁动"的票，押短期均值回复。
- **主因子**：`final = 反转(−ret_20d) 0.40 + 低波(−volatility_20d) 0.26 + 换手回避(−turnover_rate) 0.16 + 流动性 0.12 + 不过热 0.06 + 市况`。
- **数据**：日线历史因子（`ret_20d`/`volatility_20d`）+ 实时换手率/成交额。历史因子关闭时回退 `sixty_day_pct`/`amplitude` 近似。
- **适用市况**：震荡/超跌反弹市最优；单边下跌市需配合止损（接飞刀风险，见护栏）。
- **风险与护栏**：`_reversal_risk_penalty` 对"20日腰斩级别下跌""近期放量急杀""当日仍大跌"扣分，避免接实质性利空的下跌。**不能消除基本面暴雷风险**。

### 2. 小市值价值（`smallcap_value_picks`，`/api/smallcap-value-picks`）

- **思路**：A 股历史最强因子之一——小流通市值 + 低估值。
- **主因子**：`final = 小市值(−流通市值) 0.34 + 价值(−PE 0.5 −PB 0.5) 0.22 + 流动性 0.20 + 低波 0.12 + 不过热 0.12 + 市况`。
- **数据**：实时流通市值/PE/PB（东方财富行情源提供；其它降级源缺失时该策略返回空并在 `meta.note` 标注）。
- **适用市况**：小盘风格占优期；**风格切换/流动性收缩时会剧烈回撤**。
- **风险与护栏（关键）**：
  - ⚠️ **2024 年初微盘股两个月崩约 40%**，2015/2017 风格切换也崩过——这是该因子的**固有尾风险，护栏只能缓解不能消除**。
  - ① 流通市值下限 `SMALLCAP_MIN_FLOAT_CAP`（默认 8 亿）过滤纯壳/退市风险；
  - ② 剔除 `PE<=0`（亏损）与 `PB<=0`；
  - ③ 流动性地板（`turnover >= MIN_TURNOVER`）防流动性陷阱；
  - ④ 市场偏防守时（`regime risk_off`）整体降权。
  - **强烈建议分散持仓 + 设硬止损**，不要单押微盘。

### 3. 量价突破·均线多头（`breakout_picks`，`/api/breakout-picks`）

- **思路**：经典技术派趋势确认——均线多头排列（MA5>MA10>MA20>MA60）或创 20 日新高，且量能突破。
- **主因子**：`final = 动量 0.30 + 突破强度(多头排列+新高) 0.26 + 量能突破(vol/5日均量) 0.18 + 趋势 0.16 + 执行性 0.10 + 市况`，再施加过热乘法抑制。
- **数据**：需开启历史因子（`ENABLE_HISTORY_FACTORS=1`）才能算 MA60/多头排列/量能比；关闭时入选会很少。
- **适用市况**：趋势市/强势行情；**震荡市频繁假突破、反复打脸**（A 股大多数时间是震荡市）。
- **风险**：技术派单独使用通常跑不赢买入持有，建议作为**趋势确认过滤**与其它因子叠加使用。

## 三、对原动量策略的修正（`reversal_tilt`）

`short_term` 策略原本 `momentum*0.55` 正向加权，与上述 A 股短线证据方向相反。本轮新增可调键 `WEIGHTS["short_term"]["reversal_tilt"]`（默认 `0.0` = 维持原动量行为）：当 `>0` 时，对"近期涨幅(ret_5d)越高"按比例减分。

**修正幅度不靠拍脑袋，靠回测决定**：

```bash
python -m stock_analyzer.calibrate --compare-momentum
```

该命令用滚动回测对比"动量倾向"vs"反转倾向"两套信号的胜率+收益，**仅当反转确实更优时**才把建议的 `reversal_tilt` 写入 `.runtime/weights.json`；否则保持 0 不动。**回测窗口应覆盖 2024 年初微盘崩盘期**以检验尾风险。

## 四、回测闭环：怎么用记分牌判断"能不能信"

新策略**上线不等于可信**。判断流程：

1. **离线粗筛**：`POST /api/strategy-validation/backfill-samples?strategy=reversal_picks&days=260&replay_days=20&top_n=30` 用历史日线回放生成样本，立即得到命中率/收益/回撤的初步印象。
   - 注意：回放只用量价因子代理。小市值的市值/估值、科技/卡脖子的主题因子**无法历史还原**，回放分是近似（明细里 `strategy_version` 标 `*_replay_v1` 区分）。
2. **真实样本（金标准）**：每天盘后保存预测，次日/后续点「下载历史并验证」更新结果。**只有积累足够真实前瞻样本，命中率才算数。**
3. **看记分牌**：「策略验证」页顶部每策略一张卡，主指标不是统一看次日，而是按策略自己的持有周期扣成本后评估：
   - `tomorrow_picks` / `short_term`：次日；
   - `reversal_picks`：5 日；
   - `swing_picks` / `breakout_picks`：10 日；
   - `long_term` / `position_picks` / `tech_potential` / `chokepoint_picks` / `smallcap_value_picks`：20 日。
4. **样本成熟规则**：长期策略即使已有次日结果，也要等未来交易日达到主周期才计入 `主样本`。例如中长期策略必须有 20 个未来交易日，才进入主周期净胜率。
5. **真实样本优先**：记分牌拆分 `真实/回放`。真实前瞻样本不足时，回放只用于粗筛，共识权重会自动降权，不能把回放胜率当实盘结论。
6. **扣成本后再看胜率**：主周期净收益会扣 `VALIDATION_TRADE_COST_PCT`（默认 0.25%，可用环境变量调整）。主周期净胜率 ≥ 55% 且净收益为正才算“可观察”；真实样本少于 10 条或主样本少于 30 条仍视为样本不足。

## 五、已落实的风控与市况增强

### 1. 止损/持有期退出规则

当前策略验证与 AlphaLite 回测不再只看“到期收盘价”，而是统一模拟：

- 固定主周期：短期/明天为 1 日，反转为 5 日，波段/突破为 10 日，中长期/科技/卡脖子/小市值价值为 20 日。
- 固定止损：默认 `EXIT_STOP_LOSS_PCT=5.0`，盘中低点触发则按止损价退出。
- 止盈：默认 `EXIT_TAKE_PROFIT_PCT=8.0`，盘中高点触发则按止盈价退出。
- 移动止损：默认 `EXIT_TRAILING_STOP_PCT=4.0`，入场后最高价回撤触发则退出。
- 到期退出：未触发风控时按主周期最后一个可用交易日收盘退出。

验证结果会记录 `signal_exit_return`、`exit_reason`、`exit_days`、`exit_date`，并在记分牌输出 `avg_exit_return_net` 和 `win_rate_exit_net`。这解决的是“只选不卖”的评估缺口，但仍不能保证盈利：日线无法精确还原盘中先后顺序，涨跌停不可成交、冲击成本和滑点仍可能让实盘差于回测。

### 2. 市况自适应权重

系统按 `market_regime` 自动切换策略偏好：

- 偏进攻：放大 `momentum`、`trend`、`breakout`、`volume`，更适合强势延续、量价突破、波段策略。
- 偏防守：放大 `lowvol`、`quality`、`liquidity`，压低追涨、突破和量能躁动，更适合低波、质量和可执行性优先。
- 均衡震荡：轻度压缩突破/动量，略偏不过热和质量，避免震荡市频繁追高。

该权重不是简单加展示标签，而是直接参与各策略 `final_score` 排序；结果行会输出 `regime_weight_profile`，用于解释当日为什么某类策略被抬高或压低。

## 六、已知的进一步优化空间（尚未实现）

诚实列出对"实际盈利"帮助更大、但本轮未做的方向：

| 方向 | 说明 |
|---|---|
| **更精细的卖出规则** | 已有固定止损/止盈/移动止损，但还未接入 ATR 止损、跌破 MA20 离场、分批止盈和开盘跳空处理。 |
| **市况权重自动校准** | 已按 `market_regime` 动态切换，但权重仍是经验值；后续应按滚动回测/真实样本自动校准。 |
| **多因子合成 + 去拥挤** | 反转与低波相关性高，可做因子正交/IC 加权合成总分，避免重复押注。 |
| **成本/滑点建模** | 现回测扣固定费率并模拟日线退出，但未建模涨跌停不可成交、冲击成本和盘口容量。 |
