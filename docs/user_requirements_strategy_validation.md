# 用户需求整理：策略验证（明天预测复盘）

## 1. 目标

把“策略验证”从收益复盘工具改造成以“锚点口径 + 今日/累计涨跌”主导的自动化复盘页。

核心目标是：选中某天的明天预测样本后，只看该天预选股票的锚点记录，与今天行情对齐，确认涨跌方向与幅度，并据此给策略方向提示。

## 2. 需求清单

### 2.1 数据口径与显示

1. 保留昨天（或选中日期）“明天预测”保存后的股票代码列表。
2. 明细展示字段仅保留以下 4 个：
   - 锚点价格
   - 锚点涨跌
   - 今日涨跌
   - 锚点到现在
3. 不再展示“次开/次高/次低/次收/净收益/盈利”等次日 OHLC 与净收益字段。
4. 明细必须按选中的“日期批次”展开。
5. 默认展示次日方向相关口径，不再以“净收益/净表现”作为主判断语义。

### 2.2 自动化与更新机制

1. 用户不需要手动“保存今天”“手动回填结果”。
2. 页面只支持后台自动更新。
3. 未拿到真实回填样本时，前端触发并等待后台自动补齐与更新。
4. 后台更新节奏固定为每 30 分钟一次。
5. 每次打开页面后自动开始回填/更新流程，不要求点击任何按钮。

### 2.3 实时数据回填与完整性

1. 明细的“今日涨跌”和“锚点到现在”必须尽量自动回填实时行情。
2. 若行情短时不可得，页面显示为空白占位，不报错阻断业务。
3. 已有样本历史与新行情对齐后，应能在详情页即时反映。

### 2.4 去按钮化 / 页面简化

1. 删除“补历史样本”“刷新建议”“应用建议权重”等按钮及其说明文本。
2. 删除“保存今天”“手动回填结果”等按钮。
3. 保留一个简洁入口：选择日期范围和批次查看。
4. 不在页面中出现“请手工点我触发”的操作文案。

### 2.5 布局与可用性

1. 页面布局去掉多余模块，聚焦核心：当前结论、样本统计、次日涨跌对比、批次、明细。
2. 明细区和批次区布局一致、占位字段齐全、无冗余空白。
3. 重点字段信息优先展示，不堆砌高频指标。

### 2.6 策略迭代逻辑预期

1. 结论逻辑以“次日方向命中率/平均涨跌”为主，不以净收益、扣成本概念主导。
2. 明确告诉用户：样本不足时先观察，不作为交易依据。
3. 使用真实样本优先，回放样本仅作参考。

## 3. 验收标准

1. 进入策略验证页时看到的是自动更新状态，不看到手动按钮。
2. 选中任意保存批次后，明细表仅显示 7 列新字段。
3. 空行情时不再出现空的旧字段列而是显示“-”。
4. 若最新回填未到位，页面提示“后台更新/回填中”，并自动继续刷新。
5. 页面 30 分钟内能看到至少一轮状态推进（后台/前端同步）。
6. 页面文案统一为“锚点口径 + 涨跌口径”。

3. 再优化布局体验。

## 6. 需求演进（按对话顺序）

### 第1轮：复盘口径确认

1. 把“昨天的明天预测”保留样本作为核验对象。
2. 用今日实时信息判断是涨是跌。
3. 说明“昨天的明天预测保存为什么没保存”相关追问（原因追踪）。

### 第2轮：字段压缩与口径聚焦

1. 明细里不再依赖“次开/次高/次低/次收/净收益/盈利”。
2. 只看“锚点样本”并与“今天实时涨跌”对齐。
3. 请求自动化：用户不再手工触发回填。

### 第3轮：全自动化与简化体验

1. 去掉“保留今天和手动回填结果”类动作。
2. “后台每 30 分钟自动回填更新”，不需要手动执行。
3. 去掉“补历史样本、刷新建议、应用建议权重”等非必要按钮与模块。

### 第4轮：布局与字段最终定稿

1. 页面布局优化，减少按钮/噪声，集中在复盘核心。
2. 结果口径统一为：选中的那一天锚点记录 + 今日涨跌 + 锚点到现在涨跌。
3. 让策略验证流程“看锚点走势和方向变化”，并持续自动更新。

## 7. 需求归并为可执行任务（版本化）

### V1（必须先交）

1. 统一口径
   - 明细仅保留：锚点价格、锚点涨跌、今日涨跌、锚点到现在。
   - 结论与得分卡仅使用方向命中率/平均涨跌表达，不使用“净收益/扣成本净收益”作为主口径。
2. 去按钮化
   - 页面不显示手动保存、手工回填、补历史、应用建议等按钮。
3. 自动更新
   - 页面加载后自动触发后台回填与状态轮询（无需用户点击）。
4. 每 30 分钟自动更新机制可见
   - 明确展示“自动回填/更新中/成功”状态文本，且连续失败给出错误提示。

### V2（体验优化）

1. 页面布局收口
   - 统一“批次区 + 次日对比 + 明细区”结构，去掉多余说明模块。
2. 空值体验
   - 涨跌字段回填失败时显示 `-`，不阻塞列表渲染。
3. 点击逻辑
   - 选中批次后默认加载对应明细与图表联动。

### V3（可选增强）

1. 文案标准化
   - 将策略说明页中仍出现的“净收益/净胜率”等泛化表述，统一为“方向口径 / 样本口径”说明风格。
2. 异常审计
   - 记录“今日未回填原因”：行情源不可用、信号样本缺失、回填未返回结果。
