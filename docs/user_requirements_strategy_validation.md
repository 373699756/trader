# 策略验证当前需求

本文档记录当前策略验证页的有效需求。其他历史操作口径已经废弃。

## 1. 验证对象

只验证三类荐股策略：

- 今天推荐：`short_term`
- 明天推荐：`tomorrow_picks`
- 2-5 天推荐：`swing_picks`

页面顶部用三个按钮切换策略。切换后，日期批次、统计卡片、股票明细和 DeepSeek 复盘都必须联动。

## 2. 保存逻辑

- 14:30 后自动按配置间隔保存三类推荐快照。
- 同一天同策略只保留最后一次批次。
- 每类最多保存 18 支；不满足条件可以保存 0 支空批次。
- 15:00 后运行时，使用当天收盘价作为锚点。
- 如果 15:00 后无法取得收盘锚点，拒绝保存为真实回溯锚点。
- 保存成功后自动备份验证数据库。

## 3. 展示逻辑

策略验证页应优先展示：

- 当前选中的策略和批次日期。
- 有效样本数量。
- 主周期胜率，合并显示上涨/下跌个数。
- 主周期平均表现。
- 保存批次列表，每页 5 条历史数据。
- 当前批次股票明细。

无数据时：

- 显示空批次或无样本。
- 不显示上一天或其他策略的数据。
- 表格宽度自适应，最后一列和其他列保持一致背景与居中。

## 4. DeepSeek 复盘

- 每天 15:00 后自动生成一次三类策略复盘。
- 页面支持手动生成当前策略复盘。
- 复盘内容包括样本量、胜率、收益、空批次、待回填样本、过热风险和行业集中。
- 输出影子调参建议，不自动应用到正式策略。

## 5. 性能要求

- 页面刷新进入策略验证时，先轻量加载日期批次。
- 指标、明细、DeepSeek 复盘异步加载。
- 切换今天、明天、2-5 天不应重新跑荐股流。
- 几十条验证数据不应出现 30 秒级等待。

## 6. 验收标准

- 切换策略后三个区域同步更新：统计卡片、日期批次、股票明细。
- 空批次正常显示，不串旧数据。
- 胜率里的上涨/下跌个数来自数据库回填结果。
- 15:00 后能看到当天最后一次保存批次。
- DeepSeek 复盘状态清晰，影子建议不自动生效。
