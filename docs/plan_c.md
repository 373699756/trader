# DeepSeek API 高价值复核优化计划

状态：非权威执行计划，尚未实施。

产品、架构、运行、API、运维和验收以
[软件业务设计文档](software-business-design.md) 为唯一权威；候选、过滤、因子、评分、
风险、DeepSeek、融合、动作和排名以
[荐股策略文档](recommendation-strategy.md) 为唯一权威。本文只记录后续独立交付批次，
不得覆盖两份权威文档；实施引起契约变化时必须先更新对应权威文档。

## 1. 目标与现状判断

- 保留本地确定性候选、硬过滤、风险、融合、动作、TopK 和冻结主链，DeepSeek 继续是
  可降级的复核器，不能新增股票、放宽硬过滤或直接决定买卖。
- 当前 DeepSeek 已具备 188 次物理请求硬上限、原子预算、最多 8 股批处理、Flash/Pro
  双模型、严格 schema、证据引用、缓存、边界优先和失败本地降级，确定性和风险边界
  优于让多 Agent 直接输出买卖结论的开源实现。
- 当前主要浪费是：模型重复评价本地已经确定的量价和价值指标；prompt 重复发送
  ground-truth 与动态字段；固定批次和输出上限容易截断；单股格式错误可能触发整批修复；
  原始缓存受阶段身份限制，跨策略复用不足；158 次正常目标高于实际高价值复核需求。
- 借鉴 TradingAgents 的快慢模型分工、冲突复核和到期结果反馈，以及
  daily_stock_analysis 的风险优先和输入不全时保守降级；不引入多 Agent 轮番讨论、模型
  直接荐股、本地历史回测或 shadow 推荐。
- 本计划不承诺收益提升。只使用正式冻结推荐自然到期后的在线结果统计观察相关性，不重跑
  历史、不自动调整分数或预算规则。

## 2. DeepSeek 职责与结构化事实

- 发布新的策略、引擎、prompt 和 `deepseek_review_v4` schema；旧冻结继续按自身版本回放。
- DeepSeek 从五维自由打分改为提取以下点时事实：
  - 催化方向、重要度、确认状态、作用周期和证据引用；
  - 事件是否尚未反映、部分反映或已经充分反映；
  - 基本面改善、稳定、恶化或未知；
  - 行业政策正向、中性、负向或未知；
  - 监管、减持、解禁、质押、诉讼和业绩等风险事实；
  - 证据冲突、覆盖完整度和 `abstain`。
- 模型不得输出生产扣分、veto、目标价、动作、排名或最终分。自由文本、评级、模型自报
  置信度和情绪形容词只作审计，不能进入生产计算。
- 正向事实必须有官方来源或两个独立可信来源；单一软信息和传闻不得加分。官方负面事实
  允许单源进入本地风险映射。所有风险仍须通过 schema、证据子集、有效期和本地规则校验。

## 3. 本地事实映射与融合

每个 DeepSeek 维度从 50 分开始，由本地纯函数映射事实并限制在 30-70：

| 维度 | 本地映射 |
| --- | --- |
| 催化重要度 | 高/中/低按事实方向记 `±15/±8/±3` |
| 预期差 | 未反映 `±12`、部分反映 `±5`、已充分反映 `0`，符号跟随事实方向 |
| 基本面 | 改善 `+15`、稳定 `+3`、恶化 `-18` |
| 行业政策 | 正向 `+10`、中性 `0`、负向 `-12` |
| 风险覆盖 | 仅在风险数据覆盖完整且未发现风险时 `+5`；实际风险不在本维重复扣分 |

证据质量系数固定为：官方 `1.00`、两个独立可信来源 `0.85`、单一可信来源 `0.65`、
软信息 `0.40`、存在冲突时最高 `0.25`。维度结果继续按质量系数向 50 收缩。至少三个
维度有效且加权证据覆盖不低于 `0.60` 时，模型结果才可应用。

| 策略 | 催化 | 预期差 | 基本面 | 行业政策 | 风险覆盖 |
| --- | ---: | ---: | ---: | ---: | ---: |
| today | 30% | 25% | 10% | 15% | 20% |
| tomorrow | 25% | 25% | 15% | 15% | 20% |
| d25 | 20% | 20% | 25% | 20% | 15% |
| long | 10% | 10% | 35% | 20% | 25% |

实际风险只进入既有 `deepseek_risk_penalty` 和本地映射的 veto，避免在风险维度与融合扣分中
重复计算。固定融合公式、`ROUND_HALF_UP`、`83.40` 验收向量和 `local_score` 不重复扣本地
风险的规则保持不变。

## 4. 候选优先级、批次与缓存

- 三个短策略先联合构造待复核集合并按股票去重，固定优先级为：新高风险、新官方重大
  催化、动作门槛 ±5 分、TopK 边界 ±2 名、排名保护集合、19-30 名中出现新公告或财务
  证据者；其余候选使用本地结果。
- 同一股票的原始事实结果跨 today、tomorrow 和 d25 共享，再按策略权重本地投影。原始
  缓存键包含股票、交易日、证据 manifest、价格反应桶、模型、prompt 和 schema，不包含
  策略、排名、动作阈值或普通 merge epoch；事实、价格反应桶、模型、prompt 或 schema
  变化时失效。
- 每股最多发送 12 条证据：风险 3、财务 2、公告/新闻 5、结构化行情 1、反证 1。共享
  市场背景、schema 和规则只发送一次，删除 `features/values` 及 ground-truth 的重复表达。
- 使用 1-8 股自适应批次，常规请求优先凑满至少 2 股，候选仅剩 1 股或 emergency 时允许
  单股提交；单股输入不超过约 1200 token，整批输入不超过约 12000 token；输出上限按
  `clamp(300 + 180 × 股票数, 800, 2200)` 计算，Pro 每批最多 4 股。
- JSON 根结构有效时逐股校验并立即缓存有效结果，只修复缺失或非法股票；长度截断时保留
  已完整结束的结果并修复剩余项；完全不可解析时整批最多修复一次。重试、失败、超时和
  修复仍全部计入物理请求。

## 5. Flash、Pro 与预算

- V4-Flash 负责普通事实提取。V4-Pro 只处理：Flash 结果可能跨动作门槛、官方正负证据
  冲突、高风险 veto、模型事实方向与本地下行保护冲突。Pro 只能确认、删除或降低正向
  事实，不能放宽硬过滤、解除本地下行保护或新增无证据利好。
- 保留 188 次全局原子硬上限和现有桶上限作为异常余量；新增正常日软目标：共享预热 12、
  today 24、tomorrow 16、d25 14、long 4、Pro 合计最多 8、emergency 最多 5，正常日
  预计使用 70-83 次物理请求。
- 达到软目标后，只允许新高风险、冻结边界变化和已经开始请求的必要重试继续消耗；候选
  不足、证据未变化或缓存命中时不得为空耗目标。

## 6. 在线结果观察与可观测性

- 正式推荐到期后，将已有净超额收益、MAE 和严重回撤与调用原因、事实类型、模型分变化
  方向关联统计。不生成本地回溯或影子推荐，不把相关性表述为因果收益提升。
- 在线统计只用于观察和人工决定后续策略版本，不自动修改权重、事实映射、动作阈值、
  TopK 或调用优先级。
- `/api/status` 增加平均批次大小、每股输入/输出 token、部分成功率、局部修复次数、
  prompt cache token、跨策略复用数、事实数/请求和各优先级预算消耗；不得在普通推荐
  响应暴露内部模型载荷。

## 7. 实施批次与验收

本计划作为一个完整策略交付章节实施，不与当前工程重构章节混合。先更新两份权威文档、
schema 和失败先行测试，再实现领域映射、复核编排、缓存、预算和状态投影，最后完成回放、
兼容和发布验证。

测试至少覆盖：

- 同一证据在三个短策略间只产生一次物理事实请求；
- 单股 schema 错误不破坏同批有效结果，只修复错误子集；
- 低价值候选不消耗预算，重大新风险可使用 emergency；
- 软正向信息不加分，官方负面事实可映射风险；
- 自由文本、评级和模型置信度不能改变生产结果；
- Pro 不能放宽过滤、解除下行保护或新增无证据利好；
- 重试、失败、超时、修复和并发均受 188 次原子上限约束；
- 冻结不可变、缓存失效、全本地降级和固定融合结果 `83.40` 保持通过。

退出前运行 Ruff、mypy、全部测试、构建、仓库外 wheel 安装和三档桌面验收。旧 release
作为完整回退单位，不修改旧冻结数据。只有在线样本形成后才能报告观察结果；在没有可靠
对照的情况下，不宣称本计划已经提高实际荐股收益。
